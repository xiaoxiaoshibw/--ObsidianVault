# 段式存储管理

## 概念
按程序的**逻辑单位**（段）来分配内存，每个段是独立的逻辑单位（主程序、子程序、数据等）。

## 段表
| 字段 | 说明 |
|------|------|
| 段号 | 段的编号 |
| 段长 | 段的长度 |
| 基址 | 段在内存中的起始地址 |

## 地址结构
```
┌──────────┬────────────┐
│  段号 S  │ 段内偏移 W  │
└──────────┴────────────┘
```

## 特点
- **有外部碎片**（无内部碎片）
- 方便共享和保护（按逻辑单位）
- 地址转换：查段表 → 基址 + 偏移

## 分页 vs 分段

| 对比 | 分页 | 分段 |
|------|------|------|
| 划分方式 | 物理划分（固定大小） | 逻辑划分（可变大小）|
| 大小 | 固定（页大小） | 不定（段长）|
| 碎片 | 内部碎片 | 外部碎片 |
| 共享 | 不便 | 方便 |

## 📝 本章习题
**填空题2**：某段式存储管理系统中，地址长度为32位，若允许的最大段长为64KB，则段号占 **16** 位。
> 解：64KB = 2¹⁶B，所以段内偏移占16位，段号 = 32 - 16 = 16位

**填空题7**：在采用段式内存管理系统中，处理器给出的有效地址为16个比特位，系统允许的最大段长为8KB，则逻辑地址空间可达 **64**（十进制）KB，允许段的最大数量 **8**（十进制）。对于一个的有效逻辑地址9BCD（十六进制），则该地址对应的段号为 **4**（十进制）。
> 解：8KB = 2¹³B，段内偏移13位；段号 = 16-13 = 3位 → 最多8段；总空间 = 8×8KB = 64KB
> 9BCDH：二进制 1001 1011 1100 1101，高3位=100=4
