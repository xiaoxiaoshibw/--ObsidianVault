## 4.1 同步与互斥

### 先讲两个生活故事

**故事一：打印机争夺战（互斥）**

你和室友共用一台打印机。你在打印论文，室友也想打印。如果两个人同时按"打印"，纸张就会乱码、内容交错在一起。

解决办法是什么？**一个人用完，下一个人再用**。这就是**互斥**——同一个资源，一次只能一个人用。

> **互斥（Mutual Exclusion）**：多个进程**竞争**同一个**独占资源**时，一次只允许一个进程使用。

**故事二：流水线工人（同步）**

做奶茶：A负责加茶 → B负责加糖 → C负责封口。

- B必须在A完成之后才能工作
- C必须在B完成之后才能工作
- 如果A还没加茶，B就去加糖，那就做错了

这就是**同步**——大家**协作完成同一个任务**，有严格的**先后顺序**。

> **同步（Synchronization）**：多个进程**协作**完成同一任务，按约定的**顺序**执行。

### 表格对比：同步 vs 互斥

| | 互斥 | 同步 |
|---|---|---|
| 关系 | **竞争**关系 | **协作**关系 |
| 核心 | 抢同一个资源 | 配合完成任务 |
| 例子 | 抢打印机、抢共享变量 | 流水线、生产者-消费者 |
| 信号量初值 | 通常为 **1** | 通常为 **0** |

### 并发进程之间存在两种关系

多个进程同时运行时，它们之间要么是**同步关系**（协作），要么是**互斥关系**（竞争），或者**兼而有之**。

> **一个记忆口诀**：
> - 互斥 = 你死我活（抢东西）
> - 同步 = 你侬我侬（配合作）

## 4.2 信号量机制（必考！）

> ⚠️ **这是本章最核心的内容，大题全考这个！**

### 信号量是什么？

**信号量（Semaphore）** 就是一个**整型变量 `S`**，用来表示"当前有多少个资源可用"。

> **S = 当前可用资源的数量**

但是——你不能随便修改 S，只能通过两个特殊的操作来改它：**P 操作**和 **V 操作**。

### P 操作（wait / 申请资源）

```
P(S):
  S = S - 1          // 申请一个资源，S减1
  if (S < 0) then
    当前进程进入等待队列（被阻塞）
```

**大白话**：你想用资源，先跟系统说一声。系统说"好的，可用资源减1"。如果资源已经没了（S<0），你就去排队等着。

### V 操作（signal / 释放资源）

```
V(S):
  S = S + 1          // 释放一个资源，S加1
  if (S <= 0) then
    唤醒等待队列中的一个进程
```

**大白话**：你用完了资源，还回去。系统说"好的，可用资源加1"。回头一看发现有别人在等这个资源，就喊一个人过来用。

### ⚠️ 核心口诀：P减V加！

| 操作 | 含义 | 对S的影响 | 触发条件 |
|---|---|---|---|
| **P(S)** | 申请（Proberen） | **S = S - 1** | S<0 时阻塞 |
| **V(S)** | 释放（Verhogen） | **S = S + 1** | S≤0 时唤醒 |

> **P = 申请 = 减1**
> **V = 释放 = 加1**
> 记不住？P 像你"**P**ull"资源出来（拿走），V 像你"**V**olume"调大（加回去）。

### 互斥的经典写法

```
P(mutex)      // 进入区：申请
  临界区代码     // 访问共享资源
V(mutex)      // 退出区：释放
```

一个进程想进入临界区，必须先 P(mutex)，出来时 V(mutex)。

## 4.3 PV操作的三种读法

很多同学卡在这里：给了 S 的值，怎么知道系统当前是什么状态？

### S > 0：有资源可用

**S = 当前剩余资源数量**

> 例：S = 3，表示还有 3 个资源空闲，新来的进程可以直接用，不用排队。

### S = 0：资源刚好用完

**不多不少，刚好没有空闲资源了，但也没人在等。**

> 例：S = 0，表示资源刚好被占满。新来的进程如果申请，S 变成 -1，就要排队了。

### S < 0：有人在排队等资源

**|S| = 当前在等待队列中阻塞的进程数量**

> 例：S = -3，表示资源不仅用完了，还有 **3 个进程**在排队等着呢。

### 一张图看懂

| S 的值 | 含义 | 可用资源 | 等待进程 |
|---|---|---|---|
| S = 5 | 有 5 个资源空闲 | 5 个 | 0 个 |
| S = 1 | 还有 1 个资源 | 1 个 | 0 个 |
| **S = 0** | **刚好用完** | **0 个** | **0 个** |
| S = -1 | 1 个进程在等 | 0 个 | 1 个 |
| S = -5 | 5 个进程在等 | 0 个 | 5 个 |

### ⚠️ 必考选择题套路

题目说："信号量初值为 3，经过多次 PV 操作后，当前值为 -1，问有几个进程在等待？"

**答案**：S = -1，等待进程数 = |-1| = **1 个**。

## 4.4 生产者-消费者问题（必考模板）

> ⚠️ **这是PV操作最经典的题目，几乎所有学校都考！学会这道题，PV大题就拿下一半。**

### 场景

- **生产者**：往缓冲区里放数据（好比厨师做菜放上取餐台）
- **消费者**：从缓冲区里取数据（好比顾客从取餐台端走菜）
- **缓冲区**：有大小限制（取餐台只能放 n 个菜）

### 三个信号量

| 信号量 | 初值 | 含义 |
|---|---|---|
| **mutex** = 1 | 1 | 互斥：一次只能一个人动缓冲区 |
| **empty** = n | n | 空位数量（还能放几个菜） |
| **full** = 0 | 0 | 满位数量（已有几个菜） |

> 记忆：empty = 空 = 还能放几个，full = 满 = 已经放了几个

### 伪代码

```
semaphore mutex = 1;    // 互斥锁
semaphore empty = n;    // 空闲缓冲区数量
semaphore full = 0;     // 已占缓冲区数量

// ===== 生产者 =====
producer() {
  while (true) {
    生产一个产品;
    P(empty);            // ① 申请一个空位（没空位就等）
    P(mutex);            // ② 锁门，别人不能动
       把产品放入缓冲区;
    V(mutex);            // ③ 开门
    V(full);             // ④ 通知消费者：多了一个菜
  }
}

// ===== 消费者 =====
consumer() {
  while (true) {
    P(full);             // ① 申请一个满位（没菜就等）
    P(mutex);            // ② 锁门
       从缓冲区取产品;
    V(mutex);            // ③ 开门
    V(empty);            // ④ 通知生产者：空了一个位
    消费产品;
  }
}
```

### ⚠️ 核心考点：为什么先P(资源)再P(互斥)？

看这两行：
```
P(empty);   // 先申请资源（空位）
P(mutex);   // 再申请互斥锁
```

**顺序不能颠倒！** 如果写成：

```
P(mutex);   // 先锁门
P(empty);   // 再等空位 —— ⚠️ 如果没空位就死锁了！
```

**为什么？**
- 如果缓冲区满了（empty=0），生产者锁了门（mutex=0），然后等空位
- 消费者想取数据，但门被锁了进不去（mutex=0），消费者没法取
- 生产者等消费者取（等 empty），消费者等生产者开门（等 mutex）
- **两个进程互相等待 → 死锁！**

> **口诀：先资源，后互斥，顺序乱了就死锁！**

### 总结模板

```
生产者：P(资源) → P(互斥) → 操作 → V(互斥) → V(资源)
消费者：P(资源) → P(互斥) → 操作 → V(互斥) → V(资源)
```

两个的**结构一模一样**，只是资源不同：
- 生产者等的资源是 **empty**（空位），释放的是 **full**（满位）
- 消费者等的资源是 **full**（满位），释放的是 **empty**（空位）

## 4.5 PV操作计算题

> ⚠️ **送分题！填空题必考，一定要拿到这几分。**

### 核心公式

```
最终S = 初始S - P操作次数 + V操作次数
```

**口诀：P减V加，一算就对。**

### 例题1

信号量 S 初值为 6，多个并发进程先依次执行了 8 次 P 操作，又执行了 5 次 V 操作，此时 S 的值为？

**解**：
```
S = 6 - 8 + 5 = 3
```

### 例题2

信号量 S 初值为 3，经过多次 PV 操作后，当前值为 -1，此时等待进入临界区的进程数是？

**解**：
```
S = -1，等待进程数 = |S| = 1
```

### 例题3

设有 n 个进程共享一个临界区，若最多允许 m 个进程同时进入临界区（m < n），则信号量初值为？

**解**：
- 初值 = **m**（最多允许 m 个进程同时进）
- 最大值 = m（没人用时）
- 最小值 = m - n（全部 n 个进程都来申请）
- 取值范围：**[m-n, m]**

## 4.6 作业题全讲解（三缓冲同步模型）

> ⚠️ **这道题是很多学校的期末大题，把生产者-消费者扩展成了三级流水线。**

### 题目描述

三个并发进程 P1、P2、P3，两个单缓冲区 B1 和 B2，容量均为 M。

```
P1 → B1 → P2 → B2 → P3
```

- **P1**：输入数据，判断正负。正数 → B2，非正 → B1
- **P2**：从 B1 取数据，加工后送 B2
- **P3**：从 B2 取数据，输出

### 第一步：确定信号量

每一对"生产者-消费者"都需要两组信号量：资源信号量 + 互斥信号量。

**B1 缓冲区**（P1 写 → P2 读）：
| 信号量 | 初值 | 含义 |
|---|---|---|
| empty1 | M | B1 空位数量 |
| full1 | 0 | B1 满位数量 |
| mutex1 | 1 | 互斥访问 B1 |

**B2 缓冲区**（P1 和 P2 写 → P3 读）：
| 信号量 | 初值 | 含义 |
|---|---|---|
| empty2 | M | B2 空位数量 |
| full2 | 0 | B2 满位数量 |
| mutex2 | 1 | 互斥访问 B2 |

### 第二步：P1 进程（输入分拣）

```
if (data > 0) {
  // 正数 → 放入 B2
  P(empty2);          // 申请 B2 空位
  P(mutex2);          // 锁 B2
  放数据 data 到 B2 中;
  V(mutex2);          // 开 B2 锁
  V(full2);           // B2 满位 +1
} else {
  // 非正 → 放入 B1
  P(empty1);          // 申请 B1 空位
  P(mutex1);          // 锁 B1
  放数据 data 到 B1 中;
  V(mutex1);          // 开 B1 锁
  V(full1);           // B1 满位 +1
}
```

### 第三步：P2 进程（加工转发）

```
// 从 B1 取数据
P(full1);             // 等 B1 有数据
P(mutex1);            // 锁 B1
从 B1 中取出数据 data;
V(mutex1);            // 开 B1 锁
V(empty1);            // B1 空位 +1

// 加工成 DATA
将 data 处理成 DATA;

// 送 B2
P(empty2);            // 申请 B2 空位
P(mutex2);            // 锁 B2
将 DATA 放入 B2 中;
V(mutex2);            // 开 B2 锁
V(full2);             // B2 满位 +1
```

### 第四步：P3 进程（输出）

```
// 从 B2 取数据
P(full2);             // 等 B2 有数据
P(mutex2);            // 锁 B2
从 B2 中取出数据;
V(mutex2);            // 开 B2 锁
V(empty2);            // B2 空位 +1

// 输出
打印数据;
```

### 检查：信号量初值判断

题目常考填空题，问信号量初值。口诀：

| 信号量 | 初值 | 原因 |
|---|---|---|
| empty | **M**（缓冲区容量） | 一开始全是空的 |
| full | **0** | 一开始一个数据都没有 |
| mutex | **1** | 互斥锁，只能一个人用 |

### 解题套路总结

做这类PV大题，按照以下"四步法"：

1. **画图**：画进程和数据流向
2. **找缓冲**：每一段"写→读"关系对应一组 full/empty
3. **写模板**：P(资源) → P(互斥) → 操作 → V(互斥) → V(资源)
4. **检查死锁**：确保 P(资源) 写在 P(互斥) 前面

## 附：核心公式速查卡

| 公式 | 说明 |
|---|---|
| **最终 S = 初始 S - P 次数 + V 次数** | 计算题直接套 |
| **S > 0** → 有 S 个资源空闲 | 资源充足 |
| **S = 0** → 资源刚好用完 | 不剩也不缺 |
| **S < 0** → \|S\| 个进程在等待 | 有人排队 |
| **互斥：初值 = 1** | 一次一个人进 |
| **同步：初值 = 0** | 先等后执行 |
| **资源计数：初值 = 资源数** | 比如 n 个缓冲区 |

> 📌 **最后叮嘱**：PV操作就是"申请减1，释放加1"。P 像你把手伸进桶里**拿**东西（桶里东西减少），V 像你**放**东西回去（桶里东西增加）。期末考试大题只要按模板写，分步得分，就能拿满分！
