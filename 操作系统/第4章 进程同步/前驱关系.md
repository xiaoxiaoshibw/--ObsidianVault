# 前驱关系（进程同步）

## 概念
进程之间存在**先后执行顺序**的约束关系，即一个进程必须在某个/些进程完成后才能开始执行。

## 前驱图
```
    P1
   /  \
  P2  P3
   \  /
    P4
     |
    P5
```
P1 → P2、P3 → P4 → P5

## 用信号量实现前驱关系

```
semaphore a12 = 0, a13 = 0, a24 = 0, a34 = 0, a45 = 0;

P1() { 完成; V(a12); V(a13); }
P2() { P(a12); 完成; V(a24); }
P3() { P(a13); 完成; V(a34); }
P4() { P(a24); P(a34); 完成; V(a45); }
P5() { P(a45); 完成; }
```

## 关键
- 每个前驱关系对应一个**信号量**，初始值 = 0
- 前驱执行完 → **V**（发信号）
- 后继执行前 → **P**（等信号）
