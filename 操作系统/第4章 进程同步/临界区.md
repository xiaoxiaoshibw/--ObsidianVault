# 临界区

## 定义
**临界区（Critical Section）** 是访问**临界资源**的那段**代码**。

## 访问临界区的四个原则
| 原则 | 说明 |
|------|------|
| **1. 忙则等待** | 已有进程在临界区时，其他进程必须等待 |
| **2. 空闲则入** | 无进程在临界区时，可立即进入 |
| **3. 有限等待** | 等待进入临界区的进程不会无限等待 |
| **4. 让权等待（可选）** | 不能进入时释放 CPU（非忙等）|

## 代码结构
```
entry section      // 进入区（检查是否可以进入）
  critical section  // 临界区（访问共享资源）
exit section       // 退出区（释放资源）
remainder section  // 剩余区
```
