### 🔷 类型1：进程状态转换图

**题目**：下图是一个进程状态转换图，请回答：
```
    ┌──────┐
    │ 就绪 │◄──────┐
    └──┬───┘       │
       │ ①         │ ③
       ▼           │
    ┌──────┐   ┌───┴───┐
    │ 运行 │──►│ 就绪  │
    └──┬───┘   └───────┘
       │ ②
       ▼
    ┌──────┐
    │ 阻塞 │
    └──┬───┘
       │ ④
       └───────► 就绪
```
(1) 请填写①、②、③、④分别代表什么事件？
(2) 进程从阻塞态能否直接转换到运行态？为什么？
(3) 引入"创建态"和"终止态"的意义是什么？

<details>
<summary>点击查看答案</summary>

(1)
- ① **进程调度（调度程序选择运行）**
- ② **等待事件（如请求I/O、等待资源）**
- ③ **时间片到 或 高优先级进程抢占**
- ④ **事件发生（如I/O完成、资源可用）**

(2) **不能**。阻塞态必须首先转换到就绪态（事件发生唤醒），再由调度程序选择进入运行态。CPU调度只能从就绪队列中选择进程。

(3)
- **创建态**：进程正在被创建，PCB已分配但尚未就绪，避免未初始化进程被调度
- **终止态**：进程已结束，等待系统回收资源，便于善后处理
</details>

### 🔷 类型2：PV操作（生产者-消费者）

**题目**：某系统中有2个生产者进程和3个消费者进程共享一个可存放10件产品的缓冲区（每次只能放/取1件）。请用P、V操作实现同步互斥。

(1) 应设置哪些信号量？初值各为多少？
(2) 写出生产者和消费者的代码框架。

<details>
<summary>点击查看答案</summary>

(1) 信号量设置：
| 信号量 | 初值 | 作用 |
|--------|------|------|
| `mutex` | 1 | 互斥访问缓冲区 |
| `empty` | 10 | 空缓冲区数量 |
| `full` | 0 | 满缓冲区数量 |

(2) 代码：
```
// 生产者
producer() {
  while (true) {
    生产一个产品;
    P(empty);       // 申请空位
    P(mutex);       // 进入临界区
      放入缓冲区;
    V(mutex);       // 退出临界区
    V(full);        // 增加满位
  }
}

// 消费者
consumer() {
  while (true) {
    P(full);        // 申请产品
    P(mutex);       // 进入临界区
      取出产品;
    V(mutex);       // 退出临界区
    V(empty);       // 增加空位
    消费产品;
  }
}
```
⚠ 先 P(资源信号量) 后 P(互斥信号量)，顺序不可颠倒，否则可能死锁。
</details>

### 🔷 类型3：银行家算法

**题目**：系统中有5个进程P0~P4和3类资源A(10)、B(5)、C(7)。T0时刻资源分配情况如下：

| 进程 | Allocation（已分配） | Need（还需） | Available（可用） |
|------|--------------------|-------------|-----------------|
| P0 | (0,1,0) | (7,4,3) | (3,3,2) |
| P1 | (2,0,0) | (1,2,2) | |
| P2 | (3,0,2) | (6,0,0) | |
| P3 | (2,1,1) | (0,1,1) | |
| P4 | (0,0,2) | (4,3,1) | |

(1) T0时刻系统是否安全？若安全，给出一个安全序列。
(2) 若P1请求资源 Request(1,0,2)，能否分配？为什么？

<details>
<summary>点击查看答案</summary>

(1) **安全**。安全序列：P1 → P3 → P4 → P0 → P2（或P1→P3→P0→P2→P4等）

计算过程：
- Available = (3,3,2)
- P1: Need(1,2,2) ≤ Available(3,3,2) → 分配 → 完成后释放 → Available = (5,3,2)
- P3: Need(0,1,1) ≤ (5,3,2) → 分配 → 完成后释放 → Available = (7,4,3)
- P4: Need(4,3,1) ≤ (7,4,3) → 分配 → 完成后释放 → Available = (7,4,5)
- P0: Need(7,4,3) ≤ (7,4,5) → 分配 → 完成后释放 → Available = (7,5,5)
- P2: Need(6,0,0) ≤ (7,5,5) → 分配 → 完成后释放 → Available = (10,5,7)

(2) **可以分配**。
- Request(1,0,2) ≤ Need(1,2,2) ✅
- Request(1,0,2) ≤ Available(3,3,2) ✅
- 试分配后：Available=(2,3,0)，P1的Allocation=(3,0,2)，Need=(0,2,0)
- 此时可找到安全序列 P1→P3→...，系统安全，故可分配
</details>

### 🔷 类型5：磁盘调度算法

**题目**：磁盘有200个磁道（0~199），当前磁头在 **98** 号磁道，向磁道号增加方向移动。磁盘请求队列：**86, 147, 91, 177, 94, 150, 102, 175, 130**

请分别计算 SSTF、SCAN（电梯算法）、C-SCAN 的磁头移动总距离。

<details>
<summary>点击查看答案</summary>

**(1) SSTF（最短寻道时间优先）**
每次选择距离最近的磁道：
```
98 → 102(4) → 94(8) → 91(3) → 86(5) → 130(44) → 147(17) → 150(3) → 175(25) → 177(2)
```
移动总距离 = 4+8+3+5+44+17+3+25+2 = **111**

**(2) SCAN（电梯算法）**
向增加方向移动，处理沿途请求，到达最大后掉头：
```
98 → 102 → 130 → 147 → 150 → 175 → 177 → 199(终点) → 94 → 91 → 86
```
移动总距离 = (199-98) + (199-86) = 101 + 113 = **214**

**(3) C-SCAN（循环扫描）**
向增加方向移动，处理沿途请求，到达终点后直接回0，再重新开始：
```
98 → 102 → 130 → 147 → 150 → 175 → 177 → 199(终点) → 0(直接返回) → 86 → 91 → 94
```
移动总距离 = (199-98) + (199-0) + (94-86) = 101 + 199 + 8 = **308**

**结论**：SSTF(111) < SCAN(214) < C-SCAN(308)，但SSTF可能导致饥饿。
</details>

### 🔷 类型6：进程调度算法

**题目**：有5个进程P1~P5，到达时间和执行时间如下表：

| 进程 | 到达时间 | 执行时间 |
|------|---------|---------|
| P1 | 0 | 10 |
| P2 | 1 | 6 |
| P3 | 2 | 2 |
| P4 | 3 | 4 |
| P5 | 4 | 8 |

请分别用 FCFS、SJF（非抢占）、RR（时间片q=3）计算各进程的周转时间和平均周转时间。

<details>
<summary>点击查看答案</summary>

**(1) FCFS**：按到达顺序执行
```
P1(0~10) → P2(10~16) → P3(16~18) → P4(18~22) → P5(22~30)
```
| 进程 | 完成时间 | 周转时间 |
|------|---------|---------|
| P1 | 10 | 10-0=10 |
| P2 | 16 | 16-1=15 |
| P3 | 18 | 18-2=16 |
| P4 | 22 | 22-3=19 |
| P5 | 30 | 30-4=26 |

平均周转时间 = (10+15+16+19+26)/5 = **17.2**

**(2) SJF**：按执行时间短优先
```
P1(0~10) → P3(10~12) → P4(12~16) → P2(16~22) → P5(22~30)
```
| 进程 | 完成时间 | 周转时间 |
|------|---------|---------|
| P1 | 10 | 10 |
| P3 | 12 | 10 |
| P4 | 16 | 13 |
| P2 | 22 | 21 |
| P5 | 30 | 26 |

平均周转时间 = (10+10+13+21+26)/5 = **16.0**

**(3) RR (q=3)**
```
P1(0~3) P2(3~6) P3(6~8) P4(8~11) P5(11~14)
P1(14~17) P2(17~20) P4(20~21) P5(21~24)
P1(24~27) P2(27~28) P5(28~30)
```
| 进程 | 完成时间 | 周转时间 |
|------|---------|---------|
| P1 | 27 | 27 |
| P2 | 28 | 27 |
| P3 | 8 | 6 |
| P4 | 21 | 18 |
| P5 | 30 | 26 |

平均周转时间 = (27+27+6+18+26)/5 = **20.8**

**对比**：SJF(16.0) < FCFS(17.2) < RR(20.8)
</details>
