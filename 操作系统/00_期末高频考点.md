### 🔷 类型1：进程状态转换图

**题目**：下图是一个进程状态转换图，请回答：
```
    ┌──────┐
    │ 就绪 │◄──────┐
    └──┬───┘       │
       │ ①         │ ③
       ▼           │
    ┌──────┐   ┌───┴───┐
    │ 运行 │──►│ 就绪  │
    └──┬───┘   └───────┘
       │ ②
       ▼
    ┌──────┐
    │ 阻塞 │
    └──┬───┘
       │ ④
       └───────► 就绪
```
(1) 请填写①、②、③、④分别代表什么事件？
(2) 进程从阻塞态能否直接转换到运行态？为什么？
(3) 引入"创建态"和"终止态"的意义是什么？

> [!answer]- 点击查看答案
> 
> (1)
> - ① **进程调度（调度程序选择运行）**
> - ② **等待事件（如请求I/O、等待资源）**
> - ③ **时间片到 或 高优先级进程抢占**
> - ④ **事件发生（如I/O完成、资源可用）**
> 
> (2) **不能**。阻塞态必须首先转换到就绪态（事件发生唤醒），再由调度程序选择进入运行态。CPU调度只能从就绪队列中选择进程。
> 
> (3)
> - **创建态**：进程正在被创建，PCB已分配但尚未就绪，避免未初始化进程被调度
> - **终止态**：进程已结束，等待系统回收资源，便于善后处理

### 🔷 类型2：PV操作（生产者-消费者）

**题目**：某系统中有2个生产者进程和3个消费者进程共享一个可存放10件产品的缓冲区（每次只能放/取1件）。请用P、V操作实现同步互斥。

(1) 应设置哪些信号量？初值各为多少？
(2) 写出生产者和消费者的代码框架。

> [!answer]- 点击查看答案
> 
> **(1) 信号量设置：**
> 
> | 信号量 | 初值 | 作用 |
> |---|---:|---|
> | `mutex` | 1 | 互斥访问缓冲区 |
> | `empty` | 10 | 空缓冲区数量 |
> | `full` | 0 | 满缓冲区数量 |
> 
> **(2) 代码：**
> 
> ```c
> // 生产者
> producer() {
>   while (true) {
>     生产一个产品;
>     P(empty);
>     P(mutex);
>       放入缓冲区;
>     V(mutex);
>     V(full);
>   }
> }
> 
> // 消费者
> consumer() {
>   while (true) {
>     P(full);
>     P(mutex);
>       取出产品;
>     V(mutex);
>     V(empty);
>     消费产品;
>   }
> }
> ```
> 
> **易错点：**
> 
> 先 `P(资源信号量)`，后 `P(互斥信号量)`，顺序不可颠倒，否则可能死锁。

### 🔷 类型3：银行家算法

**题目**：系统中有5个进程P0~P4和3类资源A(10)、B(5)、C(7)。T0时刻资源分配情况如下：

| 进程  | Allocation（已分配） | Need（还需） | Available（可用） |
| --- | --------------- | -------- | ------------- |
| P0  | (0,1,0)         | (7,4,3)  | (3,3,2)       |
| P1  | (2,0,0)         | (1,2,2)  |               |
| P2  | (3,0,2)         | (6,0,0)  |               |
| P3  | (2,1,1)         | (0,1,1)  |               |
| P4  | (0,0,2)         | (4,3,1)  |               |

(1) T0时刻系统是否安全？若安全，给出一个安全序列。
(2) 若P1请求资源 Request(1,0,2)，能否分配？为什么？

> [!answer]- 点击查看答案
> 
> (1) **安全**。安全序列：P1 → P3 → P4 → P0 → P2（或P1→P3→P0→P2→P4等）
> 
> 计算过程：
> - Available = (3,3,2)
> - P1: Need(1,2,2) ≤ Available(3,3,2) → 分配 → 完成后释放 → Available = (5,3,2)
> - P3: Need(0,1,1) ≤ (5,3,2) → 分配 → 完成后释放 → Available = (7,4,3)
> - P4: Need(4,3,1) ≤ (7,4,3) → 分配 → 完成后释放 → Available = (7,4,5)
> - P0: Need(7,4,3) ≤ (7,4,5) → 分配 → 完成后释放 → Available = (7,5,5)
> - P2: Need(6,0,0) ≤ (7,5,5) → 分配 → 完成后释放 → Available = (10,5,7)
> 
> (2) **可以分配**。
> - Request(1,0,2) ≤ Need(1,2,2) ✅
> - Request(1,0,2) ≤ Available(3,3,2) ✅
> - 试分配后：Available=(2,3,0)，P1的Allocation=(3,0,2)，Need=(0,2,0)
> - 此时可找到安全序列 P1→P3→...，系统安全，故可分配

### 🔷 类型8：有效访问时间（EAT）计算

**题目**：分页存储管理系统具有快表，内存访问时间为 **2μs**，检索快表时间为 **0.5μs**。若快表的命中率为 **80%**，且忽略快表更新时间，则有效访问时间是？

> [!answer]- 点击查看答案
> 
> **公式**：EAT = 命中率 × (快表时间 + 一次内存访问) + (1-命中率) × (快表时间 + 两次内存访问)
> 
> 代入：
> - EAT = 0.8 × (0.5 + 2) + 0.2 × (0.5 + 2 + 2)
> - = 0.8 × 2.5 + 0.2 × 4.5
> - = 2.0 + 0.9
> - = **2.9 μs** ✅
> 
> **⚠️ 易错**：
> - ❌ 忘记加快表检索时间 0.5μs → EAT=2.4 (错)
> - ❌ 未命中时只算一次内存访问 → EAT=2.5 (错)
> - ✅ **每次访问必须先查快表**，所以每次都要加 TLB 检索时间

### 🔷 类型9：段式地址空间计算

**题目**：在采用段式内存管理系统中，处理器给出的有效地址为 **16个比特位**，系统允许的最大段长为 **8KB**，则：
1. 逻辑地址空间可达多少 KB？
2. 允许段的最大数量是多少？
3. 对于有效逻辑地址 **9BCD（H）**，该地址对应的段号为多少（十进制）？

> [!answer]- 点击查看答案
> 
> **(1) 段号位数**：
> - 有效地址 16 位
> - 最大段长 8KB = $2^{13}$B → 段内偏移占 **13 位**
> - 段号占位数 = 16 − 13 = **3 位**
> 
> **(2) 段的最大数量**：
> - $2^{3}$ = **8 段**
> - 逻辑地址空间 = 8 × 8KB = **64 KB**
> 
> **(3) 段号计算**：
> - 9BCD（H）= 1001 1011 1100 1101（二进制，16位）
> - 高3位 = 段号 = 100（二进制）= **4**（十进制）✅
> - 低13位 = 段内偏移 = 1B 1100 1101 = 1BCD（H）
> 
> **⚠️ 易错**：
> - ❌ 直接按 9BCD 的十进制除以 8KB
> - ✅ 按**二进制位**拆分：高3位是段号，低13位是偏移
> - 逻辑地址空间 = 段数 × 段长，不是 $2^{16}$

### 🔷 类型10：信号量取值范围

**题目**：15个进程共享同一程序段，而每次最多允许 **4个** 进程进入该程序段。若用 P、V 操作同步机制，则记录型信号量 S 的取值范围是多少？最小值 = ？最大值 = ？

> [!answer]- 点击查看答案
> 
> **公式**：
> - 信号量初值 = 最大允许并发数 = **4**
> - 取值范围 = [初值 − 进程总数, 初值] = [4 − 15, 4] = **[-11, 4]**
> 
> **含义**：
> - S = 4：没有进程在临界区
> - S = 0：刚好有4个进程在临界区
> - S = -11：所有15个进程都试图进入，但有4个已在临界区，11个在等待
> 
> **答案为**：最小值 = **-11**，最大值 = **4**
> 
> **⚠️ 易错**：
> - ❌ 最小值写成 -15（不减初值）
> - ✅ 等公式：S_min = 初值 − 进程总数

> - 将独占设备改造成**虚拟设备**，实现多用户共享
> 
> **(2) 设备分配步骤**：
> 1. **分配设备**
> 2. **分配控制器**
> 3. **分配通道**
> 
> **说明**：必须按此顺序，只有上一级分配成功才能进入下一级分配。任一环节失败都要回退。
> 
> **⚠️ 易错**：
> - ❌ 漏掉"分配控制器"或"分配通道"
> - ✅ 三步顺序不能颠倒
