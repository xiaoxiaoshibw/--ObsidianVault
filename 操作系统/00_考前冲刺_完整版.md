## 第1章 操作系统引论

### 核心知识点

| 考点 | 内容 |
|------|------|
| **OS最基本特征** | **并发** 和 **共享**（二者互为存在条件） |
| **多道程序设计** | 提高单机资源利用率的关键技术 |
| **管态 vs 目态** | 管态=内核态，执行特权指令；目态=用户态，执行非特权指令 |
| **原语** | 不可分割的操作，**只能在管态下执行** |
| **并发 vs 并行** | 并发=同一**时间间隔**（宏观同时，微观交替）；并行=同一**时刻** |
| **操作系统接口** | 命令接口（用户）、程序接口/系统调用（程序） |

### 三种操作系统对比

| 类型 | 特点 | 缺点 |
|------|------|------|
| **批处理** | 吞吐量大、资源利用率高、系统开销小 | ❌ **响应不及时**（非交互） |
| **分时** | 多用户、交互性强、响应短、独立性好 | — |
| **实时** | 响应**及时**、**可靠性**高、确定性强 | — |

### 易错判断题

| 题目 | 答案 |
|------|:----:|
| 操作系统不确定性 = 同一程序/输入/环境，多次运行结果不同 | ✅ **正确** |
| 并发 = 两个以上事件**同一时刻**发生 | ❌ **错误**（应是同一时间间隔） |
| 并行 = 两个以上事件**同一时刻**发生 | ✅ **正确** |
| 处理机从管态→目态通过置PSW实现 | ✅ **正确** |
| 处理机从目态→管态通过置PSW实现 | ❌ **错误**（通过**中断**实现） |
| 原语可在目态下执行 | ❌ **错误** |
| 原语仅可在管态下执行 | ✅ **正确** |
| 分时操作系统必然建立在多道程序技术之上 | ✅ **正确** |
| UNIX 是 **多用户多任务** 操作系统 | ✅ |
| Windows XP 是 **单用户多任务** 操作系统 | ✅ |

### 核心简答

**批处理/分时/实时各有什么特点？**
- **批处理**：用户成批提交，不能干预，吞吐量大、资源利用率高，但交互性差、响应时间长
- **分时**：多用户通过终端同时使用，时间片轮转，交互性强、响应短、独立性好
- **实时**：在规定时间内对外部事件作出响应，响应及时、可靠性高

**管态与目态的区别？**
- **管态**：OS内核运行状态，可执行**全部指令**（包括特权指令）
- **目态**：用户程序运行状态，只能执行**非特权指令**
- 通过 PSW 中的状态位来区分

## 第2章 进程的描述与控制

### 核心知识点

| 考点 | 内容 |
|------|------|
| **进程三大基本状态** | 就绪、运行、阻塞 |
| **进程映像** | **PCB + 程序 + 数据集** |
| **进程特征** | 动态性、并发性、**异步性**（走走停停） |
| **进程并发性** | 多个进程在**同一时间间隔**内同时发生 |
| **线程 vs 进程** | 线程是调度基本单位，进程是资源拥有基本单位 |

### 进程状态转换

```
就绪 ──[进程调度]──→ 运行 ──[时间片到/被抢占]──→ 就绪
运行 ──[等待事件]──→ 阻塞
阻塞 ──[事件发生]──→ 就绪
```

- 阻塞态 **不能直接** 到运行态，必须先到就绪态
- 就绪态 **不能直接** 到阻塞态

### 线程实现模式

| 类型 | 管理方 | 优点 | 缺点 |
|------|--------|------|------|
| **用户级线程** | 用户空间线程库 | 切换快、开销小、灵活 | 一个阻塞→整个进程阻塞，不能利用多核并行 |
| **内核级线程** | OS内核 | 可多核并行、一个阻塞不影响其他 | 切换慢、系统开销大 |

### 进程切换保存内容

> 程序计数器(PC)、程序状态字(PSW)、通用寄存器、栈指针、地址变换/内存管理寄存器

### 易错点

- 进程状态发生变化 **不一定** 引起另一个进程状态变化 ❌
- 临界资源 = **一次仅允许一个进程使用**的资源 ✅
- 进程的异步性 = 走走停停 ✅

## 第3章 处理机调度与死锁

### 调度算法

| 算法 | 核心思想 | 平均周转时间 | 饥饿？ |
|------|---------|:----------:|:-----:|
| **FCFS** 先来先服务 | 按到达顺序 | 较长 | 无 |
| **SJF** 短作业优先 | 执行时间最短先 | **最短** ✅ | **有**（长作业饥饿） |
| **HRRN** 响应比高者优先 | 考虑等待+执行时间 | 较短 | 无 |
| **RR** 时间片轮转 | 时间片轮流 | 中等 | 无 |
| **优先级调度** | 优先级高的先 | — | **有** |
| **多级反馈队列** | 多队列+不同时间片+降级 | 较短 | 可通过**老化**解决 |

### 死锁

#### 四必要条件（缺一不可）

1. **互斥** — 资源一次只能给一个进程
2. **请求与保持** — 已有资源又请求新资源
3. **不可剥夺** — 资源不能被强行拿走
4. **循环等待** — 存在进程-资源循环链

#### 死锁处理策略

| 策略 | 方法 | 典型例子 |
|------|------|---------|
| **预防** | 破坏四个必要条件之一 | **按序分配**（破坏循环等待） |
| **避免** | 分配前判断是否安全 | **银行家算法** |
| **检测** | 检测是否有死锁 | **死锁定理**、资源分配图 |
| **解除** | 从死锁中恢复 | 资源抢占、杀死进程 |

#### 死锁 vs 不安全状态

```
死锁 → 一定是不安全状态
不安全状态 → 不一定死锁（但可能死锁）
```

### 信号量取值范围计算

> **公式**：信号量 S 的初值 = 最大允许并发数
> S 的取值范围：[初值 - 进程总数, 初值]
> 最小值时的含义：有 |S| 个进程在等待

**例**：15个进程共享程序段，最多允许4个进入，信号量 S 初值=4
- 最大值 = 4（没有进程进入临界区时）
- 最小值 = 4 - 15 = **-11**（所有进程都在等待时）
- 表示有 11 个进程在等待

### 核心简答

**多级反馈队列调度算法为什么能兼顾各类用户？**
- 设置多个优先级不同的就绪队列，高优先级时间片短，低优先级时间片长
- 新进程先进入高优先级队列，尽快完成 → 适合**短作业和交互型用户**
- 需长时间运行的进程逐级下降到低优先级队列，仍能获得较长时间片 → 也照顾**长作业和计算型用户**

**引起进程调度的因素有哪些？**
1. 正在执行的进程运行完毕
2. 正在执行的进程因等待某事件而阻塞
3. 正在执行的进程时间片用完
4. 有更高优先级的进程进入就绪队列
5. 正在执行的进程主动放弃处理机
6. 中断处理结束后需要重新调度

**分时系统中为什么一般没有作业调度？**
- 分时系统是交互式系统，用户作业直接进入内存形成进程
- 主要通过**进程调度**按时间片轮转分配CPU
- 作业调度主要用于**批处理系统**，从外存后备队列选择作业调入内存

## 第4章 进程同步

### PV操作核心公式

```
P(S): S = S - 1;  若 S < 0 → 进程阻塞
V(S): S = S + 1;  若 S ≤ 0 → 唤醒一个等待进程
```

### 信号量含义

| S的值 | 含义 |
|:----:|------|
| S > 0 | 可用资源数 |
| S = 0 | 资源刚好用完 |
| S < 0 | 有 \|S\| 个进程在等待 |

### PV操作互斥模板

```c
// 互斥使用临界区
P(mutex);     // mutex 初值 = 1
  临界区代码;
V(mutex);
```

### 经典同步问题

**生产者-消费者**（n 个缓冲区）：
| 信号量 | 初值 | 作用 |
|--------|:---:|------|
| `mutex` | 1 | 互斥访问缓冲区 |
| `empty` | n | 空缓冲区数 |
| `full` | 0 | 满缓冲区数 |

```c
// 生产者
P(empty);  // 先申请资源
P(mutex);  // 再互斥
  放产品;
V(mutex);
V(full);

// 消费者
P(full);
P(mutex);
  取产品;
V(mutex);
V(empty);
```

> ⚠️ **易错**：必须先 P(资源信号量)，后 P(互斥信号量) — 顺序颠倒可能导致**死锁**

### PV操作计算

> 多个并发进程先执行若干次 P 操作，再执行若干次 V 操作，求最终 S 值：
> **每次 P: S-1，每次 V: S+1**

**例**：S 初值=6，8次P后→S=-2，5次V后→S=**3**

### 进程同步模型题（多缓冲）

```
P1 → B1 → P2 → B2 → P3
      ↳直接→ B2 ↱
```

信号量设置：
- `empty1 = M, empty2 = M`（B1、B2容量）
- `full1 = 0, full2 = 0`
- `mutex1 = 1, mutex2 = 1`

> P1：正数→直接放B2，负数→放B1→P2加工→B2→P3输出

## 第5章 存储器管理

### 内存分配算法

| 算法 | 排序方式 | 策略 |
|------|---------|------|
| **FF** 首次适应 | 地址**递增** | 找第一个≥需求的空闲块 |
| **NF** 循环首次适应 | 地址递增（循环） | 从上一次位置继续找 |
| **BF** 最佳适应 | 大小**递增** | 找最小≥需求的空闲块（**外零头最小**） |
| **WF** 最坏适应 | 大小**递减** | 找最大空闲块 |

### 碎片

| 碎片类型 | 产生系统 | 说明 |
|---------|---------|------|
| **内零头** | **分页**系统 | 分配块 − 实际使用 |
| **外零头** | **分段、可变分区**系统 | 空闲块总和≥需求，但无连续块 |
| BF 最佳适应 | 可变分区 | **外零头最小** ✅ |
| 页式存储 | 分页 | 只有**内部碎片** |

### 地址重定位

| 类型 | 发生时间 | 特点 |
|------|---------|------|
| **静态重定位** | 作业装入时 | **不需**增加硬件地址变换机构 ❌ |
| **动态重定位** | 作业**执行过程中** | 需要硬件支持，作业可在内存中移动 |

### 页式存储管理

| 概念 | 说明 |
|------|------|
| 分配单位 | **块（页框）**，不是段也不是作业 |
| 页表 | 逻辑页号 → 物理块号 |
| 快表(TLB) | 保存**最近用过的页表项**，加速地址转换 |
| 页内偏移位数 | 1KB→10位，4KB→12位 |

### 地址转换计算

```
页号 = floor(逻辑地址 / 页大小)
页内偏移 = 逻辑地址 % 页大小
物理地址 = 块号 × 块大小 + 页内偏移
```

### 段页式

> 访问主存中的一条指令或数据，**至少访问 3 次内存**：
> 查段表 → 查页表 → 访问目标单元

### 有效访问时间（EAT）

> 有快表，内存访问时间 t，快表检索时间 T，快表命中率 α
> **EAT = α × (T + t) + (1-α) × (T + 2t)**

**例**：t=2μs, T=0.5μs, α=80%
EAT = 0.8 × (0.5+2) + 0.2 × (0.5+4) = 0.8×2.5 + 0.2×4.5 = **2.9μs**

### 段式地址计算

> 有效地址 16 位，最大段长 8KB
> 逻辑地址空间 = 段数 × 最大段长
> 段号位数 = 16 − log₂(8×1024) = 16 − 13 = **3 位** → 最多 **8 段**
> 逻辑地址空间 = 8 × 8KB = **64KB**

### 物理地址计算（请求页式）

> 逻辑地址 32 位，页大小 16KB（$2^{14}$B）
> 用户地址空间 = $2^{32}$B = **4096 MB**
> 页表长度 = $2^{32}/2^{14}$ = $2^{18}$ = **256K** 项（若每项4B = 1MB）

**十六进制地址转换**：
> 逻辑地址 9BCD（H），页大小 16KB（$2^{14}$B = 4000H）
> 页号 = 9BCD / 4000 = 9BCD >> 14 = **2**（十进制）
> 偏移 = 9BCD & 3FFF = **1BCD（H）**
> 若块号 = 111（十进制）= 6F（H）
> 物理地址 = 6F × 4000 + 1BCD = 1B000 + 1BCD = **1BDBCD（H）**

### 填空题速记

| 题目 | 答案 |
|------|:----:|
| 系统有4MB内存，最大有效物理地址 | **3FFFFFH**（$2^{22}-1$） |
| 4MB内存，页长1KB，总页框数 | **4096** |
| 段式管理，地址32位，最大段长64KB，段号占 | **16位** |
| 两级页表，CPU每存取数据需访问内存 | **3次** |
| 页大小1KB，虚拟地址0x3C8F的页号= | **15**（十进制），页内偏移=**08FH** |

## 第6章 虚拟存储器

### 核心特征

| 特征 | 含义 |
|------|------|
| **多次性** | 作业分多次调入内存 |
| **对换性** | 作业在内存外存之间对换 |
| **虚拟性** | 从逻辑上扩充内存容量 |

### 实现方案

> 可以实现虚拟存储器的方案：**请求页式** ✅
> （固定分区、可变分区、纯分页 → ❌ 都不能实现虚拟存储器）

### 硬件支持

实现请求分页需要：
1. **页表机制**
2. **缺页中断机构**
3. **地址变换机构**

### 页表项新增字段

请求分页页表在基本分页基础上增加：
- **状态位** — 页面是否在内存
- **访问位** — 页面最近是否被访问（供置换算法参考）
- **修改位** — 页面是否被修改（决定换出时是否写回外存）
- **外存地址** — 页面在外存的位置

### 页面置换算法

| 算法 | 策略 | 缺页率（典型） | 特点 |
|------|------|:------------:|------|
| **FIFO** | 淘汰最先进入的页 | 最高 | 实现简单，可能Belady异常 |
| **LRU** | 淘汰最久未使用的页 | 中等 | 性能好，硬件开销大 |
| **OPT** | 淘汰未来最远使用的页 | 最低 | ❌ **无法实现**，仅理论参考 |

> **缺页率**：OPT < LRU < FIFO

## 第7章 设备管理

### 设备分类

| 分类方式 | 类别 | 例子 |
|---------|------|------|
| **传输单位** | **字符设备** | 键盘、显示器、打印机 |
| **传输单位** | **块设备** | 磁盘、磁带、光盘 |
| **分配角度** | **独占设备** | 打印机（静态分配） |
| **分配角度** | **共享设备** | 磁盘（动态分配） |
| **分配角度** | 虚拟设备 | SPOOLing技术改造 |

### 设备分配步骤

> 分配设备 → 分配控制器 → 分配通道

### SPOOLing技术

> 通过 **SPOOLing（假脱机）** 把原**独享设备**改造成能为若干用户共享的**虚拟设备**

### 磁盘读写时间

```
总访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间
```

- **寻道时间**：磁头移动到目标磁道（占大头）→ 调度算法的优化目标
- **旋转延迟**：扇区旋转到磁头下（平均约半圈）
- **传输时间**：读写数据（通常较小）

### 平均旋转延迟

> 转速 7200 转/分钟
> 每转时间 = 60/7200 = 0.00833s = **8.33ms**
> 平均旋转延迟 = 半圈 = 8.33/2 = **4.17ms** ✅

### 磁盘调度算法

| 算法 | 策略 | 总寻道距离 |
|------|------|:---------:|
| **FCFS** | 按请求顺序依次服务 | 最大 |
| **SSTF** | 每次选最近的请求 | 最小（但可能**饥饿**） |
| **SCAN（电梯）** | 单向到端，反向继续 | 中等 |
| **C-SCAN（循环扫描）** | 单向到端，甩回另一端继续 | 较大（更公平） |

> SCAN 和 C-SCAN 的区别：SCAN 到端后反向扫描；C-SCAN 到端后**直接回到 0** 再继续

### I/O控制方式

| 方式 | CPU利用率 | 数据路径 | 适用 |
|------|:---------:|---------|------|
| **程序查询** | 极低（忙等） | CPU参与每一步 | 简单设备 |
| **中断** | 较高 | CPU→I/O→内存 | **打印机**等低速设备 |
| **DMA** | 高 | I/O ↔ 内存（直接） | **磁盘**等块设备（批量传输） |
| **通道** | 最高 | 专用处理器执行通道程序 | 大型系统 |

### 中断频率计算

> **例**：RS-232 通信速率 9.6KB/s（B=Bit），8位寄存器作为缓冲
> 每秒传输 = 9.6 × 1024 × 8 = 98304 bit/s（或 9.6 × 1000 × 8 按题目）
> 每字节中断一次
> 每秒中断次数 = 9.6KB/s = 9600 次/秒（更简化）
> ⏱ 中断间隔 = 1/9600 ≈ **0.104ms** 
> 
> 教材标准答案：**0.833ms**（以下面方式计算）
> 9.6KB/s = 9600 B/s → 传输一个字节时间 = 1/1200 ≈ 0.833ms
> CPU 必须在 **0.833ms** 内响应

### 填空题速记

| 题目 | 答案 |
|------|:----:|

## 计算题专项训练

### 题型1：进程调度计算（必考）

> **5个进程同时到达（或不同时到达），计算各种算法的平均周转时间**

**模板**（同时到达，SJF）：
```
执行时间：J1=T1, J2=T2, J3=T3, 且 T1<T2<T3
SJF顺序：J1 → J2 → J3
周转时间：J1=T1, J2=T1+T2, J3=T1+T2+T3
平均周转时间 = (T1 + (T1+T2) + (T1+T2+T3)) / 3
            = (3T1 + 2T2 + T3) / 3
```

### 题型2：物理地址转换（必考）

**页式**：
> 逻辑地址 → 查页表得块号 → 物理地址 = 块号×页大小 + 页内偏移

**段式**：
> 逻辑地址 → 查段表得基址 → 物理地址 = 基址 + 段内偏移（需检查越界）

**请求页式十六进制**：
> 确定页大小 → 高位为页号，低位为偏移 → 查表 → 计算物理地址

### 题型3：有效访问时间

> EAT = 命中率 × (快表时间 + 一次内存访问) + (1-命中率) × (快表时间 + 两次内存访问)

### 题型4：PV操作计算

> 初值 S₀，先 N₁ 次 P 操作，再 N₂ 次 V 操作
> 最终 S = S₀ − N₁ + N₂

### 题型5：银行家算法

> 三步判断：
> 1. Request ≤ Need?
> 2. Request ≤ Available?
> 3. 试分配后能否找到**安全序列**？
> 三步全 ✅ 才能分配

### 题型6：页面置换

> 给定页面序列和物理块数，分别用 FIFO/LRU/OPT 计算缺页次数
> 注意：OPT 是理论最优，不可实现

### 题型7：磁盘调度

> 给定磁头位置、方向、请求队列
> SSTF：每次选最近的
> SCAN：单向到端再反向
> C-SCAN：单向到端后直接跳到另一端

## 易混概念对比

| 易混点 | 区分 |
|--------|------|
| **并发 vs 并行** | 并发=同一时间间隔（宏观）；并行=同一时刻（微观） |
| **管态 vs 目态** | 管态→目态：**置PSW**；目态→管态：**中断** |
| **静态重定位 vs 动态重定位** | 静态=装入时；动态=**执行过程中** |
| **内零头 vs 外零头** | 内=分页（分配多了用不完）；外=分段（碎片总和够但不连续） |
| **FIFO vs LRU vs OPT** | FIFO淘汰最早进入的；LRU淘汰最久未用的；OPT淘汰未来最远使用的 |
| **SCAN vs C-SCAN** | SCAN到端反向；C-SCAN到端直接回0 |
| **SJF vs HRRN** | SJF只考虑执行时间（短作业饥饿）；HRRN同时考虑等待时间（无饥饿） |
| **预防 vs 避免** | 预防=破坏必要条件；避免=银行家算法（分配前判断安全性） |
| **用户级线程 vs 内核级线程** | 用户级=库管理（快但不支持多核）；内核级=OS管理（慢但支持多核） |
| **页式 vs 段式** | 页=固定大小（**内部碎片**）；段=逻辑单位（**外部碎片**） |
| **顺序分配 vs 按序分配** | 按序分配（资源有序分配法）破坏**循环等待**条件 |
| **独占设备 vs 共享设备** | 独占=静态分配；共享=动态分配 |

## 填空题必背清单 （高频）

### 第1章
1. 操作系统为用户提供两种接口：**命令接口** 和 **程序接口**
2. 多道批处理系统的硬件支持：**中断技术** 和 **通道技术**
3. 实时系统两个基本特征：**及时性** 和 **可靠性**
4. 提高单机资源利用率的关键技术：**多道程序设计技术**
5. 现代OS最基本的两个特征：**并发** 和 **共享**

### 第2章
1. 进程的映像：**PCB + 程序 + 数据集**
2. 进程的异步性：进程执行"**走走停停**"
3. 进程的并发性：多个进程在**同一时间间隔**内发生
4. 不会因竞争**可剥夺资源**而发生死锁

### 第3章
1. 作业调度算法：**FCFS、SJF、优先级调度、HRRN**
2. 作业经历的调度：**高级调度、低级调度、中级调度**
3. 死锁处理方法：**预防死锁、避免死锁、检测与解除死锁**
4. 一次仅允许一个进程使用的资源：**临界资源**
5. 访问临界资源的代码段：**临界区**
6. 临界区置于**P操作**和**V操作**之间

### 第5章
1. 内存管理三种基本方式：**分区存储管理、分页存储管理、分段存储管理**
2. 把程序逻辑地址变成物理地址：**地址重定位**
3. 可变分区存储管理：在作业装入和处理过程中**动态地划分**存储空间
4. 缓存技术主要目的：提高CPU和设备之间的**并行操作**程度

### 第6章
1. 虚拟存储器基本特征：**多次性、对换性、虚拟性**
2. 关键技术：**请求调页（段）** 和 **页（段）置换**
3. 请求分页页表新增：**状态位、访问位、修改位、外存地址**
4. 硬件支持：**页表机制、缺页中断机构、地址变换机构**

### 第7章
1. 磁盘读写时间因素：**寻道时间、旋转延迟时间、传输时间**
2. SPOOLing：把独享设备改造为**共享**设备（虚拟设备）
3. 设备分类：**独占设备、共享设备、虚拟设备**
4. 设备管理功能：提供接口、设备分配、**并行操作**、缓冲区管理
5. 设备分配步骤：**分配设备 → 分配控制器 → 分配通道**
6. UNIX 中所有设备文件被看成**特殊文件**
7. 独占设备采用**静态**分配，共享设备采用**动态**分配
