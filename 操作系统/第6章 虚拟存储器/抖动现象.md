# 抖动现象

## 定义
**抖动（Thrashing）** 是指进程频繁**缺页**，导致大部分时间花在**页面置换**上，CPU 利用率急剧下降的现象。

## 产生原因
- 进程的**驻留集太小**，小于工作集
- 频繁的缺页中断 → 页面置换 → I/O 操作
- CPU 等待 I/O → 利用率下降
- 系统引入更多进程 → 恶性循环

## 防止抖动的策略
| 策略 | 说明 |
|------|------|
| **工作集模型** | 保证驻留集 ≥ 工作集 |
| **缺页频度控制** | 缺页率过高时增加驻留集 |
| **挂起部分进程** | 减少活跃进程数，释放内存 |
