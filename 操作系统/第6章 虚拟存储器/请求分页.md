# 请求分页

## 概念
**请求分页**是在**分页系统**基础上，增加了**请求调页**和**页面置换**功能的虚拟存储技术。

## 页表项扩充
| 字段 | 说明 |
|------|------|
| **状态位** | 页面是否在内存中 |
| **访问位** | 页面是否被访问过 |
| **修改位** | 页面是否被修改过 |
| **外存地址** | 页面在外存中的位置 |

## 所需硬件支持
1. 页表机制（含扩充字段）
2. 缺页中断机构
3. 地址变换机构

## 实现流程
```
访问页面 → 查页表
  ├─ 在内存 → 地址转换 → 访问
  └─ 不在内存 → 缺页中断 → 调页 → 更新页表 → 重新执行
```

## 📝 本章习题
**填空题8**（第5章）：一个逻辑地址32个比特位采用请求页式管理、页大小为16KB的计算机系统，其用户地址空间可达 **4096** MB；页表长度为 **256**（十进制）KB；若处理器给出的逻辑地址为9BCD（十六进制），地址变换机构将计算得到该逻辑地址的页号为 **2**（十进制），如果该页所对应的物理块号为111（十进制），则9BCD所对应的物理地址为 **1BDBCDH**（十六进制）。
> 解：2³²=4GB=4096MB；页大小16KB=2¹⁴B，页内偏移14位，页号=32-14=18位，页表=2¹⁸×2B？不对，页表长度=2¹⁸个表项
> 9BCDH：二进制 1001 1011 1100 1101，高18位=00 0000 0000 0010 01=2（十进制）
> 物理地址 = 111×4000H + 9BCD的低14位(3BCD) = 1B6000 + 3BCD = 1BDBCDH
