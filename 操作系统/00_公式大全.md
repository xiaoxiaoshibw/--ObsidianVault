## 一、进程调度公式

### 1.1 周转时间

| 公式 | 含义 |
|------|------|
| $$周转时间 = 完成时间 - 到达时间$$ | 从提交到结束的总时间 |
| $$带权周转时间 = \frac{周转时间}{服务时间（执行时间）}$$ | 越接近 1 越好 |
| $$平均周转时间 = \frac{1}{n}\sum_{i=1}^{n} 周转时间_i$$ | 系统总体性能 |
| $$平均带权周转时间 = \frac{1}{n}\sum_{i=1}^{n} 带权周转时间_i$$ | 系统总体公平性 |

> 带权周转时间 **≥ 1**，越接近 1 说明调度效率越高

### 1.2 其他时间指标

| 公式 | 含义 |
|------|------|
| $$响应时间 = 首次运行时间 - 到达时间$$ | 分时系统中重要 |
| $$等待时间 = 周转时间 - 执行时间$$ | 进程在就绪队列等待的总和 |
| $$完成时间 = 到达时间 + 周转时间$$ | 进程结束的时刻 |

### 1.3 时间片轮转 (RR) 相关

| 公式 | 含义 |
|------|------|
| $$时间片 q = \frac{最大容忍响应时间}{最大用户数}$$ | 时间片估算 |
| $$响应时间 \approx n \times q$$ | n 个进程，时间片 q |
| $$切换开销比例 = \frac{切换时间}{q + 切换时间}$$ | 时间片越小，开销比例越大 |

## 二、页面置换公式

### 2.1 缺页率

| 公式 | 含义 |
|------|------|
| $$缺页率 p = \frac{缺页次数}{总访问次数}$$ | 越低越好 |
| $$命中率 = 1 - p$$ | 页面在内存中的概率 |

### 2.2 有效访问时间 (EAT)

| 公式 | 含义 |
|------|------|
| $$EAT = (1-p) \times t_m + p \times t_f$$ | 考虑缺页的平均访存时间 |

- $t_m$：内存访问时间（约 100ns）
- $t_f$：缺页中断处理时间（约 25ms，含磁盘I/O）
- 缺页率 $p$ 即使很小（如 0.01），对性能影响也很大

### 2.3 考虑快表 (TLB) 的有效访问时间

| 情况 | 公式 |
|------|------|
| TLB 命中 | $$EAT = t_{TLB} + t_m$$ |
| TLB 未命中（页表在内存） | $$EAT = t_{TLB} + 2 \times t_m$$ |
| TLB 未命中（页表在快存） | $$EAT = t_{TLB} + t_c + t_m$$ |
| 综合（TLB 命中率 $\alpha$） | $$EAT = \alpha \times (t_{TLB} + t_m) + (1-\alpha) \times (t_{TLB} + 2t_m)$$ |

## 三、存储管理公式

### 3.1 分页存储管理

| 公式 | 含义 |
|------|------|
| $$页号 = \lfloor 逻辑地址 / 页大小 \rfloor$$ | 取整 |
| $$页内偏移 = 逻辑地址 \bmod 页大小$$ | 取余 |
| $$物理地址 = 块号 × 块大小 + 页内偏移$$ | 地址转换 |

**换算关系：**

| 页大小 | 页内偏移占位数 |
|--------|--------------|
| 1 KB = $2^{10}$ B | 偏移量占 10 位 |
| 4 KB = $2^{12}$ B | 偏移量占 12 位 |
| 8 KB = $2^{13}$ B | 偏移量占 13 位 |

**例题**：页大小 4KB，逻辑地址 8192，求页号和页内偏移

```
8192 / 4096 = 2（页号）
8192 % 4096 = 0（页内偏移）
```

### 3.2 分段存储管理

| 公式 | 含义 |
|------|------|
| $$物理地址 = 段基址 + 段内偏移$$ | 地址转换 |

> 段内偏移必须 ≤ 段长，否则产生越界中断

### 3.3 段页式存储管理

$$
逻辑地址 \xrightarrow{\text{段号→段表}} 页表起始地址 \xrightarrow{\text{页号→页表}} 物理块号 \xrightarrow{+页内偏移} 物理地址
$$

**三次访存：** 查段表 → 查页表 → 访问目标单元

### 3.4 内存分配算法

| 算法 | 策略 | 公式/规则 |
|------|------|-----------|
| **FF** 首次适应 | 地址递增 | 从低地址开始，找**第一个**≥需求空闲块 |
| **NF** 循环首次适应 | 地址递增（循环） | 从上一次位置继续，找**下一个**≥需求空闲块 |
| **BF** 最佳适应 | 大小递增 | 找**最小**的≥需求空闲块（碎片最多） |
| **WF** 最坏适应 | 大小递减 | 找**最大**的空闲块（减少小碎片） |

#### 内部碎片与外部碎片

| 碎片类型 | 公式/说明 |
|---------|----------|
| **内部碎片** | 分配块大小 − 实际使用大小（分页系统特有） |
| **外部碎片** | 空闲块总和 ≥ 需求，但无连续块满足（分段系统特有） |
| **分页** | 只有内部碎片，最大为 `页大小 − 1` |
| **分段** | 只有外部碎片 |

### 3.5 页表大小计算

| 公式 | 含义 |
|------|------|
| $$页表大小 = 页表项数 \times 每项字节数$$ | |
| $$页表项数 = 进程地址空间 / 页大小$$ | 即逻辑地址空间的总页数 |
| $$页表占用内存 = 页表大小 / 页大小$$ | 页表本身占多少页 |

**例题**：逻辑地址 32 位，页大小 4KB，页表项 4B，求页表占用内存

```
页表项数 = 2^32 / 2^12 = 2^20 = 1,048,576 项
页表大小 = 2^20 × 4B = 4MB
页表占页 = 4MB / 4KB = 1024 页（即 1 页页表需要 1024 页存储）
```

## 四、PV 操作 — 信号量公式

### 4.1 信号量定义

| 公式 | 含义 |
|------|------|
| $$S = 可用资源数$$ | 信号量初值 = 资源数量 |
| $$S > 0$$ | 有资源可用 |
| $$S = 0$$ | 资源刚好用完 |
| $$S < 0$$ | 有 $|S|$ 个进程在等待 |

### 4.2 P 操作（wait）

$$
P(S):\ \begin{cases}
S = S - 1\\
\text{若 } S < 0 \text{，则进程阻塞，进入等待队列}
\end{cases}
$$

### 4.3 V 操作（signal）

$$
V(S):\ \begin{cases}
S = S + 1\\
\text{若 } S \leq 0 \text{，则唤醒等待队列中的一个进程}
\end{cases}
$$

### 4.4 经典同步问题信号量设置

| 问题 | 信号量设置 |
|------|-----------|
| **生产者-消费者** | `mutex=1`, `empty=n`, `full=0` |
| **读者-写者** | `mutex=1`（读计数互斥）, `rw=1`（读写互斥）, `count=0` |
| **哲学家进餐** | `chopstick[5]={1,1,1,1,1}`, 或 `mutex=1` 限制最多 4 人进餐 |
| **前驱关系** | 每条有向边设一个同步信号量，初值=0 |

## 五、死锁公式

### 5.1 死锁四必要条件

> 四者缺一不可

| 条件 | 含义 |
|------|------|
| **互斥** | 资源一次只能分配给一个进程 |
| **请求与保持** | 进程已占资源，又请求新资源 |
| **不可剥夺** | 资源不能强行拿走 |
| **循环等待** | 存在进程—资源的循环等待链 |

### 5.2 死锁判定

$$
\begin{aligned}
&安全状态 \Rightarrow 无死锁\\
&不安全状态 \nRightarrow 一定死锁，但可能死锁\\
&死锁 \Rightarrow 一定是不安全状态
\end{aligned}
$$

### 5.3 银行家算法

| 公式 | 含义 |
|------|------|
| $$Need_{ij} = Max_{ij} - Allocation_{ij}$$ | 还需资源 = 最大需求 − 已分配 |
| $$Available_j = Total_j - \sum_{i} Allocation_{ij}$$ | 可用资源 = 总量 − 所有进程已分配 |
| $$Request_i \leq Need_i$$ | 请求不能超过还需量（否则非法） |
| $$Request_i \leq Available$$ | 请求不能超过可用量（否则等待） |
| **试分配后** | 若能找到安全序列则分配，否则回滚 |

#### 剩余资源矩阵快速判断

P 能完成的条件：$Need_P \leq Available$

## 六、磁盘调度公式

### 6.1 寻道距离

| 公式 | 含义 |
|------|------|
| $$每次寻道距离 = \|目标磁道 - 当前磁道\|$$ | 单次移动距离 |
| $$总寻道距离 = \sum\|下一磁道 - 当前磁道\|$$ | 所有移动之和 |

### 6.2 各算法计算规则

| 算法 | 计算规则 |
|------|---------|
| **FCFS** | 按请求顺序依次服务，直接计算相邻差值 |
| **SSTF** | 每次选距当前磁头**最近**的请求，逐个累加 |
| **SCAN（电梯）** | 单向移动到端，再反向服务剩余请求 |
| **C-SCAN（循环扫描）** | 单向移动到端，直接**甩到另一端**再继续 |

### 6.3 磁盘访问总时间

$$
总访问时间 = 寻道时间 + 旋转延迟 + 传输时间
$$

- **寻道时间**：磁头移动到目标磁道（占大头，调度算法的优化目标）
- **旋转延迟**：扇区旋转到磁头下（平均约半圈）
- **传输时间**：读写数据（通常较小）

## 七、I/O 控制方式对比

| 方式 | CPU 利用率 | 数据流向 | 特点 |
|------|-----------|---------|------|
| **程序查询** | 极低（忙等） | CPU → I/O → 内存 | 简单，CPU 被占用 |
| **中断** | 较高 | CPU → I/O → 内存 | 每次中断需保存/恢复现场 |
| **DMA** | 高 | I/O ↔ 内存（直接） | 批量传输，周期窃取 |
| **通道** | 最高 | I/O ↔ 内存（直接） | 专用处理器，可执行通道程序 |

## 八、调度算法对比总结

| 算法              | 决策点    | 抢占？ | 平均周转时间 | 平均响应时间     | 有无饥饿          |
| --------------- | ------ | --- | ------ | ---------- | ------------- |
| **FCFS**（先来先服务） | 到达时    | 否   | 较长     | 长          | 无             |
| **SJF**（短作业优先）  | 到达/完成时 | 均可  | **最短** | 较长         | **有**（长作业饥饿）  |
| **优先级调度**       | 到达/完成时 | 均可  | —      | —          | **有**（低优先级饥饿） |
| **RR**（时间片轮转）   | 时间片到   | 是   | 中等     | **短**（响应快） | 无             |
| **多级反馈队列**      | 多种     | 是   | 较短     | 短          | 可通过老化解决       |

## 📝 助记口诀

```
周转 = 完成 - 到达
带权 = 周转 / 服务
页号 = 地址 / 页大小
偏移 = 地址 % 页大小
FF 地址增，BF 大小增，WF 大小减
P 减 V 加，P 负阻塞，V 负唤醒
缺页率 = 缺页 / 总访问
SSTF 选最近，SCAN 到端回
```
