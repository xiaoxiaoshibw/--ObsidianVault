# 死锁四必要条件

**四个条件必须同时满足**才会发生死锁：

## 1. 互斥条件（Mutual Exclusion）
- 资源一次只能被一个进程使用

## 2. 请求与保持条件（Hold and Wait）
- 进程保持已有资源的同时，请求其他资源

## 3. 不可剥夺条件（No Preemption）
- 资源只能由进程主动释放，不能被强行剥夺

## 4. 循环等待条件（Circular Wait）
- 存在一个进程-资源的循环等待链

## 死锁预防
**破坏任意一个条件即可预防死锁：**
- 破坏互斥：共享资源（不现实）
- 破坏请求与保持：一次性分配所有资源
- 破坏不可剥夺：允许剥夺资源
- 破坏循环等待：资源有序分配法
