# 银行家算法

## 概念
**银行家算法（Banker's Algorithm）** 是著名的**死锁避免**算法，由 Dijkstra 提出。

## 核心思想
在分配资源之前，计算分配后是否处于**安全状态**。如果是，才分配；否则不分配。

## 数据结构
| 结构 | 含义 |
|------|------|
| Available | 可用资源数 |
| Max | 各进程最大需求 |
| Allocation | 已分配资源数 |
| Need | 还需资源数（Need = Max - Allocation）|

## 安全状态判定步骤
1. 寻找 Need ≤ Available 的进程
2. 假设该进程完成，回收其 Allocation
3. 重复直到所有进程都完成
4. 若找到这样的序列 → **安全状态**
5. 若找不到 → **不安全状态**
