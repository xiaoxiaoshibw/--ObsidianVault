# 优先级调度

## 定义
每个进程分配一个**优先级**，优先调度优先级高的进程。

## 分类
| 类型 | 说明 |
|------|------|
| **静态优先级** | 创建时确定，运行中不变 |
| **动态优先级** | 随运行时间动态调整 |

## 抢占式 vs 非抢占式
- **非抢占式**：运行中不会被抢走 CPU
- **抢占式**：高优先级进程到达时立即抢走 CPU

## 问题
- **饥饿**：低优先级进程可能永远得不到执行
- 解决：**动态优先级**（随等待时间增加而提高优先级）

## 📝 本章习题
**简答题（多级反馈队列）**：多级反馈队列调度算法综合了多种调度策略的优点：
1. 设置多个优先级不同的就绪队列，高优先级队列时间片短，低优先级队列时间片长。
2. 新进程先进入高优先级队列，若能很快完成，可得到快速响应，因此适合短作业和交互型用户。
3. 若进程需要较长运行时间，会逐级下降到低优先级队列，仍能获得较长时间片运行，因此也能照顾长作业。
所以它能同时保证交互用户的响应时间，兼顾短作业和长作业的执行效率。

**简答题（非剥夺优先级调度）**：优先级B=5&gt;E=4&gt;A=3&gt;C=2&gt;D=1
运行顺序：B→E→A→C→D
- B: 0→6, E: 6→14, A: 14→24, C: 24→26, D: 26→30
- 平均周转时间 = (24+6+26+30+14)/5 = **20分钟**
