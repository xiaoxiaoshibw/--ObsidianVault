# 进程控制

## 进程的创建
- **事件**：用户登录、作业调度、提供服务、应用请求
- **原语**：`create()` → 申请空白 PCB → 分配资源 → 初始化 PCB → 插入就绪队列

## 进程的终止
- **原因**：正常结束、异常结束、外界干预
- **原语**：`destroy()` → 回收资源 → 撤销 PCB

## 进程的阻塞与唤醒
- **阻塞**：`block()` → 运行→阻塞，保护现场，插入阻塞队列
- **唤醒**：`wakeup()` → 阻塞→就绪，从阻塞队列移出，插入就绪队列

## 进程的挂起与激活
- 挂起：内存→外存（对换）
- 激活：外存→内存

## 📝 本章习题
**简答题（进程切换）**：进程切换时，应保存当前进程的 CPU 现场信息，包括程序计数器、程序状态字、通用寄存器、栈指针以及有关地址变换和内存管理寄存器等内容，以便以后恢复该进程继续执行。

**简答题（进程调度因素）**：引起进程调度的因素有：
1. 正在执行的进程运行完毕。
2. 正在执行的进程因等待某事件而阻塞。
3. 正在执行的进程的时间片用完。
4. 有更高优先级的进程进入就绪队列。
5. 正在执行的进程主动放弃处理机。
6. 中断处理结束后需要重新调度。
