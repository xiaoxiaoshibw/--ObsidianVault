# 进程状态转换

## 三种基本状态

```
     ┌─────────┐
     │   就绪   │◄──────────────────┐
     └────┬─────┘                   │
          │ 调度运行                 │
          ▼                         │
     ┌─────────┐   时间片到      ┌────┐
     │   运行   │────────────────►│就绪│
     └────┬─────┘                └────┘
          │ 等待事件
          ▼
     ┌─────────┐
     │  阻塞   │
     └────┬─────┘
          │ 事件发生（唤醒）
          └──────────────────► 就绪
```

| 状态 | 说明 |
|------|------|
| **就绪（Ready）** | 万事俱备，只欠 CPU |
| **运行（Running）** | 正在使用 CPU 执行 |
| **阻塞（Blocked/Wait）** | 等待某事件发生（I/O 完成等）|

## 其他状态
- **创建状态（New）**：正在创建，尚未就绪
- **终止状态（Terminated）**：已结束，等待回收

## 状态转换规律
- 就绪 → 运行：调度程序选择
- 运行 → 就绪：时间片到/优先级被抢占
- 运行 → 阻塞：请求资源/等待事件
- 阻塞 → 就绪：事件发生

## 📝 本章习题
**选择题2**：一个进程的基本状态可以从其他两种基本状态转变过来，这个基本的状态一定是**就绪状态**。（选 B）
- A. 运行状态 ❌
- B. 就绪状态 ✅
- C. 完成状态 ❌
- D. 阻塞状态 ❌
> 就绪→运行、阻塞→就绪，就绪由其他两种状态转变而来

**简答题（进程状态）**：
1. **就绪状态**：进程已经具备运行条件，除 CPU 外，其他资源都已具备，只等待系统分配处理机。
2. **运行状态**：进程已经获得 CPU，正在处理机上执行。
3. **阻塞状态**：进程因等待某事件发生而暂时不能继续执行，例如等待 I/O 完成、等待资源、等待信号等。

**判断题1**：一个进程的状态发生变化必引起另一个进程的状态发生变化。 → **错误** ❌
