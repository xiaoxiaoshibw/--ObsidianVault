# 线程

## 定义
**线程（Thread）** 是 CPU 调度的**基本单位**，是进程中的一条执行路径。

## 进程 vs 线程

| 对比项      | 进程        | 线程                |
| -------- | --------- | ----------------- |
| **资源分配** | 是         | 否（共享所属进程资源）       |
| **调度单位** | 早期        | 是（现代 OS 以线程为调度单位） |
| **地址空间** | 独立        | 共享进程地址空间          |
| **系统开销** | 大（创建/切换）  | 小                 |
| **通信**   | 需 IPC（复杂） | 直接读写共享数据          |

## 线程分类
- **用户级线程**：用户空间管理，OS 无感知
- **内核级线程**：内核管理，OS 可见
- **混合模型**：多对多映射

## 引入线程的好处
1. 创建/撤销/切换开销小
2. 同一进程内线程可并发执行
3. 适合多处理器系统

---

## 📝 本章习题
**简答题（线程实现模式）**：
- **用户级线程**：线程的创建、撤销、切换和调度都由用户空间的线程库完成，内核不知道线程的存在，内核只把整个进程作为调度单位。特点：切换速度快、开销小、调度灵活；但一个线程阻塞可能导致整个进程阻塞，不能利用多处理机并行。
- **内核支持线程**：线程的创建、撤销、切换和调度都由操作系统内核完成。特点：内核能直接调度线程，多个线程可在多处理机上并行执行，一个线程阻塞不影响同进程其他线程；但线程管理需要进入内核，系统开销较大。
