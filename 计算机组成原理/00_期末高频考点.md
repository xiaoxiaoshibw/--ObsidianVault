## 一、计算机系统概述

### 冯·诺依曼计算机
- **工作方式**：控制流驱动方式
- **基本特点**：按地址访问并顺序执行指令
- **存储程序**：指令和数据以二进制存放在存储器，形式上无差别
- CPU区分指令和数据的依据：**指令周期的不同阶段**
- 硬件能直接执行：**机器语言**

### 性能指标

| 公式 | 含义 |
|------|------|
| $CPI = \dfrac{总时钟周期数}{指令条数}$ | 执行一条指令所需时钟周期数 |
| $MIPS = \dfrac{主频(MHz)}{CPI}$ | 每秒百万条指令 |
| **MFLOPS** | 科学计算用，浮点性能（科学计算最重要） |
| $执行时间 = \dfrac{指令条数 \times CPI}{主频}$ | 程序运行时间 |
| $时钟周期 = \dfrac{1}{主频}$ | 最小时间单位 |

**CPI与什么无关**：**时钟频率**（与系统结构、指令集、计算机组织有关）

**CPU提速计算**：
- 原CPU时间90s + I/O 10s = 100s
- CPU提速50% → 90/1.5 + 10 = **70s**

**MIPS计算**：主频1.2GHz，各类指令CPI分别为2(50%)、3(20%)、4(10%)、5(20%)
- 平均CPI = 2×0.5 + 3×0.2 + 4×0.1 + 5×0.2 = **3**
- MIPS = 1200/3 = **400**

### 编译与解释
- **编译程序**：一次翻译成目标代码，运行快
- **解释程序**：逐句翻译执行，不生成目标文件，速度慢
- 高级语言→可执行文件：**预处理→编译→汇编→链接**

### 计算机字长
- 机器字长 ≠ 指令字长 ≠ 存储字长（三者可等可不等）
- **机器字长**：CPU运算器位数 = 通用寄存器宽度
- **存储字长**：一个存储单元的位数 = **MDR位数**
- **指令字长**：指令的二进制位数

## 二、数据表示和运算

### 原码 / 反码 / 补码 / 移码

| 码制 | 0的表示 | n位整数范围 |
|------|---------|------------|
| 原码 | +0和-0 | $-(2^{n-1}-1) \sim +(2^{n-1}-1)$ |
| 反码 | +0和-0 | $-(2^{n-1}-1) \sim +(2^{n-1}-1)$ |
| 补码 | **唯一** | $-2^{n-1} \sim 2^{n-1}-1$ |
| 移码 | **唯一** | 同补码（阶码用移码） |

> 0表示唯一的码制：**补码和移码**
> 8位原码能表示 255 个不同数据（0有两个表示）
> 8位补码能表示 256 个不同数据（-128~127）

**补码转原码**：负数→符号位不变，数值位取反加1
**求[-X]补**：[X]补**连同符号位**取反，末位加1
**主存地址用**：无符号数表示

**常考题**：-66的8位补码
- 66 = 0100 0010 → 取反 1011 1101 → +1 → **1011 1110 = BEH**

**32位补码范围**：$-2^{31} \sim 2^{31}-1$

### 溢出判断

| 方法 | 规则 |
|------|------|
| **单符号位** | 最高位进位 $C_1$ ⊕ 符号位进位 $C_0$ = 1 则溢出 |
| **双符号位** | $S_1S_2$ = 01 正溢出，10 负溢出 |
| **硬件实现** | 用**异或门**判断溢出 |

- 溢出根本原因：**寄存器位数有限**
- 左移溢出条件：符号位 ≠ 最高数位

### IEEE 754 单精度浮点数

| 字段 | 位数 |
|------|------|
| 符号位 S | 1位 |
| 阶码 E | 8位（**移码**，偏置值127） |
| 尾数 M | 23位（原码小数，隐含整数位1） |

$$x = (-1)^S \times 1.M \times 2^{E-127}$$

**不能精确表示**：**1.2**（二进制无限循环）
**能精确表示**：1.25, 2.0, 2.5

### 移位运算

| 类型 | 规则 |
|------|------|
| 算术左移 | 低位补0，高位丢失，符号位变化则**溢出** |
| 算术右移 | 高位**补符号位**，低位丢失 |
| 逻辑右移 | 高位**补0**，低位丢失 |

### 原码乘法
- 符号位单独处理（**异或**运算）
- 绝对值相乘

## 三、存储系统

### 存储器分类
- **RAM**（随机存取）：SRAM（Cache）、DRAM（主存）
- **ROM**（只读）：掩模ROM、PROM、EPROM、闪存
- **DAM**（直接存取）：**磁盘**
- **相联存储器**：按**内容或地址**寻址（TLB、Cache）

### 芯片引脚计算

| 芯片类型 | 地址线 | 数据线 |
|---------|--------|-------|
| SRAM | $\log_2(深度)$ | 位宽 |
| DRAM | $\frac{1}{2}\log_2(深度)$（复用）| 位宽 |

**例题**：1024×8位 SRAM → 地址线10 + 数据线8 = **18根**
**例题**：1024×8位 DRAM → 地址线5 + 数据线8 = **13根**
**例题**：4M×8位 DRAM → 地址线11 + 数据线8 = **19根**

### Cache

| 概念 | 说明 |
|------|------|
| **Cache功能** | 全部由**硬件**实现 |
| **替换单位** | **块**（不是字） |
| **LRU算法** | 与局部性原理有关 |
| **全写法** | 同时写Cache和主存，**安全性高** |
| **回写法** | 只写Cache，换出时写回，**速度快** |

**平均存取时间**：$T_{avg} = H \times T_{cache} + (1-H) \times (T_{cache}+T_{mem})$

**例题**：Cache 2ns，主存 40ns，要求平均 3ns
→ $2x + 42(1-x) = 3$ → $x = 97.5\%$

### Cache行位数
Cache行位数 = 数据位 + Tag位 + 脏位 + 有效位

### 虚拟存储器
- 基础：**局部性原理**
- 程序访问主存是**不均匀的**
- **页表不命中 → Cache一定也不命中**

## 四、指令系统

### 指令格式
- **定长指令**：所有指令长度相同
- **变长指令**：扩展操作码技术

**扩展操作码**：短操作码留出编码作为扩展窗口

### 寻址方式速查

| 方式 | 有效地址 | 特点 |
|------|---------|------|
| 立即寻址 | 操作数在指令中 | 最快取数 |
| **直接寻址** | EA = A | 地址码即地址 |
| **间接寻址** | EA = (A) | 多次访存 |
| 寄存器寻址 | 操作数在寄存器 | 最快 |
| 寄存器间接 | EA = (R) | 寄存器存地址 |
| **变址寻址** | EA = (IX) + A | 适合数组 ✅ |
| **相对寻址** | EA = (PC) + A | 转移指令 |
| 基址寻址 | EA = (BR) + A | 程序重定位 |

- 偏移寻址：基址、相对、变址（不含间接寻址）
- 一地址指令：既可处理单操作数，也可处理双操作数（隐含一个）

### 相对寻址计算
- 目标地址 = (PC) + 偏移量
- 注意：PC在取指过程中每取一个字节就自动+1

## 五、中央处理器

### 寄存器可见性

| 可见 | 透明（不可见） |
|-----|-------------|
| PC（程序计数器） | IR（指令寄存器） |
| 通用寄存器 | MAR / MDR |
| 状态/标志寄存器 | 微指令寄存器 |

### 指令周期
- **时钟周期**：最小时间单位
- **机器周期**：由**存取周期**决定
- **指令周期**：取指令 + 分析指令 + 执行指令
- 第一个机器周期：**取指周期**
- 取指令是控制器固有功能，不受操作码控制

### 数据通路
- **单总线**：ALU一个输入端接总线，另一输入端通过**暂存器**接总线
- **三态门**：控制部件与总线的连接/断开
- 同一时刻：只能一个器件**发送**，可以多个器件**接收**
- **操作元件**：ALU、多路选择器（MUX）
- **状态元件**：寄存器、PC、通用寄存器组

### 控制器对比

| 特性 | 硬布线控制器 | 微程序控制器 |
|------|------------|------------|
| 速度 | **快** | 慢（读控制存储器） |
| 修改难度 | 难 | 容易 |
| 时序系统 | 复杂 | **简单** |

- 控制存储器（CS）：**CPU内部**，**ROM**实现，存**微指令**
- 一条机器指令 = 一段微程序 = 若干微指令
- 微指令后继地址确定：**断定方式**

### 微命令
- **相容性微命令**：同一CPU周期可同时出现
- **互斥性微命令**：同一CPU周期不允许同时出现

### 流水线
- 提高指令**吞吐率**（不是单条指令执行速度）
- 可能发生：结构冲突、数据冲突、控制冲突

## 📝 必背公式速查

```
MIPS = 主频(MHz) / 平均CPI
执行时间 = 指令数 × CPI / 主频
平均CPI = Σ(CPI_i × 比例_i)
Cache平均访问 = H × Tc + (1-H) × (Tc+Tm)
32位补码范围：-2^31 ~ 2^31-1
芯片地址线 = log₂(深度)，DRAM减半
相对寻址：EA = (PC) + 偏移量
```

## ⚠️ 易错点
1. CPI与时钟频率**无关**
2. 机器字长≠指令字长≠存储字长
3. DRAM地址线复用（减半），SRAM不复用
4. 补码和移码的0是唯一的（原码和反码有两个0）
5. Cache功能全部由**硬件**实现
6. 取指令**不受**操作码控制
7. 指令寄存器（IR）对程序员**透明**，PC**可见**
8. 科学计算最有用参数：**MFLOPS**（不是MIPS）
