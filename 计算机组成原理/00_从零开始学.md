## 1.1 计算机系统的组成

### 1.1.1 完整的计算机系统 = 硬件 + 软件

**生活类比**：把计算机看作一个人。
- **硬件** = 人的身体（骨骼、肌肉、五官）-- 看得见摸得着的部分
- **软件** = 人的知识、经验、思维 -- 让身体能做事的"灵魂"

缺了硬件，软件无处安放；缺了软件，硬件只是一堆废铁。**两者缺一不可**。

```
完整的计算机系统
├── ⚙️ 硬件系统
│   ├── 运算器
│   ├── 控制器
│   ├── 存储器
│   ├── 输入设备
│   └── 输出设备
└── 📦 软件系统
    ├── 系统软件（操作系统、编译程序等）
    └── 应用软件（游戏、浏览器等）
```

**⚠️ 考点：完整的计算机系统 = 硬件系统 + 软件系统**（不是只有硬件，也不是只有主机）


### 1.1.2 冯·诺依曼计算机

冯·诺依曼是计算机界的"祖师爷"。他在 1945 年提出的**存储程序**思想，至今仍然是几乎所有通用计算机的基石。

#### (1) 存储程序思想 -- 核心中的核心

**生活类比**：做菜。
- 传统方式：师傅在旁边一步步指导你（边想边做）
- 存储程序方式：师傅先把**整本菜谱**写在纸上，你拿着菜谱一步步照着做

菜谱 = 程序（指令的集合）；纸 = 存储器；你 = CPU

**关键点**：指令和数据**以二进制形式存放在存储器中，形式上没有差别**。

```
存储器中的一段二进制：10110011 01001110
它究竟是指令还是数据？
→ CPU在"取指阶段"把它当指令
→ CPU在"执行阶段"把它当数据
→ 依据：指令周期的不同阶段 ⚠️
```

#### (2) 冯·诺依曼机的五大部件

- **运算器**：负责算术运算和逻辑运算（像计算器）
- **控制器**：负责指挥协调，从存储器取指令、分析指令、控制执行（像指挥官）
- **存储器**：存放指令和数据（像仓库）
- **输入设备**：把外部信息送进计算机（键盘、鼠标）
- **输出设备**：把结果展示出来（显示器、打印机）

#### (3) 基本特点总结

| 特点 | 说明 |
|------|------|
| **控制流驱动** | 按指令序列依次执行（不是数据驱动，也不是请求驱动） |
| **按地址访问** | 指令和数据都通过地址来读写 |
| **顺序执行指令** | 默认一条接一条执行（除非遇到跳转指令） |
| **存储程序** | 程序和数据预先存入存储器 |
| **二进制表示** | 指令和数据都用0和1表示 |

**⚠️ 区分指令和数据的依据**：**指令周期的不同阶段**
- 取指阶段 → 从存储器读出的内容当作**指令**，送入IR
- 执行阶段 → 从存储器读出的内容当作**数据**，送入运算器


### 1.1.3 运算器的组成

**运算器包含**：
- ⭕ **ALU**（算术逻辑单元）-- 真正做计算的部件
- ⭕ **状态寄存器**（标志寄存器）-- 记录运算结果的状态（是否溢出、是否为0等）
- ⭕ **数据总线** -- 传输数据的通路

**运算器不包含**：
- ❌ **地址寄存器** -- 地址寄存器属于CPU的**控制器**部分，用于存放要访问的内存地址

> 记忆口诀：运算器做"算术" -- ALU算数，状态寄存器记结果，数据总线传数据。**地址**的事情归控制器管。

### Q2 冯·诺依曼机工作方式

> **题目**：冯·诺依曼计算机的工作方式是（ ）

> B. 数据流驱动方式
> C. 请求驱动方式
> D. 事件驱动方式

**解析**：
- **冯·诺依曼机是典型的"控制流驱动"计算机**

### Q4 运算器不包含

## 1.2 计算机的工作层次与语言

### 1.2.1 三种语言层次

**生活类比**：不同层次的沟通方式。

| 语言层次 | 类比 | 特点 |
|---------|------|------|
| **机器语言** | 0和1组成的"机器脑电波" | 计算机唯一能直接"听懂"的语言 |
| **汇编语言** | 给机器语言起的英文缩写"外号" | 一一对应机器指令，但和人更亲近一点 |
| **高级语言** | 我们说的人话（C、Java、Python） | 最接近人类思维方式，但计算机听不懂 |

**层次关系**：

```
高级语言（C/Java）    ← 人类最容易写
     ↓ 编译/解释
汇编语言              ← 用助记符（ADD、MOV等）
     ↓ 汇编
机器语言（二进制）     ← 计算机唯一可直接执行 ⚠️
```


### 1.2.2 编译程序 vs 解释程序

高级语言不能被计算机直接执行，需要翻译。翻译方式有两种：

**生活类比**：把一本英文小说变成中文电影。

| 方式 | 类比 | 过程 | 特点 |
|------|------|------|------|
| **编译程序** | 先把整本书翻译成中文剧本，再拍电影 | 全部翻译 → 生成目标文件 → 执行 | 一次翻译，执行**快** |
| **解释程序** | 找个翻译在旁边，说一句翻一句 | 边翻译边执行 | 不生成目标文件，执行**慢** |

**⚠️ 考点**：
- **编译程序**：一次翻译全部源代码 → 生成目标代码文件 → 运行速度快
  - 代表语言：C、C++
- **解释程序**：逐句翻译执行 → 不生成目标文件 → 运行速度慢
  - 代表语言：JavaScript、Python


### 1.2.3 从高级语言到可执行文件的四个阶段

**生活类比**：做一道复杂的菜。
1. **预处理** → 洗菜、切菜、准备调料（处理 #include、宏定义等）
2. **编译** → 按菜谱步骤做出半成品（高级语言 → 汇编语言）
3. **汇编** → 把半成品装盘（汇编语言 → 机器语言目标文件）
4. **链接** → 把各个盘子组合成完整宴席（多个目标文件 → 可执行文件）

**⚠️ 记忆口诀**："预编译汇链"（预 → 编 → 汇 → 链）

```
源代码(.c/.cpp)
    ↓ 预处理（处理 #define、#include 等）
预处理后的源文件(.i)
    ↓ 编译（高级语言 → 汇编语言）
汇编文件(.s/.asm)
    ↓ 汇编（汇编语言 → 机器语言）
目标文件(.o/.obj)
    ↓ 链接（多个目标文件 + 库文件 → 可执行文件）
可执行文件(.exe)
```


### Q5 编译程序与解释程序

> **题目**：关于编译程序和解释程序，下列说法正确的是（ ）
> A. 编译程序边翻译边执行
> B. 解释程序生成目标代码文件
> C. 编译程序一次翻译全部源代码，生成目标代码文件
> D. 解释程序的执行速度比编译程序快

**解析**：
- A 错误：**边翻译边执行是解释程序**的特点，不是编译程序
- B 错误：**不生成目标文件的是解释程序**，编译程序会生成目标文件
- ⭕ **C 正确**：编译程序一次翻译全部源代码 → 生成目标代码文件 → 运行时直接执行目标代码
- D 错误：解释程序**逐句翻译执行，速度慢**；编译程序一次翻译后直接执行，速度快

**答案：C**

### Q7 语言级别描述错误

> **题目**：下列关于计算机语言层次的描述，错误的是（ ）
> A. 机器语言是计算机唯一可直接执行的语言
> B. 汇编语言与机器结构相关，不可移植
> C. 高级语言编写的程序计算机可以直接执行

**解析**：
- A 正确：机器语言是二进制代码，CPU直接识别执行

- ❌ **C 错误：高级语言编写的程序必须经过编译或解释才能被计算机执行**，不能直接执行
- D 正确：两者目标都是把高级语言翻译成机器能懂的形式

**答案：C**

### Q10 冯·诺依曼思想错误选项

> **题目**：下列关于冯·诺依曼思想的描述，错误的是（ ）
> A. 采用存储程序方式
> B. 指令和数据用二进制表示
> C. 指令和数据在存储器中形式上没有差别
> D. CPU根据指令的地址码来区分指令和数据

**解析**：
- A 正确：存储程序是冯·诺依曼思想的核心

- ❌ **D 错误：CPU区分指令和数据的依据是指令周期的不同阶段，不是地址码**
  - 取指阶段：从存储器取出的内容 → 送到IR → 当作指令处理
  - 执行阶段：从存储器取出的内容 → 送到运算器 → 当作数据处理

**答案：D**


### Q11 编译链接过程

> **题目**：将高级语言程序转换为可执行文件，正确的顺序是（ ）
> A. 编译 → 汇编 → 预处理 → 链接
> B. 预处理 → 编译 → 汇编 → 链接
> C. 预处理 → 汇编 → 编译 → 链接
> D. 编译 → 预处理 → 汇编 → 链接

**解析**：
- 正确顺序是：**预处理 → 编译 → 汇编 → 链接**
- **预处理**：处理 #define（宏定义）、#include（文件包含）等预处理指令
- **编译**：将预处理后的源文件翻译成**汇编语言**（高级 → 汇编）
- **汇编**：将汇编语言翻译成**机器语言目标文件**（汇编 → 机器）
- **链接**：将多个目标文件和库文件组合成**可执行文件**

> 记忆口诀：**"预编译汇链"** -- 预习、编译、汇编、串链

**答案：B**


## 1.3 计算机的性能指标

### 1.3.1 三种"字长" -- 最容易混淆的概念

**生活类比**：三条不同宽度的公路。

| 字长类型 | 定义 | 类比 |
|---------|------|------|
| **机器字长** | CPU一次能处理的**数据**位数（通用寄存器宽度） | 一次能拉多少货 |
| **指令字长** | 一条指令的二进制位数 | 一张送货单有多长 |
| **存储字长** | 一个存储单元中的二进制位数 = **MDR的位数** | 一个仓库隔间的大小 |

**⚠️ 核心考点：三者数值上可以相等也可以不等！**

也就是说：CPU一次处理64位数据（机器字长），但一条指令可能是32位（指令字长），一个存储单元也可能是32位（存储字长）-- 这完全正常。

#### 机器字长（最重要）

- CPU**运算器**的位数
- = **通用寄存器**的宽度
- = 数据总线的宽度（通常）
- 决定了计算机的**运算精度**
- 常见值：16位、32位、64位

> 注意区分：
> - **机器字长**：CPU处理数据的宽度
> - **指令字长**：一条指令的二进制位数（和机器字长无关）
> - **存储字长**：一个存储单元的位数 = MDR的位数

**⚠️ 易错**：计算机字长**不一定都是32位**，有16位、32位、64位等。


### 1.3.2 时钟相关概念

#### 时钟周期（Clock Cycle）

**生活类比**：心跳。
- 心脏每跳动一次 = 一个时钟周期
- 心跳越快，完成动作越快

**定义**：CPU的最小时间单位，是CPU工作的"节拍"。

```
时钟周期 = 1 / 主频
```

#### 主频（Clock Frequency）

**定义**：时钟频率，即每秒有多少个时钟周期。

单位：**Hz**（赫兹）

| 单位 | 换算 | 举例 |
|------|------|------|
| 1 Hz | 1次/秒 | -- |
| 1 MHz | 10^6 Hz | 早期的CPU |
| 1 GHz | 10^9 Hz | 现代CPU（如3.0GHz） |

**例如**：主频 2GHz = 2 × 10^9 Hz
时钟周期 = 1 / (2 × 10^9) = 0.5 ns（纳秒）

**⚠️ 时钟周期是计算机操作的最小时间单位**


### 1.3.3 CPI -- 平均执行一条指令需要几个时钟周期

**生活类比**：做一道菜需要几个步骤。
- CPI = 平均做一道菜所需的步骤数
- 不同"菜"（指令）需要的步骤不同
  - 简单的菜（空操作 NOP）：1步
  - 复杂的菜（乘法 MUL）：可能5步

**CPI = 总时钟周期数 / 指令条数**

**⚠️ CPI与什么无关？ -- 与时钟频率（主频）无关！**

CPI由**系统结构**、**指令集**、**计算机组织**决定，但不取决于时钟跑多快。

> 就好比：做一道菜需要几个步骤（CPI）和你手脚多快（主频）是两回事。
> 步骤固定了，不论你动作多快，步骤数不变。


### 1.3.4 MIPS -- 每秒百万条指令

**定义**：Million Instructions Per Second

```
MIPS = 主频（MHz）/ CPI
```

**生活类比**：你的工作效率。
- 主频 = 你每秒能眨眼的次数（节拍数）
- CPI = 做一件事需要眨几次眼
- MIPS = 你每秒能做多少件事

**单位注意**：主频用 MHz（兆赫兹），计算时 1GHz = 1000MHz


### 1.3.5 MFLOPS -- 科学计算的王者

**定义**：Million Floating-point Operations Per Second，每秒百万次浮点运算。

**⚠️ 重点：科学计算最看重的指标是MFLOPS，不是MIPS！**

为什么？
- 科学计算主要做大量的浮点运算（小数运算），而不是整数运算
- MIPS衡量的是指令执行速度，但不同的指令"含金量"不同
- MFLOPS专门衡量**浮点运算能力**，对科学计算更有意义

| 指标 | 适用场景 |
|------|---------|
| MIPS | 通用性能衡量 |
| **MFLOPS** | **科学计算**、工程仿真 |


### Q12 字长相关错误

> **题目**：下列关于字长的说法，错误的是（ ）
> A. 机器字长是指CPU一次能处理的二进制数据位数
> B. 存储字长是指一个存储单元的二进制位数
> C. 机器字长、指令字长、存储字长三者必须相等
> D. 指令字长是指一条指令的二进制位数

**解析**：
- A 正确：机器字长 = CPU运算器的位数 = 通用寄存器宽度
- B 正确：存储字长 = MDR的位数
- ❌ **C 错误：三者数值上可能相等也可能不等，不必须相等！** -- 这是最常考的易错点
- D 正确：指令字长就是一条指令的二进制位数

**答案：C**


### Q13 机器字长/指令字长/存储字长

> **题目**：某计算机的机器字长为32位，指令字长为16位，存储字长为8位，以下说法正确的是（ ）
> A. 三者必须统一，设计不合理
> B. CPU一次能够处理32位数据
> C. 每个存储单元有32位
> D. 指令字长必须等于机器字长

**解析**：
- A 错误：**三者可以不等**，这是常见的设计
- ⭕ **B 正确：机器字长32位 = CPU一次能处理32位数据**
- C 错误：存储字长8位 = 每个存储单元8位，不是32位
  > 注意：机器字长和存储字长没有必然的大小关系
- D 错误：指令字长可以小于、等于或大于机器字长

**答案：B**


### Q14 MFLOPS

> **题目**：以下哪个指标最适合衡量计算机的科学计算性能（ ）
> A. CPI
> B. MIPS
> C. MFLOPS
> D. 主频

**解析**：
- A CPI：执行一条指令所需时钟周期数，是内部效率指标，不是整体性能指标
- B MIPS：每秒百万条指令，但**不同指令的"含金量"不同**，不能反映浮点运算能力
- ⭕ **C MFLOPS：每秒百万次浮点运算，专门衡量科学计算中的浮点运算能力**
- D 主频：只是时钟频率，不能直接作为性能衡量标准（不同架构的CPU同主频性能可能差很多）

> 扩展：科学计算中大量使用浮点数运算，MFLOPS衡量的是真实的浮点运算吞吐量。

**答案：C**


## 1.4 性能计算（必考！）

### 1.4.1 三大公式

#### 公式一：执行时间

```
执行时间 = 指令条数 × CPI / 主频
```

**生活类比**：完成一项任务的耗时。
- 指令条数 = 多少个工作任务
- CPI = 每个任务需要几步
- 主频 = 每秒能做几步

```
执行时间 = (工作量 × 每步耗时) / 速度
         = 指令条数 × (CPI / 主频)
```

#### 公式二：平均CPI

当程序的指令有不同种类时，需要计算加权平均：

```
平均CPI = Σ (CPI_i × 比例_i)
```

#### 公式三：MIPS

```
MIPS = 主频（MHz）/ 平均CPI
```

### 1.4.2 性能变化计算（必考计算题型）

#### 场景一：CPU速度提高

公式：
```
新执行时间 = CPU时间 / 提速倍数 + I/O时间
```

注意：**I/O时间不受CPU提速影响**，只有CPU计算时间会缩短。

#### 场景二：编译优化

公式：
```
新执行时间 = 原时间 × 新指令比例 × 新CPI比例
```


### Q15 对用户透明的寄存器

> **题目**：以下哪个寄存器对程序员（用户）是透明的（不可见的）（ ）
> A. 通用寄存器
> B. 程序计数器（PC）
> C. 指令寄存器（IR）
> D. 状态寄存器

**解析**：

首先理解"透明"的含义：在计算机中，**"透明"意味着"不可见"**，跟日常生活中的"透明"正好相反。

| 寄存器 | 对程序员可见性 | 说明 |
|--------|--------------|------|
| 通用寄存器 | **可见** | 程序员可以显式读写 |
| PC（程序计数器） | **可见** | 程序员可以访问（如跳转指令） |
| 状态/标志寄存器 | **可见** | 程序员可以读取标志位 |
| **IR（指令寄存器）** | **透明（不可见）** | 存放当前正在执行的指令，程序员无需关心 |
| MAR / MDR | **透明（不可见）** | 与主存交互的中间寄存器 |
| 微指令寄存器 | **透明（不可见）** | 微程序控制器内部使用 |

- A 通用寄存器：可见 ✅
- B PC：可见 ✅
- ❌ **C 指令寄存器（IR）：透明（不可见）**
- D 状态寄存器：可见 ✅

**答案：C**


### Q16 最小时间单位

> **题目**：微型计算机操作的最小时间单位是（ ）
> A. 指令周期
> B. 机器周期
> C. 时钟周期
> D. 存取周期

**解析**：

首先理清几个"周期"的关系：

```
指令周期 > 机器周期 > 时钟周期
```

- **指令周期**：从取指令到执行完这条指令的时间（最长）
- **机器周期**：由存取周期决定，通常为一次内存访问所需时间
- **时钟周期**：主频的倒数，CPU工作的基本节拍
- **存取周期**：存储器进行一次完整读写操作所需时间

- A 指令周期：不是最小单位，它包含多个机器周期
- B 机器周期：不是最小单位，它包含多个时钟周期
- ⭕ **C 时钟周期：CPU的最小时间单位**
  > 时钟周期 = 1/主频，是CPU一切操作的"心跳"节奏
- D 存取周期：是存储器的性能参数，不是CPU的操作单位

**答案：C**


### Q17 CPI影响因素

> **题目**：以下哪个因素会影响CPU的CPI（ ）
> A. 时钟频率
> B. 系统结构
> C. 主存容量
> D. 硬盘速度

**解析**：

**⚠️ CPI与时钟频率（主频）无关！** 这是最重要的考点之一。

CPI（Cycles Per Instruction）取决于：
- ✅ **系统结构**（是否采用流水线、超标量等）
- ✅ **指令集**（RISC vs CISC，指令复杂度不同）
- ✅ **计算机组织**（硬件实现方式）

CPI与以下因素**无关**：
- ❌ 时钟频率（主频）
- ❌ 主存容量
- ❌ 硬盘速度

- A ❌ 时钟频率：**不相关**！CPI是"步骤数"，时钟频率是"每秒步骤数"，两者独立
- ⭕ **B 系统结构：相关**，不同的系统结构导致CPI不同
- C ❌ 主存容量：不直接影响每个指令的时钟周期数
- D ❌ 硬盘速度：不直接影响

**答案：B**


### Q18 CPU速度提高计算

> **题目**：某程序在计算机上运行，CPU时间占90s，I/O时间占10s，总共100s。若CPU速度提高50%，则新的总执行时间为（ ）
> A. 70s
> B. 66.7s
> C. 75s
> D. 60s

**解析**：

这是经典题型，关键要点：
1. **CPU速度提高50%** → 原来的1.5倍快 → CPU时间**缩短**，而不是变长
2. **I/O时间不受CPU提速影响**

计算过程：
```
原CPU时间 = 90s
CPU速度提高50% → 新CPU时间 = 90 / 1.5 = 60s
I/O时间不变 = 10s
新总时间 = 60 + 10 = 70s
```

> 常见错误：90 × 0.5 = 45，然后 90 - 45 = 45 + 10 = 55 ❌
> 正确理解：速度提高50%"意味着"变成原来的1.5倍"，时间是原来的1/1.5

**答案：A**


### Q19 MIPS计算

> **题目**：某CPU主频为1.2GHz，各类指令的CPI和比例如下：
> - A类指令：CPI=2，占50%
> - B类指令：CPI=3，占20%
> - C类指令：CPI=4，占10%
> - D类指令：CPI=5，占20%
>
> 则该CPU的平均CPI和MIPS分别是（ ）
> A. CPI=3, MIPS=400
> B. CPI=3.5, MIPS=342
> C. CPI=2.8, MIPS=428
> D. CPI=3, MIPS=360

**解析**：

**第一步：计算平均CPI**

平均CPI = Σ（CPI_i × 比例_i）

```
平均CPI = 2×50% + 3×20% + 4×10% + 5×20%
        = 2×0.5 + 3×0.2 + 4×0.1 + 5×0.2
        = 1.0 + 0.6 + 0.4 + 1.0
        = 3.0
```

**第二步：计算MIPS**

先换算主频：1.2GHz = 1200MHz

```
MIPS = 主频(MHz) / 平均CPI
     = 1200 / 3
     = 400
```

**答案：A**


### Q20 编译优化执行时间

> **题目**：某程序P在原编译器下执行时间为20s。经过编译优化后，指令条数减少到原来的70%，但CPI增加到原来的1.2倍。假设主频不变，问优化后的执行时间为（ ）
> A. 14.0s
> B. 16.8s
> C. 24.0s
> D. 11.7s

**解析**：

利用公式：执行时间 = 指令条数 × CPI / 主频

主频不变 → 执行时间与（指令条数 × CPI）成正比

```
原执行时间 = 原指令条数 × 原CPI / 主频 = 20s

新指令条数 = 原指令条数 × 70% = 原指令条数 × 0.7
新CPI = 原CPI × 1.2

新执行时间 = 新指令条数 × 新CPI / 主频
           = (原指令条数 × 0.7) × (原CPI × 1.2) / 主频
           = 原执行时间 × 0.7 × 1.2
           = 20 × 0.7 × 1.2
           = 20 × 0.84
           = 16.8s
```

> 注意：指令数减少到70% × CPI增大到1.2倍 = 0.7 × 1.2 = 0.84，所以时间缩短到原来的84%

**答案：B**


### Q21 不同主频CPI比较

> **题目**：有两台计算机M1和M2，M1主频1.5GHz，CPI=2；M2主频1.2GHz，CPI=1。运行同一程序，比较两台计算机的执行时间（ ）
> A. M1比M2快
> B. M2比M1快
> C. 一样快
> D. 无法比较

**解析**：

运行**同一程序** → 指令条数相同，设为N

**计算M1的执行时间**：
```
T1 = N × CPI1 / 主频1
   = N × 2 / (1.5 × 10^9)
   = N × 1.33 × 10^(-9)
```

**计算M2的执行时间**：
```
T2 = N × CPI2 / 主频2
   = N × 1 / (1.2 × 10^9)
   = N × 0.83 × 10^(-9)
```

对比：
```
T1 / T2 = (2/1.5) / (1/1.2) = 1.333 / 0.833 = 1.6

M2的执行时间是M1的1/1.6 = 62.5%
M2比M1快！(M1时间 : M2时间 = 1.6 : 1)
```

> 速算方法：
> M1时间 : M2时间 = (CPI1/主频1) : (CPI2/主频2)
> = (2/1.5) : (1/1.2)
> = 1.333 : 0.833
> = 1.6 : 1
>
> 所以M2更快

**扩展思考**：主频高不一定快！M1主频1.5GHz > M2主频1.2GHz，但M2的CPI更低，反而跑得更快。

**答案：B**

## 2.1 数制转换

### 2.1.1 为什么有不同数制？

**生活类比**：
- 十进制（Decimal）：我们日常用的，0-9，逢十进一
- 二进制（Binary）：计算机用的，0-1，逢二进一
- 八进制（Octal）：0-7，逢八进一
- 十六进制（Hexadecimal）：0-9和A-F，逢十六进一

计算机内部所有数据都是**二进制**（0和1），但二进制写起来太长，所以用**八进制**或**十六进制**作为缩写法。

```
二进制    八进制    十进制    十六进制
0000        0        0         0
0001        1        1         1
0010        2        2         2
0011        3        3         3
0100        4        4         4
0101        5        5         5
0110        6        6         6
0111        7        7         7
1000       10        8         8
1001       11        9         9
1010       12       10         A
1011       13       11         B
1100       14       12         C
1101       15       13         D
1110       16       14         E
1111       17       15         F
```

**十六进制为什么常用？**
- 1位十六进制数 = 4位二进制数，一一对应
- 例如：二进制 `1011 1110` = 十六进制 `BE`
- 比二进制短4倍，比八进制更紧凑

### 2.1.2 十进制转二进制 -- 整数部分

**方法：除2倒取余法**

用十进制数不断除以2，记录余数，直到商为0，然后将余数从下往上倒序排列。

**例：将十进制 66 转为二进制**

```
66 ÷ 2 = 33 ... 余 0  ↑
33 ÷ 2 = 16 ... 余 1  │
16 ÷ 2 =  8 ... 余 0  │
 8 ÷ 2 =  4 ... 余 0  │
 4 ÷ 2 =  2 ... 余 0  │
 2 ÷ 2 =  1 ... 余 0  │
 1 ÷ 2 =  0 ... 余 1  │
                       │
从下往上读：1000010
```

所以 66(D) = 1000010(B)

**验证**：1×2^6 + 0×2^5 + 0×2^4 + 0×2^3 + 0×2^2 + 1×2^1 + 0×2^0 = 64 + 2 = 66 ✅

**扩充到8位（补足高位）**：0100 0010

**口诀**：除2取余，倒序排列。

### 2.1.3 十进制转二进制 -- 小数部分

**方法：乘2取整法**

用小数部分不断乘以2，取出整数部分，直到小数部分为0或达到精度要求。

**例：将十进制 0.625 转为二进制**

```
0.625 × 2 = 1.25 → 取整数 1  ↓
0.25  × 2 = 0.5  → 取整数 0  │
0.5   × 2 = 1.0  → 取整数 1  │
                                │
从上往下读：0.101
```

所以 0.625(D) = 0.101(B)

### 2.1.4 任何十进制整数都可以用二进制表示，但任何十进制小数不一定

**重要结论**：任何一个十进制整数都能用有限位二进制精确表示，但**不是所有十进制小数都能用有限位二进制表示**。

**例：十进制 0.1 转二进制（无限循环）**

```
0.1 × 2 = 0.2 → 0
0.2 × 2 = 0.4 → 0
0.4 × 2 = 0.8 → 0
0.8 × 2 = 1.6 → 1
0.6 × 2 = 1.2 → 1
0.2 × 2 = 0.4 → 0  ← 开始循环！
0.4 × 2 = 0.8 → 0
...
```

所以 0.1(D) = 0.00011001100110011...(B) **无限循环小数**

**这意味着什么？**
- 计算机无法精确表示 0.1！
- 这就是为什么在编程中 `0.1 + 0.2 != 0.3`（浮点精度问题）
- 也是 IEEE754 中 1.2 无法精确表示的原因

**哪些小数可以精确表示？**
- 分母是2的幂的分数可以精确表示
- 例如：0.5(=1/2), 0.25(=1/4), 0.125(=1/8), 0.625(=5/8)
- 例如：0.3(=3/10)不能精确表示，因10不是2的幂

### 2.1.5 二进制、八进制、十六进制之间的转换

**二进制转八进制**：每3位一组（从右向左），每组转换为1位八进制数

**例**：`010 000 010`(B) → `2 0 2`(O) = 202(O)

**二进制转十六进制**：每4位一组（从右向左），每组转换为1位十六进制数

**例**：`0100 0010`(B) → `4 2`(H) = 42(H)

**十六进制转二进制**：1位十六进制数展开为4位二进制数

**例**：`BE`(H) → `1011 1110`(B)


## 2.2 原码、反码、补码、移码

### 2.2.1 为什么需要不同的编码方式？

**生活类比**：如何表示"欠钱"和"存钱"？

假设你有一个数字显示牌，只能显示"+"和"-"以及数字。现在我们要在计算机中用0和1表示正负数。

计算机面临的问题：**只有0和1，没有+和-，怎么办？**

### 2.2.2 原码 -- 最直观的方式

**定义**：最高位为**符号位**（0正1负），其余位为**数值的绝对值**。

```
原码 = 符号位 + 绝对值
```

**例：用8位原码表示**

```
+66 = 0 1000010 = 0100 0010
-66 = 1 1000010 = 1100 0010
  +0 = 0000 0000
  -0 = 1000 0000
```

**特点**：
- 直观易懂，符号位+绝对值
- **0有两种表示**：+0 = 0000...0，-0 = 1000...0
- 8位原码能表示 255 个不同数据（0占两个位置）
- n位原码整数范围：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$

**缺点**：
1. 0有两种表示，浪费一个编码
2. 做加减法时，符号位需单独处理，电路复杂

**记忆口诀**："原码最直观，符号+绝对值，0有两个，运算最麻烦。"

### 2.2.3 反码 -- 过渡方案

**定义**：
- **正数**：与原码相同
- **负数**：原码符号位不变，数值位**按位取反**

**例：用8位反码表示**

```
+66 = 0100 0010（同原码）
-66 = 1011 1101（符号位1不变，数值位取反）
  +0 = 0000 0000
  -0 = 1111 1111
```

**特点**：
- **0有两种表示**：+0=0000...0，-0=1111...1
- n位反码整数范围：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$（与原码相同）
- 反码是**补码的中间步骤**

**记忆口诀**："反码正同原，负则符号不变位取反，0也有两个。"

### 2.2.4 补码 -- 计算机最常用的编码

**定义**：
- **正数**：与原码相同
- **负数**：原码符号位不变，数值位**取反再加1**（即反码+1）

**例：求-66的8位补码**

```
  +66的原码 = 0100 0010
  取反     = 1011 1101（这就是-66的反码）
  加1      = 1011 1110（这就是-66的补码）
```

所以 [-66]补 = 1011 1110 = BE(H) = 0xBE

**例：求-1的8位补码**

```
  +1的原码 = 0000 0001
  取反     = 1111 1110
  加1      = 1111 1111（-1的补码）
```

**特点**：
- **0只有一种表示**：0000...0 ✅ -- 这是补码最重要的优势
- n位补码整数范围：$-2^{n-1} \sim 2^{n-1}-1$（比原码多表示一个数）
- 8位补码范围：-128 ~ 127（共256个数）
- **计算机系统中实际使用的是补码**

**为什么负数补码用"取反加1"？**

根本原因是：**让加减法统一用加法器实现**，不需要单独的减法器。

```
[x]补 + [y]补 = [x+y]补
[x]补 - [y]补 = [x]补 + [-y]补
```

**关键转换 -- 由[X]补求[-X]补**：

规则：**[X]补连同符号位取反，末位加1**（注意是"连同符号位"！）

```
例：[66]补 = 0100 0010
求[-66]补：
取反：1011 1101
加1： 1011 1110  ✅ = 0xBE

例：[-66]补 = 1011 1110
求[66]补：
取反：0100 0001
加1： 0100 0010  ✅
```

**补码的符号位扩展**：
- 正数补码：高位补0
- 负数补码：高位补1（符号位扩展）

```
例：将8位补码扩展为16位
[66]补 = 0100 0010 → 0000 0000 0100 0010
[-66]补 = 1011 1110 → 1111 1111 1011 1110
```

### 2.2.5 移码 -- 浮点数的阶码

**定义**：补码的符号位取反即得移码。

```
[X]移 = 2^(n-1) + X（偏置值为2^(n-1)）
```

**特点**：
- **0只有一种表示**（与补码一样）
- 数值部分与补码相同，符号相反
- 主要用于浮点数的**阶码**表示
- IEEE754中移码的偏置值为127（单精度）或1023（双精度）

**补码和移码的关系 -- 常考！**

| 对比项 | 补码 | 移码 |
|--------|------|------|
| 0的表示 | 唯一：000...0 | 唯一：100...0 |
| 数值部分 | 与移码相同 | 与补码相同 |
| 符号位 | 与移码相反 | 与补码相反 |
| 表示范围 | $-2^{n-1} \sim 2^{n-1}-1$ | 同补码 |

**记忆口诀**："补码移码是一家，数值相同符号反。"

**例：比较8位补码和移码**

```
真值     补码         移码（补码符号位取反）
  0     0000 0000    1000 0000
  1     0000 0001    1000 0001
 -1     1111 1111    0111 1111
127     0111 1111    1111 1111
-128    1000 0000    0000 0000
```

**⚠️ 注意**：移码的偏置值可以是不同的值（取决于位数），不只是2^(n-1)。IEEE754中用的是2^(n-1)-1。


## 2.3 定点数

### 2.3.1 什么是定点数？

**定义**：小数点位置固定不变的数。

计算机中定点数有两种：
- **定点整数**：小数点在最低位之后（可以认为小数点固定在末尾）
- **定点小数**：小数点在最高位之后（符号位之后）

### 2.3.2 n位定点整数范围

**n位无符号定点整数**：0 ~ 2^n - 1

**n位原码定点整数**：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$

**n位补码定点整数**：$-2^{n-1} \sim 2^{n-1}-1$

| 位数 | 补码范围 |
|------|---------|
| 8位 | -128 ~ 127 |
| 16位 | -32768 ~ 32767 |
| 32位 | -2^31 ~ 2^31-1 |
| n位 | $-2^{n-1} \sim 2^{n-1}-1$ |

### 2.3.3 补码的符号位扩展

将一个n位补码扩展为m位（m>n）时：
- **正数**补码：高位补0
- **负数**补码：高位补1

```
例1：将8位补码[66]补 = 0100 0010 扩展为16位
    16位[66]补 = 0000 0000 0100 0010
    高位全部补0（因为符号位是0）

例2：将8位补码[-66]补 = 1011 1110 扩展为16位
    16位[-66]补 = 1111 1111 1011 1110
    高位全部补1（因为符号位是1）
```

**验证**：`1111 1111 1011 1110` 的补码 = ？
- 取反：`0000 0000 0100 0001`
- 加1：`0000 0000 0100 0010` = 66
- 所以这个是-66 ✅

### 2.3.4 n位定点小数

对于n位补码定点小数（1位符号位 + n-1位数值位）：
- 范围：$-1 \sim 1-2^{-(n-1)}$
- 最小负数：-1（补码表示：1.000...0）
- 最大正数：$1-2^{-(n-1)}$（补码表示：0.111...1）


## 2.4 IEEE754浮点数（难点！）

### 2.4.1 为什么需要浮点数？

**生活类比**：科学记数法。

我们在科学计算中经常需要表示：
- 很大的数：$6.02 \times 10^{23}$（阿伏伽德罗常数）
- 很小的数：$1.6 \times 10^{-19}$（电子电荷量）

如果用定点数表示，位数不够用。所以需要用**浮点数**，类似于科学记数法。

```
科学记数法：  ±1.xxxx × 2^(yyyy)
浮点数：      符号位 + 阶码 + 尾数
```

### 2.4.2 IEEE754单精度浮点数（32位）

**格式**：

```
┌─────┬───────────┬──────────────────────────────┐
│ 1位 │    8位     │             23位              │
│  S  │     E      │              M               │
│符号位│  阶码（移码）│          尾数（原码）           │
└─────┴───────────┴──────────────────────────────┘
```

| 字段 | 位数 | 含义 |
|------|------|------|
| S | 1位 | 符号位（0正1负） |
| E | 8位 | 阶码，用**移码**表示，偏置值**127** |
| M | 23位 | 尾数，原码小数，**隐含整数位1** |

**真值计算公式**：

$$x = (-1)^S \times 1.M \times 2^{E-127}$$

**关键点 -- 隐含整数位1（"隐藏1"）**

规格化浮点数的尾数部分满足 $1 \leq 尾数 < 2$，因此整数部分总是1。为了省1位精度，这个1**不存储**，只存小数部分（M）。

所以：实际尾数 = 1.M

**例如**：
- 如果M = 010000...0（23位）
- 则实际尾数 = 1.01...（二进制）

### 2.4.3 IEEE754双精度浮点数（64位）

**格式**：

```
┌─────┬─────────────┬────────────────────────────────────────┐
│ 1位 │    11位      │                  52位                   │
│  S  │      E       │                   M                   │
│符号位│ 阶码（移码）  │              尾数（原码）               │
└─────┴─────────────┴────────────────────────────────────────┘
```

| 字段 | 位数 | 含义 |
|------|------|------|
| S | 1位 | 符号位 |
| E | 11位 | 阶码，偏置值**1023** |
| M | 52位 | 尾数，隐含整数位1 |

$$x = (-1)^S \times 1.M \times 2^{E-1023}$$

### 2.4.4 规格化数、非规格化数、无穷大、0 的区分

IEEE754 用阶码E的特殊值来区分不同类型：

| 类型 | E的值 | M的值 | 含义 |
|------|-------|-------|------|
| **0** | E=0 | M=0 | 真值0（+0和-0） |
| **非规格化数** | E=0 | M≠0 | 非常接近0的数，隐含整数位为0 |
| **规格化数** | 1≤E≤254 | 任意 | 正常浮点数，隐含整数位为1 |
| **无穷大** | E=255(全1) | M=0 | 正无穷/负无穷 |
| **NaN** | E=255(全1) | M≠0 | Not a Number（非法运算结果） |

**规格化数**（E的范围：1 ~ 254）：
- 隐含整数位：1
- 公式：$x = (-1)^S \times 1.M \times 2^{E-127}$
- 单精度范围：约 $\pm 1.18 \times 10^{-38} \sim \pm 3.4 \times 10^{38}$

**非规格化数**（E=0，M≠0）：
- 隐含整数位：0（不是1！）
- 公式：$x = (-1)^S \times 0.M \times 2^{-126}$
- 用于表示非常接近0的数，填补了规格化数到0之间的"间隙"

**零**（E=0，M=0）：
- 有+0（S=0）和-0（S=1）两种

**无穷大**（E=255，M=0）：
- 有正无穷（S=0）和负无穷（S=1）

### 2.4.5 单精度浮点数举例

**例：将十进制数 -66.0 转为IEEE754单精度**

1. 先转二进制：66 = 1000010(B)
2. 规格化：66.0 = 1.000010 × 2^6
3. 符号位 S = 1（负数）
4. 阶码 E = 6 + 127 = 133 = 10000101(B)
5. 尾数 M = 00001000000000000000000（去掉整数1，取23位）

所以：S=1, E=10000101, M=00001000000000000000000
合起来：1 10000101 00001000000000000000000
= 1100 0010 1000 0100 0000 0000 0000 0000
= C2 84 00 00(H)

**例：IEEE754单精度 0x3F800000 表示什么？**

0x3F800000 = 0 01111111 00000000000000000000000

- S = 0（正数）
- E = 01111111 = 127
- M = 000...0

真值 = $(-1)^0 \times 1.0 \times 2^{127-127} = 1.0$

所以 0x3F800000 表示 1.0 ✅

### 2.4.6 不能精确表示的十进制小数

**结论**：**1.2 不能用IEEE754精确表示**（二进制无限循环）。

原因：1.2 = 1 + 0.2
- 1 = 1(B)
- 0.2 = 0.0011001100110011...(B) **无限循环**

所以 1.2(B) = 1.0011001100110011...(无限循环) → 只能近似存储

**能精确表示的十进制数**：
- 1.25（二进制1.01，有限位）
- 2.0（二进制10.0，有限位）
- 2.5（二进制10.1，有限位）
- 0.5, 0.25, 0.125 等（分母为2的幂的分数）


## 2.5 边界对齐与大小端

### 2.5.1 为什么需要边界对齐？

**生活类比**：图书馆的书架。

- **对齐排放**：每本书对应一个固定大小的格子，找书快（效率高）
- **不对齐**：书可以随意放，但找书慢（效率低）

在计算机中：
- **边界对齐**：数据的起始地址必须是数据长度的整数倍
- 如：int（4字节）的地址必须是4的倍数
- 如：short（2字节）的地址必须是2的倍数

**边界对齐（Alignment）规则**：

```
数据的起始地址 = 数据长度（字节数）的整数倍

char（1字节）：地址可以是任意值
short（2字节）：地址必须是2的倍数
int（4字节）：地址必须是4的倍数
double（8字节）：地址必须是8的倍数
```

**为什么需要对齐？**
- 现代CPU一次读取固定长度的数据（如4字节或8字节）
- 如果数据跨边界（不对齐），CPU需要两次内存访问，效率低
- 对齐后一次就能读完

**结构体中的对齐 -- 常考！**

结构体成员按照**定义顺序**依次分配空间，每个成员按其对齐系数（自身大小）对齐，**结构体总大小是最大成员大小的整数倍**。

**例**：
```c
struct {
    char  a;   // 1字节，偏移0
    short b;   // 2字节，偏移2（需2的倍数，跳过1个字节）
    int   c;   // 4字节，偏移4（需4的倍数）
} s;
// sizeof(s) = 8

内存布局：
地址：0   1   2   3   4   5   6   7
     [a] [pad] [  b  ] [     c     ]
```

### 2.5.2 大端和小端

**生活类比**：写信的地址格式。

- **大端（Big-Endian）**：先写国名、再写城市、再写街道（高位在前）
- **小端（Little-Endian）**：先写街道、再写城市、再写国名（低位在前）

**正式定义**- **大端（Big-Endian）**：高位字节存放在低地址（最高有效字节在前）
- **小端（Little-Endian）**：低位字节存放在低地址（最低有效字节在前）

**例**：存储32位整数 0x12345678

```
内存地址递增 → 
大端：  12  34  56  78
小端：  78  56  34  12
```

**如何记忆？**

- **大端**：和书写习惯一致（从左到右，高位到低位），类似人类的阅读习惯
- **小端**：和计算机的运算习惯一致（低位先运算）

**判断大小端的方法**：在C语言中，可以用union来测试。

```c
union {
    int i;
    char c[4];
} u = {0x12345678};
// 如果 u.c[0] == 0x12 → 大端
// 如果 u.c[0] == 0x78 → 小端
```

**x86架构都是小端**，网络协议通常用大端。

> B. 补码和移码的数值部分相反，符号相同

- ⭕ **A 正确**：补码和移码的数值部分相同，符号相反
- B 错误：数值部分相反？错！是**数值部分相同，符号相反**
- C 错误：补码和移码的0表示都是**唯一的**

**记忆方法**："补码和移码，数值相同符号反" -- 这是最简洁的记忆口诀。

求-66的8位补码：

第一步：+66的原码 = 0100 0010

### Q24 32位补码整数范围

> **题目**：32位补码能够表示的最大正整数和最小负整数分别是（ ）
> A. 2^31-1 和 -2^31
> B. 2^31 和 -2^31
> C. 2^31-1 和 -(2^31-1)
> D. 2^32-1 和 -2^32

**解析**：

n位补码整数的范围公式：$-2^{n-1} \sim 2^{n-1}-1$

32位补码（n=32）：
- 最大正整数：$2^{32-1}-1 = 2^{31}-1$
- 最小负整数：$-2^{32-1} = -2^{31}$

| 位数 | 补码范围 |
|------|---------|
| 8位 | -128 ~ 127 |
| 16位 | -32768 ~ 32767 |
| 32位 | -2147483648 ~ 2147483647 |
| | $-2^{31}$ ~ $2^{31}-1$ |

**为什么最大是2^(n-1)-1不是2^(n-1)？**
- 因为0占了一个正数编码（000...0）

- 100...0 表示 -2^(n-1)（没有对应的正数）

**答案：A**


### Q25 溢出判断电路

> **题目**：补码加减法运算中，判断溢出的电路最常用的是（ ）
> A. 与门
> B. 或门
> C. 异或门
> D. 非门

**解析**：

补码加减法采用**单符号位+进位判断法**：

溢出条件 = 符号位进位 $C_0$ ⊕ 最高数值位进位 $C_1$

即：$溢出 = C_0 \oplus C_1$

**为什么是异或？**

| $C_0$（符号位进位） | $C_1$（最高数位进位） | 异或结果 | 含义 |
|:---:|:---:|:---:|------|
| 0 | 0 | **0** | 无溢出 |
| 0 | 1 | **1** | **正溢出**（正+正=负） |
| 1 | 0 | **1** | **负溢出**（负+负=正） |
| 1 | 1 | **0** | 无溢出 |

- 只有 $C_0 \neq C_1$ 时才溢出，异或门正好实现这个功能
- **异或门 = 判断两个输入是否不同**

因此溢出判断电路的核心是**异或门**。

**答案：C**

> B. 借位一定溢出
> C. 溢出一定借位
> D. 两者没有必然联系

**解析**：

- 表示数值"不够减"

- 判断依据是符号位进位⊕最高数位进位

```

所以两者**没有必然联系**。

### Q31 结构体对齐与小端

> **题目**：在32位小端机中，有以下结构体：
> ```c
> struct {
>     char  a;    // 1字节
>     short b;    // 2字节
>     int   c;    // 4字节
> } s = {0x12, 0x3456, 0x789ABCDE};
> ```
> 若结构体起始地址为0x1000，则地址0x1002中的内容是（ ）
> A. 0x12
> B. 0x34
> C. 0x56
> D. 0xCD

**解析**：

**第一步：分析结构体成员的内存布局（边界对齐）**

```
struct {
    char  a;   // 1字节，对齐系数1，偏移0
    short b;   // 2字节，对齐系数2，偏移必须是2的倍数
    int   c;   // 4字节，对齐系数4，偏移必须是4的倍数
};
```

内存布局：
```
地址    内容      说明
0x1000  0x12      a（char，1字节）
0x1001  [填充]    对齐填充（short b需要2字节对齐，地址0x1002才是2的倍数）
0x1002  0x34      b的高字节 ← 注意：小端模式下
0x1003  0x56      b的低字节
0x1004  0x78      c的最高字节（小端：低位在低地址）
0x1005  0x9A
0x1006  0xBC
0x1007  0xDE      c的最低字节
```

**第二步：理解小端存储**

小端（Little-Endian）：**低位字节在低地址**

对于 short b = 0x3456：
- 低字节 = 0x56 → 存在低地址 0x1003
- 高字节 = 0x34 → 存在高地址 0x1002

所以地址0x1002中存放的是b的高字节 = 0x34

**答案：B**


### Q32 IEEE754精确表示

> **题目**：以下哪个十进制数可以用IEEE754单精度浮点数精确表示（ ）

> C. 1.3

- 01：正确符号位0，但出现了进位→**正溢出**
- 10：正确符号位1，但出现了进位→**负溢出**

**答案：B**


#### Q179 符号位扩展

> **题目**：将8位补码 10110011 扩展为16位，结果是（ ）
> A. 0000000010110011
> B. 1111111110110011
> C. 1111111110110010
> D. 0000000010110100

**解析**：

补码符号位扩展规则：
- **负数补码**：高位补1

8位补码 10110011，最高位1 → 负数

扩展为16位：高位补8个1

```
10110011 → 11111111 10110011
```

验证：这个数是否真的是负数？
- 8位补码 10110011，数值 = -(0100 1100 + 1) = -0100 1101 = -77
- 16位补码 11111111 10110011，数值 = -(00000000 01001100 + 1) = -00000000 01001101 = -77 ✅

**答案：B**


#### Q180 补码求负

> **题目**：已知 [X]补 = 11010011，则 [-X]补 为（ ）
> A. 00101101
> B. 00101100
> C. 11010011
> D. 10101101

**解析**：

由 [X]补 求 [-X]补 的规则：**连同符号位取反，末位加1**

```
[X]补 = 1101 0011
取反  = 0010 1100（连同符号位！）
加1   = 0010 1101
```

所以 [-X]补 = 0010 1101

**注意**：是"连同符号位取反"，不是"符号位不变"！这要和"补码转原码"区分开。

**对比记忆**：
- 补码求原码：符号位不变，数值位取反加1
- 补码求[-X]补：连同符号位取反，末位加1

**答案：A**


#### Q181 原码乘法符号处理

> **题目**：原码乘法中，乘积的符号位由两个乘数的符号位如何决定（ ）
> A. 相加
> B. 相乘
> C. 异或
> D. 同或

**解析**：

原码乘法规则：
1. **符号位单独处理**：乘积符号 = 乘数符号 ⊕ 被乘数符号（**异或**运算）
2. **数值部分**：绝对值相乘

为什么用异或？
- 正 × 正 = 正：0⊕0=0 ✅
- 正 × 负 = 负：0⊕1=1 ✅
- 负 × 正 = 负：1⊕0=1 ✅
- 负 × 负 = 正：1⊕1=0 ✅

异或正好实现"相同为正，不同为负"的符号规则。

**答案：C**


#### Q182 移位运算

> **题目**：补码的算术右移，高位补入的是（ ）
> A. 0
> B. 1
> C. 符号位
> D. 最低位

**解析**：

算术右移的规则：**高位补符号位**，低位丢失。

- 正数补码（符号位0）：高位补0 → 相当于除以2后向下取整
- 负数补码（符号位1）：高位补1 → 相当于除以2后向下取整

| 移位类型 | 规则 | 用途 |
|---------|------|------|
| 算术左移 | 低位补0，高位丢失（符号位变化则溢出） | 乘以2 |
| 算术右移 | 高位**补符号位**，低位丢失 | 除以2 |
| 逻辑右移 | 高位**补0**，低位丢失 | 无符号数移位 |

**例**：
```
[-66]补 = 1011 1110
算术右移1位：1101 1111（高位补1=符号位）
验证：-66/2 = -33，而 1101 1111 正是 [-33]补 ✅
```

**答案：C**


#### Q183 补码左移溢出判断

> **题目**：补码左移时，发生溢出的条件是（ ）
> A. 符号位变化
> B. 符号位与最高数位不同
> C. 最高位丢失
> D. 移出1

**解析**：

补码左移溢出的判断标准：**移位后符号位发生变化**。

具体来说：移位前符号位 ≠ 移位后符号位 → 溢出

或者等价地：**符号位与最高数位不同时，左移会溢出**。

```　　
例1：无溢出
   [66]补 = 0100 0010
   左移1位 = 1000 0100（此时符号位=1，与原来不同！）
   但仔细看：符号位从0变1，但这个变化是因为进位导致的
   实际上左移前符号位=0 ≠ 最高数位=1 → 左移后会溢出

   验证：66×2=132，超出8位补码范围-128~127 → 确实溢出
    
例2：无溢出
   [32]补 = 0010 0000
   左移前：符号位=0，最高数位=0（相同）
   左移后：0100 0000 = 64（32×2，在范围内）
   符号位不变，无溢出 ✅
```

**答案：A**（符号位变化）

更精确地说：**左移前，符号位 ≠ 最高数位 → 左移后符号位变化 → 溢出**

但题目中A选项"符号位变化"是最直接的判断标准。

**答案：A**


#### Q184 IEEE754规格化数

> **题目**：IEEE754单精度浮点数中，规格化数的阶码E的取值范围是（ ）
> A. 0 ~ 255
> B. 1 ~ 254
> C. 0 ~ 254
> D. 1 ~ 255

**解析**：

IEEE754中阶码E的区分：

| E的值 | 类型 |
|:----:|------|
| **0** | **0**或**非规格化数** |
| **1 ~ 254** | **规格化数** |
| **255** | **无穷大**或**NaN** |

规格化数的阶码范围：**1 ~ 254**

主要区别：
- E=0：隐含整数位为0，表示非常接近0的数或0
- E=255：表示无穷大或NaN（非法运算结果）
- 1 ≤ E ≤ 254：隐含整数位为1，正常规格化浮点数

**答案：B**


#### Q185 IEEE754真值计算

> **题目**：IEEE754单精度浮点数 0x40400000 的十进制真值是（ ）
> A. 3.0
> B. 4.0
> C. 2.5
> D. 3.5

**解析**：

0x40400000 = 0100 0000 0100 0000 0000 0000 0000 0000

分解：
```
S = 0（正数）
E = 10000000 = 128
M = 10000000000000000000000
```

真值 = $(-1)^0 \times 1.1 \times 2^{128-127} = 1.1 \times 2^1 = 11.0(B) = 3.0$

**关键步骤**：
- E = 128，减去偏置值127，指数 = 1
- 尾数 M = 100...0，实际尾数 = 1.1（二进制）
- 1.1(B) = 1 + 0.5 = 1.5
- 1.5 × 2 = 3.0

**答案：A**

#### Q188 浮点数表示范围

> **题目**：关于浮点数表示范围的说法，正确的是（ ）
> A. 阶码位数越多，精度越高
> B. 尾数位数越多，表示范围越大
> C. 阶码位数越多，表示范围越大
> D. 尾数位数越多，表示范围越大

**解析**：

- **阶码**：决定**表示范围**（指数大小）

- A ❌ 阶码位数影响范围，不是精度

**记忆口诀**："阶码扩范围，尾数提精度"。

**答案：C**

#### Q191 小端存储

> **题目**：小端存储方式中，32位整数 0x12345678 在内存中从低地址到高地址的字节顺序是（ ）
> A. 12 34 56 78
> B. 78 56 34 12
> C. 78 34 56 12
> D. 12 78 56 34

**解析**：

小端（Little-Endian）：**低位字节在低地址**。

0x12345678 的字节分解：
- 最高字节（Byte3）= 0x12
- 次高字节（Byte2）= 0x34
- 次低字节（Byte1）= 0x56
- 最低字节（Byte0）= 0x78

小端存储（从低地址到高地址）：
```
地址 n+0: 78（最低字节）
地址 n+1: 56
地址 n+2: 34
地址 n+3: 12（最高字节）
```

所以顺序是：**78 56 34 12**

**对比记忆**：
- **大端**（Big-Endian）：12 34 56 78（和书写习惯相同，高字节在低地址）
- **小端**（Little-Endian）：78 56 34 12（低字节在低地址）

**答案：B**

# 第3章 运算方法和运算电路

> 对应题库：Q192 ~ Q216


## 3.1 ALU（算术逻辑单元）

### 3.1.1 电路类型

**ALU属于组合逻辑电路**（Combinational Logic Circuit），**不是**时序逻辑电路。

- **组合逻辑电路**：输出只取决于当前输入，与历史状态无关，无存储功能
- **时序逻辑电路**：输出不仅取决于当前输入，还与历史状态有关，有存储功能（如触发器、寄存器）

### 3.1.2 ALU的功能

ALU的核心功能分为两大类：

| 运算类型 | 具体操作 |
|---------|---------|
| **算术运算** | 加、减、乘、除、加1、减1、取负 |
| **逻辑运算** | 与、或、非、异或、同或 |

### 3.1.3 运算器的组成

一个完整的运算器包含：

| 包含的部件 | 说明 |
|-----------|------|
| **ALU** | 核心运算部件 |
| **状态寄存器** | 存放运算结果的状态（零标志ZF、符号标志SF、溢出标志OF、进位标志CF等） |
| **数据总线** | 在运算器各部件之间传输数据 |
| **通用寄存器** | 存放操作数和中间结果 |

**不包含**的部件：
- **地址寄存器（MAR）** -- 属于存储系统，不在运算器中
- **指令寄存器（IR）** -- 属于控制器
- **程序计数器（PC）** -- 属于控制器

### 3.1.4 ALU的电路结构

最基本的ALU由以下电路构成：
- **加法器**（全加器FA）-- 核心
- **逻辑运算电路**（与门、或门、非门、异或门）
- **多路选择器（MUX）** -- 选择执行哪种运算
- **控制信号输入** -- 决定ALU执行何种操作


## 3.2 移位运算（必考！）

### 3.2.1 算术移位

算术移位用于**有符号数**的运算，符号位保持不变。

| 移位方向 | 规则 | 低位/高位处理 |
|---------|------|------------|
| **算术左移** | 低位补0，高位丢弃 | 符号位变化 $\Rightarrow$ **溢出** |
| **算术右移** | 高位**补符号位**，低位丢弃 | 正数补0，负数补1 |

**算术右移示例**：
- 正数：$[+66]_{补} = 0100\ 0010$ 右移1位 $\rightarrow 0010\ 0001 = +33$（相当于除以2）
- 负数：$[-66]_{补} = 1011\ 1110$ 右移1位 $\rightarrow 1101\ 1111 = -33$（相当于除以2）

### 3.2.2 逻辑移位

逻辑移位用于**无符号数**，所有位（包括符号位）一起移动。

| 移位方向 | 规则 |
|---------|------|
| **逻辑左移** | 高位丢弃，低位**补0** |
| **逻辑右移** | 高位**补0**，低位丢弃 |

**逻辑右移示例**：
- $1010\ 1010$ 逻辑右移1位 $\rightarrow 0101\ 0101$（高位补0）

### 3.2.3 算术左移溢出判断

**核心规则**：左移前后符号位不同 $\Rightarrow$ **溢出**

或者等价地：

**补码左移溢出条件**：移出的高位 $\neq$ 新符号位 $\Rightarrow$ **溢出**

**示例**：
- $[+66]_{补} = 0100\ 0010$ 左移1位 $\rightarrow 1000\ 0100 = -124$
  - 原符号位=0，新符号位=1 $\Rightarrow$ **正溢出**（正数变负数）
- $[-66]_{补} = 1011\ 1110$ 左移1位 $\rightarrow 0111\ 1100 = +124$
  - 原符号位=1，新符号位=0 $\Rightarrow$ **负溢出**（负数变正数）

### 3.2.4 移位运算对比表

| 类型 | 左移低位补 | 右移高位补 |
|------|-----------|-----------|
| 算术左移 | 0 | -- |
| 算术右移 | -- | **符号位** |
| 逻辑左移 | 0 | -- |
| 逻辑右移 | -- | **0** |


## 3.3 溢出判断（必考！）

### 3.3.1 溢出的根本原因

**寄存器的位数有限**，无法表示超出表示范围的数据。

例如，8位补码表示范围：$-128 \sim +127$。计算结果超出该范围 $\Rightarrow$ **溢出**。

### 3.3.2 双符号位法（模4补码）

运算时使用两个符号位 $S_1S_2$，存储时只保留一个。

| 双符号位 $S_1S_2$ | 含义 |
|-----------------|------|
| **00** | 正数，无溢出 |
| **01** | **正溢出**（上溢） |
| **10** | **负溢出**（下溢） |
| **11** | 负数，无溢出 |

**判断方法**：
- 两个符号位相同 $\rightarrow$ 无溢出
- 两个符号位不同 $\rightarrow$ **溢出**
  - $S_1S_2 = 01$：正溢出（结果大于最大正数）
  - $S_1S_2 = 10$：负溢出（结果小于最小负数）

### 3.3.3 进位判断法

**公式**：
$$\text{溢出} = C_0 \oplus C_1$$

其中：
- $C_0$：符号位的进位（最高位进位）
- $C_1$：最高数值位的进位（次高位进位）

**规则**：$C_0$ 和 $C_1$ 异或结果为1 $\Rightarrow$ **溢出**

| $C_0$（符号位进位） | $C_1$（数值最高位进位） | $C_0 \oplus C_1$ | 结果 |
|-----------------|--------------------|-----------------|------|
| 0 | 0 | 0 | 无溢出 |
| 0 | 1 | **1** | **溢出** |
| 1 | 0 | **1** | **溢出** |
| 1 | 1 | 0 | 无溢出 |

### 3.3.4 硬件实现

溢出判断电路用**异或门**实现：将符号位进位 $C_0$ 和最高数值位进位 $C_1$ 输入异或门，输出即为溢出标志 OF。

### 3.3.5 三种溢出判断方法对比

| 方法 | 原理 | 实现难度 |
|------|------|---------|
| 双符号位法 | 两个符号位是否相同 | 需要2位符号位 |
| 进位判断法 | $C_0 \oplus C_1$ | 简单，常用 |
| 硬件异或门 | 异或门电路 | 最简单 |


## 3.4 原码乘法

### 3.4.1 基本原理

原码乘法的核心思想：

1. **符号位单独处理**：乘积符号 = 乘数符号 $\oplus$ 被乘数符号（异或运算）
2. **绝对值相乘**：对绝对值部分进行无符号数乘法

### 3.4.2 运算步骤

1. 提取符号位：$X_s$ 和 $Y_s$
2. 乘积符号：$P_s = X_s \oplus Y_s$（同号为正，异号为负）
3. 取绝对值：$|X|$ 和 $|Y|$
4. 绝对值相乘：得到乘积的绝对值
5. 加上符号位：$P = P_s \cdot |X \times Y|$

### 3.4.3 示例

**例**：$X = -3, Y = +2$（4位原码）

1. $[X]_{原} = 1011, [Y]_{原} = 0010$
2. $X_s = 1, Y_s = 0$
3. $P_s = 1 \oplus 0 = 1$（结果为负）
4. $|X| = 011, |Y| = 010$
5. $011 \times 010 = 0110 = 110$
6. 结果：$1110 = -6$

### 3.4.4 硬件构成

原码乘法器由以下部件构成：
- **符号处理电路**：异或门
- **绝对值乘法器**：无符号乘法阵列
- **加法器**：累加部分积


## 3.5 补码加减法

### 3.5.1 补码加法

$$[X+Y]_{补} = [X]_{补} + [Y]_{补} \pmod{2^n}$$

补码加法运算规则：
1. 符号位参与运算
2. 按模 $2^n$ 相加（丢弃最高位进位）

### 3.5.2 补码减法

**核心公式**：
$$[X-Y]_{补} = [X]_{补} + [-Y]_{补}$$

其中：
$$[-Y]_{补} = [Y]_{补}\text{连同符号位取反，末位加1}$$

### 3.5.3 减法器实现

硬件通过以下方式将加法器改造为加减法器：

- 控制信号 **Sub**
  - Sub = 0：执行加法，Y直接进入加法器
  - Sub = 1：执行减法，Y取反，且低位进位 $C_{in}=1$

**电路原理**：
1. Y的每一位与Sub异或（Sub=1时Y取反）
2. Sub同时作为加法器的低位进位输入 $C_{in}$
3. 实现：$X + (\overline{Y} + 1) = X - Y$

**减法器输入信号**：
- $X$（被减数）
- $Y$（减数）
- **Sub**（控制信号，Sub=1表示做减法）

### 3.5.4 补码运算示例

**例1**：$X = -68, Y = -80$，求 $X+Y$ 和 $X-Y$

1. $[X]_{补} = BCH,\ [Y]_{补} = B0H$
2. $X+Y$：$BCH + B0H = 6CH$（无符号结果，但溢出标志OF=1，因为正+正=负）
3. $[-Y]_{补} = 50H$（B0H取反得4FH，+1得50H）
4. $X-Y$：$BCH + 50H = 0CH$（OF=0）

**例2**：$X = FFFF FFDFH, Y = 0000 0041H$
1. $X = -33, Y = 65$
2. $X-Y$：$FFFF FFDFH + FFFF FFBFH = FFFF FF9EH$（其中 $[-Y]_{补} = FFFF FFBFH$）

     - OF = 1 $\oplus$ 0 = 1，**溢出** ✓
   - 双符号位验证：$11\ 00\ 1010 + 11\ 01\ 1010 = 10\ 10\ 0100$，符号位10 $\Rightarrow$ **负溢出** ✓

4. 结论：
   - $X+Y$：无溢出，结果为 +32
   - $X-Y$：**负溢出**，结果超出范围

**答案**：
1. $X=-54, Y=+86$
2. $X+Y=0010\ 0000=+32$，无溢出
3. $X-Y$ 结果溢出（负溢出），正确结果应为 $-140$（超出8位补码表示范围）

## 4.1 存储器分类

### 按存取方式分类

| 类型 | 全称 | 特点 | 典型器件 | 能否随机访问 |
|------|------|------|----------|:----------:|
| **RAM** | 随机存取存储器 | 可读可写, 断电丢失 | SRAM, DRAM | 能 |
| **ROM** | 只读存储器 | 断电不丢失 | EPROM, Flash | 能 |
| **SAM** | 顺序存取存储器 | 按顺序访问 | 磁带 | 不能 |
| **DAM** | 直接存取存储器 | 先直接定位再顺序访问 | **磁盘** | 不能 |
| **相联存储器** | Content-Addressable Memory (CAM) | **按内容寻址** | TLB, 相联Cache | 能 |

### 4.1.1 RAM（随机存取存储器）

**随机存取**：存取时间与存储单元物理位置无关，任何单元访问时间相同。

#### SRAM（静态RAM）

| 项目 | 说明 |
|------|------|
| 存储元件 | 触发器（6个晶体管） |
| 特点 | 速度快, 不需要刷新 |
| 用途 | **Cache**（高速缓存） |
| 地址线 | 不采用地址复用 → 地址线数 = log2(存储单元数) |
| 集成度 | 低（晶体管多） |
| 功耗 | 较大 |
| 价格 | 贵 |

#### DRAM（动态RAM）

| 项目 | 说明 |
|------|------|
| 存储元件 | 电容（1个晶体管+1个电容） |
| 特点 | **需要定期刷新**（电容会漏电） |
| 用途 | **主存** |
| 地址线 | **采用地址复用技术** → 地址线数 = log2(存储单元数) / 2 |
| 集成度 | 高（晶体管少） |
| 功耗 | 较小 |
| 价格 | 便宜 |

> **核心区别**：DRAM需要刷新、采用地址复用技术；SRAM不需要刷新、不采用地址复用。

### 4.1.2 ROM（只读存储器）

| 类型 | 说明 |
|------|------|
| **MROM** (掩模ROM) | 出厂时写入, 不可更改 |
| **PROM** (可编程ROM) | 用户一次性写入, 不可擦除 |
| **EPROM** (可擦除可编程ROM) | 紫外线擦除, 可多次编程 |
| **EEPROM** (电可擦除ROM) | 电擦除, 可逐字节修改 |
| **Flash** (闪存) | 电擦除, 块为单位擦写, U盘/SSD使用 |

### 4.1.3 SAM（顺序存取存储器）

- 数据按顺序存储和读取
- 存取时间取决于数据位置
- 典型：**磁带**
- 特点：容量大、速度慢、成本低

### 4.1.4 DAM（直接存取存储器）

- 先直接定位到目标区域，再顺序存取
- 典型：**磁盘（硬盘）**
- 存取时间 = 寻道时间 + 旋转延迟 + 传输时间

### 4.1.5 相联存储器（按内容寻址）

- 不按地址访问，而是按**内容（关键字）** 并行查找
- 典型应用：
  - **TLB**（快表，页表缓存）
  - **相联Cache**（全相联映射方式的Cache）
- 特点：速度快但硬件成本高


## 4.2 存储器层次结构

### 层次结构图

```
         CPU寄存器  ← 速度快、容量小、价格高
            ↑
         Cache (SRAM)
            ↑
         主存 (DRAM)
            ↑
         辅存 (磁盘/SSD)  ← 速度慢、容量大、价格低
```

### CPU访问权限

| 存储层次 | CPU能否直接访问 |
|:--------:|:--------------:|
| 寄存器 | **能** |
| Cache | **能** |
| 主存 | **能** |
| 硬盘（辅存） | **不能**（必须先调入主存） |

> **关键结论**：CPU只能直接访问**寄存器、Cache、主存**。访问磁盘需要先将数据调入主存。

### 存取周期（重要概念！）

| 概念 | 定义 |
|------|------|
| **存取时间** (Access Time) | 一次读或写操作所需的平均时间 |
| **存取周期** (Memory Cycle Time) | **连续两次独立读/写操作之间的最短时间间隔** |
| 关系 | 存取周期 **≥** 存取时间（因为需要恢复时间） |

> **考试陷阱**：存取周期 ≠ 存取时间！存取周期 = 存取时间 + 恢复时间。存取周期是**连续两次访问的最小时间间隔**。

### 存储系统总容量

- 存储系统总容量 = **主存容量**（不是各层容量简单相加）
- Cache中存放的是主存中某部分信息的**副本**


## 4.3 半导体存储器计算（必考！）

### 4.3.1 基本公式

**存储芯片容量表示**：`M × N` 位
- M = 存储单元数（地址范围）
- N = 每个存储单元的位数（数据宽度）

**地址线数**：
- SRAM: `地址线数 = log2(M)`（不采用地址复用）
- DRAM: `地址线数 = log2(M) / 2`（采用地址复用，行/列地址分时传送）

**数据线数**：
- `数据线数 = N`（每个单元有多少位，数据线就有多少根）

**总引脚数（仅地址+数据）**：
- SRAM: `log2(M) + N`
- DRAM: `log2(M)/2 + N`

### 4.3.2 SRAM vs DRAM 引脚对比

**例题：1024×8位的芯片**

| 类型 | 地址线 | 数据线 | 地址+数据引脚数 |
|:---:|:------:|:------:|:--------------:|
| SRAM | log2(1024)=**10** | **8** | **18** |
| DRAM | log2(1024)/2=**5** | **8** | **13** |

**例题：4M×8位的芯片**
- 4M = 4 × 2^20 = 2^22

| 类型 | 地址线 | 数据线 | 地址+数据引脚数 |
|:---:|:------:|:------:|:--------------:|
| SRAM | log2(4M)=**22** | **8** | **30** |
| DRAM | log2(4M)/2=**11** | **8** | **19** |

> **DRAM地址复用规律**：每增加1根地址线，容量提高到**4倍**（因为行地址和列地址各增加1位 → 2×2=4倍）。

### 4.3.3 芯片扩展

#### 位扩展（增加数据线宽度）

- 多个芯片的地址线**并联**（全部连在一起）
- 数据线**拼接**（各芯片的数据线分别连到CPU数据线的不同位）
- 片选信号**连在一起**（同时选中所有芯片）

**例题**：用 1024×1位的芯片组成 1024×8位的存储器
- 需要芯片数 = 8/1 = **8片**
- 地址线：所有芯片共用 log2(1024) = 10根地址线
- 数据线：每片1根数据线，共8根

#### 字扩展（增加地址范围/存储单元数）

- 多个芯片的地址线和数据线**并联**
- 片选信号**分开**（通过高位地址译码来片选）
- 每次只选中一个芯片

**例题**：用 1024×8位的芯片组成 4K×8位的存储器
- 需要芯片数 = 4K/1024 = **4片**
- 每片地址范围：1024个单元
- 需要高位地址线做片选译码

#### 字位同时扩展

**例题**：用 1024×1位的芯片组成 4K×8位的存储器
- 总位数 = 4K × 8 = 32K位
- 每片位数 = 1024 × 1 = 1024位
- 需要芯片数 = (4K×8) / (1024×1) = 32K位/1024位 = **32片**
  - 或：字扩展 4倍 × 位扩展 8倍 = 32片

### 4.3.4 编址方式

| 编址方式 | 说明 | 举例 |
|:--------:|------|------|
| **按字节编址** | 每个字节（8位）有一个地址 | 最常见 |
| **按字编址** | 每个字（字长）有一个地址 | 字长=32位时, 每个地址对应4个字节 |

> **注意**：按字编址时，相邻地址之间的差值为**1个字**而非1个字节。同样容量的内存，按字编址的地址范围小于按字节编址。

**例题**：内存容量 4GB，字长32位
- 按字节编址：地址范围 = 4G = 2^32，需要32位地址
- 按字编址：每个字4字节，地址范围 = 1G = 2^30，需要30位地址


## 4.4 主存与CPU连接

### 4.4.1 地址线分配

主存地址通常分为两部分：
- **高位地址** → 片选信号（选择哪个芯片）
- **低位地址** → 芯片内部地址（选择芯片内哪个单元）

```
高位地址 ──→ 译码器 ──→ 片选信号
低位地址 ──→ 直接连到芯片地址引脚
```

### 4.4.2 MAR（存储器地址寄存器）

- MAR的位数决定了CPU能访问的**最大地址空间**
- MAR位数 = 寻址范围对应的位数

**例题**：MAR为16位，按字节编址
- 寻址范围 = 2^16 = 64K 字节

### 4.4.3 存储芯片数量计算

**基本公式**：
```
芯片数量 = (目标总容量) / (单芯片容量)
         = (目标存储单元数 × 目标数据宽度) / (单芯片存储单元数 × 单芯片数据宽度)
```

**例题**：用 64K×1位的SRAM芯片组成 256K×16位的存储器
- 字扩展倍数 = 256K/64K = 4
- 位扩展倍数 = 16/1 = 16
- 总芯片数 = 4 × 16 = **64片**

**例题**：用 1M×4位的DRAM芯片组成 4M×32位的存储器
- 字扩展倍数 = 4M/1M = 4
- 位扩展倍数 = 32/4 = 8
- 总芯片数 = 4 × 8 = **32片**

### 4.4.4 片选信号

| 片选方式 | 说明 | 优点 | 缺点 |
|:--------:|------|:----:|:----:|
| 线选法 | 用一根地址线选一个芯片 | 简单 | 地址空间不连续 |
| 全译码 | 所有地址线参与译码 | 地址连续且唯一 | 需要译码器 |
| 部分译码 | 部分地址线不参与译码 | 简单 | 地址重叠 |


## 4.5 Cache（必考！）

### 4.5.1 Cache 基本概念

- **功能完全由硬件实现**（对程序员和操作系统透明）
- Cache和主存以**块**为单位交换数据
- Cache中存放的是主存中某部分信息的**副本**
- CPU按**主存物理地址**访问

> **核心原则**：Cache的一切操作都由**硬件**自动完成。

### 4.5.2 地址映射方式

#### 直接映射

| 项目 | 说明 |
|------|------|
| 映射规则 | 每个主存块只能映射到Cache中**唯一**的一行 |
| 地址结构 | Tag + Index（行号）+ Offset（块内偏移） |
| 优点 | 实现简单, 查找速度快 |
| 缺点 | **冲突率高**（多个主存块竞争同一Cache行） |
| 替换时 | 无需选择, 直接覆盖 |

**地址划分**：
```
主存地址 = [Tag] [Index] [Offset]
```

**例题**：32位地址，32KB Cache，32B块，直接映射
- 块内偏移：log2(32B) = **5位**
- Cache行数：32KB / 32B = 1024行 → Index = log2(1024) = **10位**
- Tag：32 - 10 - 5 = **17位**

#### 全相联映射

| 项目 | 说明 |
|:----:|------|
| 映射规则 | 每个主存块可映射到Cache**任意**一行 |
| 地址结构 | Tag + Offset（无Index） |
| 优点 | **冲突率最低**, Cache利用率高 |
| 缺点 | 比较器硬件成本高, 查找速度慢 |

**地址划分**：
```
主存地址 = [Tag] [Offset]
```

#### 组相联映射（直接映射+全相联的折中）

| 项目 | 说明 |
|:----:|------|
| 映射规则 | 主存块映射到**某一组**中的**任意**一行 |
| 地址结构 | Tag + Group Index + Offset |
| 优点 | 冲突率适中, 硬件成本适中 |
| 常用 | **n路组相联**（n通常为2/4/8） |

**地址划分**：
```
主存地址 = [Tag] [Group Index] [Offset]
```

**组数计算**：
```
组数 = Cache行数 / 路数
Index位数 = log2(组数)
Tag位数 = 总地址位数 - Index位数 - Offset位数
```

**例题**：32位地址，32KB Cache，32B块，**四路组相联**
- 块内偏移：log2(32) = **5位**
- Cache行数：32KB/32B = 1024行
- 组数：1024/4 = 256组 → Index = log2(256) = **8位**
- Tag：32 - 8 - 5 = **19位**

### 4.5.3 替换算法

| 算法 | 说明 | 与局部性的关系 |
|:----:|------|:-------------:|
| **RAND**（随机） | 随机选择一行替换 | 无关系 |
| **FIFO**（先进先出） | 替换最早调入的行 | 无关 |
| **LRU**（近期最少使用） | 替换最近最久未使用的行 | **与局部性原理有关** ✅ |

> **LRU与局部性原理**：LRU利用了**时间局部性**——最近被访问过的块很可能在不久的将来再次被访问，所以保留近期访问过的块，替换最久未访问的块。

### 4.5.4 写策略

#### 命中时的处理

| 策略 | 做法 | 特点 |
|:----:|------|:----:|
| **全写法** (Write Through) | 同时写入Cache和主存 | 数据一致性高, 安全性好, 速度慢 |
| **回写法** (Write Back) | 只写Cache, 标记脏位 | 速度快, 换出时才写回主存 |

#### 不命中时的处理

| 策略 | 做法 |
|:----:|------|
| **写分配法** (Write Allocate) | 先将块调入Cache，再更新Cache |
| **非写分配法** (No-Write Allocate) | 直接写主存，不调入Cache |

#### 常用搭配

| 组合 | 适用场景 |
|:----:|:--------:|
| **回写法 + 写分配法** | 写密集型应用（速度快） |
| **全写法 + 非写分配法** | 安全性要求高的应用 |

### 4.5.5 Cache行结构

每行Cache包含以下字段：

| 字段 | 位宽 | 说明 |
|:----:|:----:|------|
| **有效位** (Valid) | 1位 | 该行数据是否有效（初始为0） |
| **标记位** (Tag) | Tag位数 | 用于比较判断是否命中 |
| **脏位** (Dirty) | 1位 | **回写法**使用，表示数据已被修改未写回主存 |
| **LRU位** | 取决于路数 | LRU算法使用，记录使用顺序 |
| **数据块** | 块大小×8位 | 存放从主存调入的数据 |

### 4.5.6 Cache总位数计算（必考！）

**公式**：
```
Cache总位数 = Cache行数 × (数据位 + 标记位 + 有效位 + 脏位)
```

> **注意**：数据位 = 块大小 × 8（因为1字节=8位），不要忘记乘以8！

**例题**：32位地址，32KB Cache，32B块，**直接映射，回写法**
- 块内偏移：log2(32) = **5位**
- Cache行数：32KB/32B = 1024行 → Index = log2(1024) = **10位**
- Tag：32 - 10 - 5 = **17位**
- 每行数据位：32B × 8 = **256位**
- 每行总位数：256（数据）+ 17（Tag）+ 1（有效位）+ 1（脏位）= **275位**
- Cache总位数：1024 × 275 = **281,600位**

**扩展：四路组相联+LRU**
- 四路组相联：每组4行
- LRU位：log2(4) = 2位（记录4行中的使用顺序）
- 每行总位数：256（数据）+ Tag + 1（有效）+ 1（脏位）+ 2（LRU位）
- **LRU位数 = log2(路数)**

### 4.5.7 平均存取时间（必考！）

**公式**：
```
T_avg = H × T_cache + (1-H) × (T_cache + T_mem)
```

其中：
- H = 命中率 (Hit Rate)
- T_cache = Cache访问时间
- T_mem = 主存访问时间
- 缺失时间 = T_cache + T_mem（先查Cache未命中，再访问主存）

**更通用的形式**：
```
T_avg = H × T_cache + (1-H) × T_miss
```

**例题**：Cache访问时间2ns，主存访问时间40ns，要求平均访问时间不超过3ns，求最低命中率。
- 设命中率为 h
- T_avg = h × 2 + (1-h) × (2+40) = 2h + 42(1-h) = 42 - 40h
- 42 - 40h ≤ 3
- 40h ≥ 39
- h ≥ 0.975 = **97.5%**

> **结论**：Cache命中率必须达到97.5%以上才能满足性能要求。

### 4.5.8 Cache与主存的数据交换宽度

- **Cache数据宽度** = Cache与主存一次交换的数据量
- 通常等于**一个Cache块的大小**
- 也等于**主存一次读/写的位数**


## 例题精讲

### Q33 边界对齐

**题目**：关于边界对齐的叙述，正确的是？

**解析**：
边界对齐是指数据在内存中的起始地址必须是其自身大小的整数倍。
- 如：int（4字节）的起始地址必须是4的倍数
- short（2字节）的起始地址必须是2的倍数
- 对齐可以提高存取效率，但可能会浪费少量空间
- 不对齐则可能一次访存无法读完数据

**答案要点**：按边界对齐存储时，可能浪费存储空间但提高存取速度。


### Q34 磁盘类型

**题目**：磁盘属于哪种存取类型的存储器？

**解析**：
磁盘是**直接存取（DAM）** 存储器。
- 先通过磁头直接定位到目标**磁道/扇区**（直接定位）
- 再在扇区内顺序读写（顺序访问）
- 既不是纯粹的随机存取，也不是纯粹的顺序存取

**答案**：DAM（直接存取存储器）


### Q35 存取周期

**题目**：关于存取周期的说法，正确的是？

**解析**：
存取周期是**连续两次独立读/写操作之间的最短时间间隔**。
- 存取时间 ≤ 存取周期
- 存取周期 = 存取时间 + 恢复时间

> **常见错误**：误以为存取周期 = 存取时间。存取周期必须考虑线路恢复时间，所以 > 存取时间。

**答案**：存取周期 ≥ 存取时间


### Q36 相联存储器

**题目**：相联存储器的寻址方式是？

**解析**：
相联存储器（CAM）按**内容（关键字）** 进行寻址，而不是按地址寻址。
- 将数据内容作为关键字进行并行比较
- 典型应用：TLB（快表）、相联Cache

**答案**：按内容寻址


### Q37 CPU不能直接访问

**题目**：CPU不能直接访问下列哪个存储器？

A. 寄存器 B. Cache C. 主存 D. 硬盘

**解析**：
CPU可以直接访问寄存器、Cache和主存。硬盘属于辅存，CPU不能直接访问，必须先将其数据调入主存后才能访问。

**答案**：D. 硬盘


### Q38 数据传输速率

**题目**：某磁盘转速为7200RPM，平均寻道时间为8ms，每个磁道有1000个扇区，每个扇区512字节，求数据传输速率。

**解析**：
- 旋转一圈时间 = 60/7200 ≈ 8.33ms
- 平均旋转延迟 = 8.33/2 ≈ 4.17ms
- 平均存取时间 = 寻道时间 + 旋转延迟 = 8 + 4.17 = 12.17ms
- 每圈可读数据 = 1000 × 512 = 512,000 字节
- 数据传输速率 = 512,000 / (60/7200) = 512,000 × 120 = 61,440,000 字节/s ≈ **61.44 MB/s**

**答案**：约 61.44 MB/s


### Q39 字编址寻址单元

**题目**：某计算机字长32位，内存容量4GB，按字编址，寻址范围是多少？

**解析**：
- 按字编址：每个地址对应一个字（32位=4字节）
- 总字数 = 4GB / 4B = 1G = 2^30
- 寻址范围 = 0 ~ 2^30 - 1 = 1G
- 地址线需要 30 根

**对比**：如果按字节编址，寻址范围是 0 ~ 2^32 - 1 = 4G，需要32根地址线。

**答案**：1G（2^30）


### Q40 芯片引脚计算（SRAM）

**题目**：一个 1024×8 位的 SRAM芯片，地址线和数据线引脚共多少根？

**解析**：
- SRAM不采用地址复用
- 地址线数 = log2(1024) = **10根**
- 数据线数 = **8根**
- 总引脚数 = 10 + 8 = **18根**

**答案**：18根


### Q41 芯片引脚计算（DRAM）

**题目**：一个 1024×8 位的 DRAM芯片，地址线和数据线引脚共多少根？

**解析**：
- DRAM采用地址复用技术（行地址和列地址分时传送）
- 地址线数 = log2(1024) / 2 = 10/2 = **5根**
- 数据线数 = **8根**
- 总引脚数 = 5 + 8 = **13根**

**对比**：同样容量，DRAM比SRAM少了5根地址引脚。

**答案**：13根


### Q42 芯片容量计算

**题目**：某DRAM芯片引脚中地址线11根，数据线8根，该芯片容量为？

**解析**：
- DRAM采用地址复用，实际地址位数 = 11 × 2 = 22位
- 存储单元数 = 2^22 = 4M
- 数据线8根 = 每个单元8位
- 芯片容量 = 4M × 8位

**答案**：4M × 8位


### Q43 芯片总引脚

**题目**：4M×8位的DRAM芯片，地址线和数据线共多少根引脚？

**解析**：
- 4M = 4 × 2^20 = 2^22
- 地址线（DRAM复用后）= 22/2 = **11根**
- 数据线 = **8根**
- 总引脚 = 11 + 8 = **19根**

**答案**：19根


### Q44 SRAM vs DRAM 引脚对比

**题目**：同样为 1M×8位的芯片，SRAM与DRAM的地址+数据引脚数相差多少？

**解析**：
- 1M = 2^20
- SRAM地址线：20根，数据线：8根 → 共28根
- DRAM地址线：10根，数据线：8根 → 共18根
- 相差：28 - 18 = **10根**

**答案**：10根（DRAM少10根）


### Q45 芯片扩展-位扩展

**题目**：用 1024×1位的DRAM芯片组成 1024×8位的存储器，需要多少片？

**解析**：
- 位扩展：数据线宽度从1位扩展到8位
- 需要芯片数 = 8 / 1 = **8片**
- 所有芯片地址线并联，片选信号连在一起

**答案**：8片


### Q46 芯片扩展-字扩展

**题目**：用 1024×8位的SRAM芯片组成 4K×8位的存储器，需要多少片？

**解析**：
- 字扩展：增加地址范围
- 4K / 1024 = 4倍
- 需要芯片数 = **4片**
- 每片地址线相同，通过高位地址译码产生不同片选信号

**答案**：4片


### Q47 字位同时扩展

**题目**：用 64K×1位的SRAM芯片组成 256K×16位的存储器，需要多少片？

**解析**：
- 字扩展：256K/64K = **4倍**
- 位扩展：16/1 = **16倍**
- 总芯片数 = 4 × 16 = **64片**

**答案**：64片


### Q48 扩展芯片数综合

**题目**：用 1M×4位的DRAM芯片组成 4M×32位的存储器，共需多少片？

**解析**：
- 总位数：4M × 32 = 128M 位
- 每片位数：1M × 4 = 4M 位
- 芯片数 = 128M / 4M = **32片**
- 或：字扩展 4倍 × 位扩展 8倍 = 32片

**答案**：32片


### Q49 磁盘组成

**题目**：关于磁盘的叙述，错误的是？

**解析**：
磁盘由盘片、磁头、主轴电机、磁头臂等组成。
- 数据存储在同心圆的磁道上
- 每个磁道分为多个扇区
- 存取时间 = 寻道时间 + 旋转延迟 + 传输时间
- 常见转速：5400RPM, 7200RPM, 10000RPM, 15000RPM

**答案**：根据具体选项判断


### Q50 SSD vs 磁盘

**题目**：关于SSD和磁盘的说法，正确的是？

**解析**：
- SSD使用Flash芯片，无机械部件 → 速度快、抗震、功耗低
- 磁盘使用磁介质，有机械部件 → 容量大、成本低
- SSD没有寻道时间和旋转延迟
- SSD写入次数有限（磨损问题）

**答案**：根据具体选项判断


### Q51 存储系统容量

**题目**：某计算机的存储系统包括：寄存器32个，Cache 64KB，主存 512MB，硬盘 1TB。该存储系统总容量为？

**解析**：
- 存储系统总容量 = **主存容量**（这是重要考点）
- Cache中存放的是主存副本，不计入总容量
- 寄存器也不计入
- 磁盘是辅存，不计入
- 所以 = **512MB**

> **陷阱**：不要把所有层次容量相加！

**答案**：512MB


### Q52 Cache数据宽度

**题目**：Cache与主存一次交换的数据量称为？

**解析**：
Cache数据宽度（或称为Cache行大小、块大小）是指Cache与主存之间一次数据交换的位数。
- 通常等于一个Cache块的大小
- 也等于主存一次读/写的位数
- 过小：频繁调块，开销大
- 过大：空间局部性利用不充分

**答案**：Cache数据宽度 = 块大小


### Q53 Cache写策略

**题目**：同时写Cache和主存的策略称为？

**解析**：
- **全写法（Write Through）**：同时写入Cache和主存
- 优点：数据一致性好
- 缺点：写操作速度慢

**答案**：全写法（Write Through）


### Q54 写策略选择

**题目**：写密集型应用应选用哪种写策略？为什么？

**解析**：
- 应选用 **回写法（Write Back）**
- 原因：写操作只写Cache，标记脏位
- 减少了写主存的次数，提高写操作速度
- 通常在换出时才将脏块写回主存
- 搭配：回写法 + 写分配法

**答案**：回写法（Write Back），减少写主存次数，速度更快


### Q55 Cache硬件实现

**题目**：Cache的功能由什么实现？

**解析**：
Cache的所有功能（地址映射、查找、替换、写策略等）**全部由硬件实现**。
- 对程序员透明
- 对操作系统透明
- 用户无需任何干预

**答案**：全部由硬件实现


### Q56 LRU与局部性

**题目**：LRU替换算法利用了哪种局部性原理？

**解析**：
LRU（Least Recently Used）替换算法利用了**时间局部性**。
- 时间局部性：刚被访问的块在不久的将来很可能再次被访问
- LRU保留最近访问过的块，替换最久未访问的块
- 如果程序有良好的时间局部性，LRU命中率很高

> **扩展**：LRU在Cache组相联映射中通过LRU位实现。

**答案**：时间局部性


### Q57 CPU根据什么地址访问

**题目**：CPU访问Cache/主存时使用的是哪种地址？

**解析**：
CPU访问Cache和主存时使用**主存物理地址**。
- CPU发出的地址是主存物理地址
- 直接映射/组相联中，CPU用物理地址的Tag、Index、Offset字段访问Cache
- 虚拟存储系统中，CPU发出的是虚拟地址，由MMU转换为物理地址

**答案**：主存物理地址


### Q58 Cache命中率计算

**题目**：Cache的访问时间为2ns，主存访问时间为50ns，CPU执行一段程序共访问存储器10000次，其中访问Cache命中9800次，求：
(1) Cache命中率
(2) 平均存取时间
(3) 若将命中率提高至99%，平均存取时间变为多少？

**解析**：

**(1) 命中率**：
- H = 9800 / 10000 = **98%**

**(2) 平均存取时间**：
- T_avg = 0.98 × 2 + 0.02 × (2+50) = 1.96 + 1.04 = **3.0ns**

**(3) 命中率99%时**：
- T_avg = 0.99 × 2 + 0.01 × (2+50) = 1.98 + 0.52 = **2.5ns**

**答案**：(1) 98% (2) 3.0ns (3) 2.5ns


### Q59 Cache行位数

**题目**：某计算机32位地址，Cache容量64KB，块大小32B，采用直接映射和回写法，求Cache每行的位数和Cache总位数。

**解析**：

**(1) 计算各部分位数**：
- 块内偏移 = log2(32) = **5位**
- Cache行数 = 64KB/32B = 2048行 → Index = log2(2048) = **11位**
- Tag = 32 - 11 - 5 = **16位**

**(2) 每行位数**：
- 数据位：32B × 8 = **256位**
- Tag位：**16位**
- 有效位：**1位**
- 脏位（回写法）：**1位**
- 每行总位数 = 256 + 16 + 1 + 1 = **274位**

**(3) Cache总位数**：
- 总位数 = 2048 × 274 = **561,152位** = 68.6KB

**答案**：每行274位，总位数561,152位


### Q60 虚拟存储的基本概念

**题目**：虚拟存储器利用了什么原理？

**解析**：
虚拟存储器利用**局部性原理**（时间局部性+空间局部性）。
- 时间局部性：刚访问的程序和数据在不久的将来可能再次被访问
- 空间局部性：刚访问的地址附近的地址很可能也被访问
- 虚拟存储器将不常用的部分放在磁盘上，常用的部分调入主存
- 使得程序可以运行在比实际主存更大的地址空间中

**答案**：局部性原理


### Q61 局部性类型

**题目**：循环结构的程序主要利用了哪种局部性？

**解析**：
循环结构**同时利用了时间局部性和空间局部性**：
- **时间局部性**：循环体中的指令被反复执行
- **空间局部性**：循环体中的指令在内存中连续存储，顺序执行

**答案**：时间局部性和空间局部性


### Q62 虚拟存储器与Cache对比

**题目**：虚拟存储器和Cache的区别？

**解析**：

| 对比项 | Cache | 虚拟存储器 |
|:------:|:-----:|:----------:|
| 实现层次 | CPU-主存 | 主存-磁盘 |
| 管理方式 | **硬件**自动管理 | 硬件+**操作系统**共同管理 |
| 块大小 | 较小（数十字节） | 较大（数KB） |
| 不命中代价 | 较小（ns级） | 较大（ms级，缺页中断） |
| 替换算法 | LRU/FIFO/RAND | LRU近似 |
| 写策略 | 全写法/回写法 | 回写（先写主存） |

**答案**：见上表

| 公式 | 说明 |
|:----|:-----|
| 地址线数(SRAM) = log2(存储单元数) | SRAM不采用地址复用 |
| 地址线数(DRAM) = log2(存储单元数)/2 | DRAM采用地址复用 |

| T_avg = H×T_cache + (1-H)×(T_cache+T_mem) | 平均存取时间 |

## 常见陷阱提醒

1. **存取周期 ≠ 存取时间**：存取周期 = 存取时间 + 恢复时间
2. **DRAM地址线减半**：因为地址复用，行地址和列地址分时传送
3. **存储系统总容量 = 主存容量**：不是各层相加
4. **CPU不能直接访问硬盘**：必须先调入主存
5. **Cache功能全由硬件实现**：不是软件
6. **LRU与时间局部性有关**：FIFO和RAND无关
7. **全写法安全性高**：数据实时写回主存
8. **回写法速度快**：减少写主存次数
9. **Cache数据位 = 块大小×8**：别忘了乘以8得到位数
10. **按字编址时地址范围缩小**：因为每个地址对应多个字节

# 5.1 指令格式

## 5.1.1 指令的基本构成

每一条机器指令由两部分组成：

```
指令 = 操作码 + 地址码
```

| 字段 | 作用 | 说明 |
|------|------|------|
| **操作码 (OP)** | 指明指令要做什么操作 | 如：加法、减法、传送、转移 |
| **地址码 (A)** | 指明操作数在哪里 | 可以是数据本身、地址、寄存器编号等 |

**类比**：操作码是"动词"，地址码是"宾语"。
- `ADD R1, R2` → 操作码=ADD（加），地址码=R1（目标）, R2（源）


## 5.1.2 指令的分类（按地址码个数）

根据指令中包含的地址码个数，指令分为四类：

### (1) 零地址指令

```
格式：OP
```

- 指令中只有操作码，没有地址码
- 适用场景：
  - **空操作指令**（NOP）：什么都不做
  - **停机指令**（HLT）：停止CPU
  - **堆栈操作**：操作数隐含在栈顶（如 POP, PUSH 在某些架构中）

### (2) 一地址指令

```
格式：OP + A1
```

- 只有一个地址码
- 两种典型用法：
  - **单目运算**：`OP (A1) → A1` 如：取反 NEG R1、自增 INC R1
  - **双目运算（隐含约定）**：`(ACC) OP (A1) → ACC`，另一个操作数隐含在累加器ACC中

> **重要理解**：一地址指令也能处理双操作数运算，只是第二个操作数隐含在ACC中。

### (3) 二地址指令

```
格式：OP + A1 + A2
```

- 最常用的指令格式
- 操作：`(A1) OP (A2) → A1`（A1既是源也是目的）
- 例：`ADD R1, R2` → R1 = R1 + R2

### (4) 三地址指令

```
格式：OP + A1 + A2 + A3
```

- 三个地址码：两个源操作数 + 一个目的操作数
- 操作：`(A1) OP (A2) → A3`
- 例：`ADD R1, R2, R3` → R3 = R1 + R2
- 优点：源操作数不会被破坏
- 缺点：指令字长长，占用空间大

### 对比总结

| 类型 | 地址码数 | 操作数来源 | 结果存放 | 示例 |
|------|---------|-----------|---------|------|
| 零地址 | 0 | 隐含（栈顶） | 隐含 | NOP, HLT |
| 一地址 | 1 | A1 + (隐含ACC) | A1 或 ACC | INC R1 |
| 二地址 | 2 | A1 + A2 | A1（覆盖） | ADD R1, R2 |
| 三地址 | 3 | A1 + A2 | A3（保留源） | ADD R1, R2, R3 |

> **发展趋势**：RISC架构普遍采用**三地址指令**格式。


## 5.1.3 定长指令字 vs 变长指令字

### 定长指令字

- 所有指令的**长度相同**
- 优点：控制简单，取指时间固定，易于流水线
- 缺点：浪费空间（短指令也被分配长指令的空间）
- 典型：**RISC架构**普遍采用定长指令（如ARM 32位定长）

### 变长指令字

- 不同指令长度可以不同
- 优点：节省存储空间（短指令用短编码）
- 缺点：控制复杂，取指时间不固定
- 典型：**CISC架构**（x86指令长度1~15字节不等）

### 核心考点

> **指令字长 ≠ 机器字长 ≠ 存储字长**（三者可以相等，也可以不等）


## 5.1.4 操作码扩展技术

### 问题背景

指令字长有限，如果操作码固定为 n 位，最多只能表示 $2^n$ 种指令。但实际需要的指令条数往往超过 $2^n$，怎么办？

### 核心思想

**短操作码留出一些编码作为"扩展窗口"，用更长的操作码表示更多指令。**

### 工作原理图解

假设指令字长16位，操作码从4位开始：

```
第一层（4位操作码）：
0000 ~ 1110   → 15条三地址指令（操作码4位）
1111          → 扩展窗口（操作码扩展到8位）

第二层（8位操作码，前4位=1111）：
1111 0000 ~ 1111 1110 → 15条二地址指令
1111 1111              → 继续扩展（操作码扩展到12位）

第三层（12位操作码，前8位全1）：
1111 1111 0000 ~ 1111 1111 1110 → 15条一地址指令
1111 1111 1111                   → 继续扩展

第四层（16位操作码，前12位全1）：
1111 1111 1111 0000 ~ 1111 1111 1111 1111 → 16条零地址指令
```

### 关键原则

1. **短操作码的编码不能是长操作码的前缀**
2. 短操作码会预留一些编码作为扩展标识
3. 每扩展一层，操作码位数增加一个固定长度

### 例题：操作码扩展

> **题目**：某计算机指令字长16位，地址码4位。要求设计三地址指令15条、二地址指令15条、一地址指令15条、零地址指令16条。是否可行？

**解析**：

| 层级 | 操作码位数 | 地址码位数 | 指令条数 | 计算过程 |
|------|-----------|-----------|---------|---------|
| 三地址 | 4 | 4+4+4=12 | 15条 | $2^4-1=15$，留1个编码扩展 |
| 二地址 | 8 | 4+4=8 | 15条 | 1×$(2^4-1)=15$，留1个编码扩展 |
| 一地址 | 12 | 4 | 15条 | 1×$(2^4-1)=15$，留1个编码扩展 |
| 零地址 | 16 | 0 | 16条 | 1×$2^4=16$ |

**答案**：可行。正满足设计要求，每层都恰好用满。

### 解题套路

**给定指令条数，判断是否可行的步骤：**

1. 确定基础操作码位数
2. 从指令条数最多的（地址码最多的）开始分配
3. 每条指令消耗 $2^{剩余地址码位数}$ 个编码空间
4. 检查是否够用


# 5.2 寻址方式（必考重点！）

## 什么是寻址方式？

寻址方式就是**指令如何找到操作数**的方式。地址码字段给出的不是完整地址，而是一个"形式地址"，需要经过某种变换才能得到"有效地址（EA）"。

## 九大寻址方式详解

### 5.2.1 立即寻址（Immediate Addressing）

| 项目 | 内容 |
|------|------|
| **操作数位置** | 指令中**直接给出** |
| **有效地址** | 无（操作数就是地址码字段本身） |
| **优点** | 最快，无需访存 |
| **缺点** | 操作数大小受限（字段位数有限） |
| **应用** | 给寄存器赋常数值 |

```
指令:   ADD R1, #100
        操作数100直接放在指令中
```

> **注意**：立即寻址的"操作数"在指令中，不需要访问内存就能拿到。

### 5.2.2 直接寻址（Direct Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = A（地址码就是操作数地址） |
| **访存次数** | 1次（取操作数） |
| **优点** | 简单直观 |
| **缺点** | 地址范围受限 |

```
指令:   LOAD R1, [1000]
        地址码=1000 → 去内存地址1000处取数据
```

### 5.2.3 间接寻址（Indirect Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = (A)（A指向的内存单元里存的是地址） |
| **访存次数** | **2次**（第一次取地址，第二次取操作数） |
| **优点** | 可寻址范围大，编程灵活 |
| **缺点** | 速度慢（两次访存） |

```
指令:   LOAD R1, [1000]    （间接寻址标记）
        地址码=1000
        第一步：去内存1000读出内容，假设是2000
        第二步：去内存2000读出真正的操作数
```

### 5.2.4 寄存器寻址（Register Addressing）

| 项目 | 内容 |
|------|------|
| **操作数位置** | **寄存器中** |
| **有效地址** | 就是寄存器编号 |
| **访存次数** | 0次（访问寄存器，不访存） |
| **优点** | 最快（比立即慢一点因为需先取到寄存器编号，但远快于访存） |

```
指令:   ADD R1, R2
        R1和R2中的内容直接参与运算
```

### 5.2.5 寄存器间接寻址（Register Indirect Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = (R)（**寄存器中存放的是地址**） |
| **访存次数** | 1次（取操作数） |
| **优点** | 比内存间接寻址快，且可灵活改变地址 |

```
指令:   LOAD R1, [R2]
        R2中存放的是地址值（如1000）
        去内存地址1000取操作数
```

> **对比**：寄存器寻址是"寄存器中就是数据"；寄存器间接寻址是"寄存器中存的是地址，地址指向的数据才是操作数"。

### 5.2.6 相对寻址（Relative Addressing）⭐必考

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (PC) + A** |
| **访存次数** | 1次 |
| **用途** | **转移指令**（如JMP、JZ、JNZ等） |
| **特点** | 与当前指令位置相关，代码可浮动 |

```
指令:   JMP +50
        目标地址 = 当前PC值 + 50
        （注意：PC在取指阶段已自动加1！）
```

> **详细计算见5.3节**

### 5.2.7 基址寻址（Base Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (BR) + A**（BR = 基址寄存器） |
| **用途** | **多道程序、程序浮动**（重定位） |
| **特点** | 基址寄存器内容由**操作系统**管理，用户不能修改 |

```
多道程序场景：
程序A装在内存地址2000开始处
程序B装在内存地址5000开始处
→ 基址寄存器存放各自的起始地址
→ 用户程序中使用"相对自己的偏移量"访问数据
→ 实际地址 = BR + 偏移量
```

> **基址寄存器对用户是透明的**（用户不可修改）

### 5.2.8 变址寻址（Indexed Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (IX) + A**（IX = 变址寄存器） |
| **用途** | **数组访问**（最适合！） |
| **特点** | 变址寄存器内容由**用户**修改 |

```
访问数组 int arr[10] 中的元素：
A = 数组首地址（固定）
IX不断自增 → 依次访问arr[0], arr[1], ..., arr[9]

变址寄存器自动修改（通过增量指令），适合循环遍历数组
```

### 5.2.9 偏移寻址（Offset Addressing）

**偏移寻址** = 基址寻址 + 变址寻址 + 相对寻址

这三个方式的共同特点：**有效地址 = 某个寄存器内容 + 偏移量**

> **间接寻址不属于偏移寻址！**（因为它没有"基准+偏移"的结构）

### 5.2.10 寻址方式速查对比表

| 寻址方式 | 有效地址公式 | 访存次数 | 典型用途 |
|----------|------------|---------|---------|
| **立即寻址** | 操作数在指令中 | 0 | 赋常数 |
| **直接寻址** | EA = A | 1 | 简单数据访问 |
| **间接寻址** | EA = (A) | 2 | 指针操作 |
| **寄存器寻址** | 操作数在寄存器R中 | 0 | 算数逻辑运算 |
| **寄存器间接寻址** | EA = (R) | 1 | 指针访问 |
| **相对寻址** | EA = (PC) + A | 1 | 转移指令 |
| **基址寻址** | EA = (BR) + A | 1 | 多道程序/程序浮动 |
| **变址寻址** | EA = (IX) + A | 1 | 数组访问 |
| **先变址后间址** | EA = ((IX) + A) | 2+ | 复合寻址 |


# 5.3 相对寻址计算（必考！）

## 核心公式

```
目标地址 = 当前指令地址 + 指令长度 + 位移量
```

或者等价的：

```
目标地址 = (PC) + 偏移量
```

## 为什么要加"指令长度"？

**因为PC在取指阶段已经自动加1了！**

详细过程：
1. CPU从PC指向的地址取出当前指令
2. 每取一个字节，PC自动+1（因为PC默认"指向下一个字节"）
3. 取完整个指令后，PC已指向**下一条指令的起始地址**
4. 相对寻址中的"相对"是相对于**下一条指令**而言的

## 记忆口诀

> **相对寻址不要慌，先更新PC再算账。**
> **PC指向下一条，加上偏移就是目标。**

## 典型例题

### 例题1：基础计算

**题目**：某指令地址为1000H，指令长度为4字节，位移量为8，则转移目标地址为？

**计算**：
```
目标地址 = 1000H + 4 + 8 = 100CH
```

### 例题2：带符号位移量

**题目**：当前指令地址为2000H，指令长度为2字节，位移量为 -10（补码表示），目标地址为？

**计算**：
```
目标地址 = 2000H + 2 + (-10)
        = 2002H - 0AH
        = 1FF8H
```

### 例题3：位移量取值范围

**题目**：如果位移量字段为8位补码，其取值范围是？

**解析**：
- 8位补码范围：-128 ~ +127
- 向前最多跳 -128（向低地址）
- 向后最多跳 +127（向高地址）
- 注意：如果指令长度为2字节，可跳转范围为：
  - 最小：当前地址 + 2 + (-128) = 当前地址 - 126
  - 最大：当前地址 + 2 + (+127) = 当前地址 + 129


# 5.4 指令格式设计

## 设计步骤

**第一步**：根据指令条数确定操作码位数
- n位操作码最多表示 $2^n$ 条指令
- 若需要 m 条指令 → 操作码位数 ≥ $\lceil \log_2{m} \rceil$

**第二步**：确定地址码位数
- 地址码位数决定寻址范围
- 若地址码为 n 位，直接寻址可访问 $2^n$ 个存储单元

**第三步**：计算指令字长
- 定长指令字长 = 操作码位数 + 各地址字段位数之和

**第四步**：字长对齐
- 指令字长必须是 **8的倍数**（字节的整数倍）
- 若不足，向上取整

## 综合例题

> **题目**：某计算机要求设计如下指令格式：
> - 三地址指令 29 条
> - 二地址指令 107 条
> - 一地址指令 50 条
> - 地址字段均为 6 位
> - 求指令字长至少为多少位？

**解析**：

```
三地址指令29条：
  操作码至少5位（2^5=32 ≥ 29）
  剩余编码数 = 32-29 = 3（作为扩展窗口）
  地址总位数 = 6+6+6 = 18

二地址指令107条：
  扩展窗口3个，每个窗口可表示2^6=64条
  最多可表示 3×64 = 192条
  需要107条 → 够用
  实际用掉 107/64 向上取整 = 2个扩展窗口 → 剩余1个窗口

一地址指令50条：
  剩余1个窗口，可表示64条
  需要50条 → 够用

指令字长 = 5 + 18 = 23位
向上取整到8的倍数 → 24位
```

**答案**：指令字长至少 **24位**


# 5.5 ISA（指令集体系结构）

## 什么是ISA？

ISA是**Instruction Set Architecture**的缩写，是**程序员/编译器**能看到的那部分计算机体系结构。

## ISA包含的内容

| 包含 | 说明 |
|------|------|
| ① **指令格式** | 指令的编码方式、长度 |
| ② **操作类型** | 支持哪些操作（算术、逻辑、转移、访存等） |
| ③ **寻址方式** | 如何找到操作数 |
| ④ **操作数类型** | 支持的数据类型（整数、浮点、字符等） |
| ⑤ **寄存器定义** | 通用寄存器、专用寄存器数量与功能 |

## ISA不包含的内容

| 不包含 | 说明 |
|-------|------|
| **每条指令执行时具体的控制信号** | 这是**微体系结构**（Microarchitecture）的事 |
| 数据通路的具体实现 | 同一ISA可以有不同实现 |
| 流水线级数 | Pentium和Core i9都是x86 ISA |
| Cache大小/策略 | 属于微体系结构 |
| 时钟频率 | 属于微体系结构 |

## 关键区分

```
ISA（指令集体系结构）
    ↓ 同一个ISA可以有多种实现
微体系结构（Microarchitecture）
    ↓ 不同的实现
物理芯片
```

**经典类比**：ISA好比"驾照考试规则"，微体系结构好比"你怎么练车"——规则相同，练习方法可以不同。

## CISC vs RISC（ISA的两种风格）

| 特性 | CISC（复杂） | RISC（精简） |
|------|------------|------------|
| 指令数量 | 多（数百条） | 少（几十条） |
| 指令格式 | 变长 | 定长 |
| 寻址方式 | 多（十几种） | 少（几种） |
| 访存方式 | 算术指令可直接访存 | **LOAD/STORE结构**（只有LOAD/STORE访存） |
| 控制方式 | 微程序控制 | 硬布线控制 |
| 典型代表 | x86 | ARM, RISC-V, MIPS |


# 题库例题精讲（Q63-Q81）


## Q63：三地址+二地址指令字长

**题目**：某计算机指令字长32位，三地址指令29条、二地址指令m条，地址字段均为6位。若采用扩展操作码方式，则m的最大值为？

**解析**：

```
三地址指令29条：
  操作码至少5位（2^5=32 ≥ 29）
  剩余编码 = 32-29 = 3个（作为扩展窗口）

三地址指令格式：OP(5) + A1(6) + A2(6) + A3(6) = 23位
  剩余32-23=9位未用... 

  等等，这里指令字长32位，不是由操作码决定的。
  重新分析：

  指令字长32位固定。
  三地址指令：地址码总位数 = 6+6+6 = 18位
  三地址指令操作码位数 = 32 - 18 = 14位
  三地址指令29条，用掉29个编码
  剩余编码空间 = 2^14 - 29 个编码

  二地址指令：地址码位数 = 6+6 = 12位
  操作码位数 = 32 - 12 = 20位
  但操作码扩展需要"短操作码的编码不是长操作码的前缀"

  更准确的计算：
  三地址29条 → 操作码14位（32-6-6-6=14）
  剩余编码数 = 2^14 - 29
  每个剩余编码可以扩展出 2^12 条二地址指令（因为地址码少了6位）

  所以m最大值 = (2^14 - 29) × 2^6

  Wait, let me think more carefully.

  操作码扩展的视角：
  三地址指令：操作码14位，用了29个编码
  剩下的编码：2^14 - 29个
  这些剩下的编码，作为二地址指令的"前缀"
  二地址指令操作码 = 14 + 6 = 20位（因为少了一个地址字段，多出的6位加入操作码）
  每个三地址剩下的编码，对应 2^6 条二地址指令

  所以 m_max = (2^14 - 29) × 2^6

  但实际上，二地址也要预留扩展编码给一地址...
  题目问的是m的最大值，所以先不预留。
```

**答案**：m的最大值 = (2^14 - 29) x 2^6


## Q64：寻址范围

**题目**：某计算机地址码字段为16位，采用直接寻址方式，则其寻址范围为多少？

**解析**：
- 直接寻址：EA = A，地址码就是有效地址
- 地址码16位 → 可以表示 $2^{16}$ 个不同的地址
- 寻址范围 = $2^{16} = 65536$ 个存储单元

**答案**：64K（或65536个存储单元）


## Q65：先变址后间址

**题目**：某计算机采用先变址后间址的寻址方式，指令中给出的形式地址为D，变址寄存器为IX，则该指令操作数的有效地址为？

**解析**：

"先变址后间址" = 先做变址运算，再做间接寻址

步骤拆解：
1. 先变址：计算临时地址 = (IX) + D
2. 后间址：以临时地址为指针，去内存读取真正的地址
3. 有效地址 EA = ((IX) + D)

**答案**：EA = ((IX) + D)


## Q66：相对寻址转移

**题目**：某指令地址为2000H，指令长度为4字节，采用相对寻址转移到地址2010H，则指令中的位移量为多少？

**解析**：

```
目标地址 = 当前指令地址 + 指令长度 + 位移量
2010H = 2000H + 4 + 位移量
位移量 = 2010H - 2000H - 4
       = 10H - 4
       = 16 - 4
       = 12 = 0CH
```

**答案**：位移量 = 0CH （即十进制的12）


## Q67：不属于偏移寻址

**题目**：以下哪种寻址方式不属于偏移寻址？
A. 基址寻址  B. 变址寻址  C. 相对寻址  D. 间接寻址

**解析**：

偏移寻址的三个成员：
- 基址寻址：EA = (BR) + A ✓
- 变址寻址：EA = (IX) + A ✓
- 相对寻址：EA = (PC) + A ✓

间接寻址：EA = (A) — 没有"基准+偏移"结构 ✗

**答案**：D. 间接寻址


## Q68：变址寻址

**题目**：变址寻址方式最适合用于？
A. 实现程序浮动  B. 实现多道程序  C. 处理数组问题  D. 实现程序转移

**解析**：
- 变址寻址：EA = (IX) + A
- IX可自动修改，适合循环遍历数组元素
- A固定为数组首地址，IX变化依次访问不同元素

| 选项 | 对应寻址方式 |
|------|------------|
| 程序浮动/重定位 | 基址寻址 |
| 多道程序 | 基址寻址 |
| **数组访问** | **变址寻址** |
| 程序转移 | 相对寻址 |

**答案**：C. 处理数组问题


## Q69：数组访问寻址

**题目**：以下哪种寻址方式最适合按下标顺序访问一维数组元素？
A. 直接寻址  B. 变址寻址  C. 寄存器寻址  D. 间接寻址

**解析**：
- 一维数组元素在内存中连续存放
- 变址寻址中，变址寄存器可自动修改（增量）
- 形式地址 = 数组首地址，变址寄存器 = 下标
- 每次循环变址寄存器+1，就能访问下一个元素

**答案**：B. 变址寻址


## Q70：基址+大端

**题目**：某计算机采用基址寻址，基址寄存器BR=1000H，形式地址为数据在结构体中的偏移量。结构体成员的偏移地址分别为：A偏移0，B偏移2，C偏移4。采用大端方式存储，变量A的值为0x1234，变量B的值为0x5678。问执行指令 LOAD R1, 2(BR) 后，R1中的内容为？

**解析**：

```
基址寻址：EA = (BR) + A = 1000H + 2 = 1002H

偏移量A=2对应结构体中的B成员

大端存储（Big Endian）：
  高字节存低地址，低字节存高地址

变量B的值为0x5678，占2个字节
在大端模式下：
  地址1002H 存 0x56（高字节）
  地址1003H 存 0x78（低字节）

LOAD R1, 2(BR) 从地址1002H读取一个字（假设字长为16位）
R1 = 0x5678
```

**答案**：R1 = 0x5678


## Q71：数组+变址

**题目**：已知数组A的首地址为2000H，每个元素占2字节。变址寄存器IX初始值为0。采用变址寻址方式，执行指令 LOAD R1, A(IX) 三次后（每次IX自增2），R1最终读到的值为？

**解析**：

```
第一次：EA = (IX) + A = 0 + 2000H = 2000H → 读A[0]
        然后IX自增2 → IX=2

第二次：EA = 2 + 2000H = 2002H → 读A[1]
        然后IX自增2 → IX=4

第三次：EA = 4 + 2000H = 2004H → 读A[2]
        R1最终 = 内存地址2004H处的内容 = A[2]的值
```

**答案**：R1中为 A[2] 的值（即地址2004H处的内容）

## 6.1 CPU的组成

### 6.1.1 CPU两大部件

同学们，我们把CPU想象成一个**工厂**。

工厂里需要两种角色：
1. **工人**（运算器）-- 负责真正干活：做加法、做比较、做逻辑判断
2. **厂长**（控制器）-- 负责指挥协调：告诉工人干什么、从哪里取材料、结果送到哪

> **CPU = 运算器 + 控制器**

这两个部件缺一不可。没有工人，工厂没人干活；没有厂长，工人不知道干啥。

```
┌─────────────────────────────────────────┐
│                CPU                       │
│  ┌─────────────────┐  ┌───────────────┐  │
│  │   运算器          │  │   控制器       │  │
│  │  ┌──────┐       │  │  ┌─────────┐  │  │
│  │  │ ALU  │       │  │  │   PC    │  │  │
│  │  ├──────┤       │  │  ├─────────┤  │  │
│  │  │ 通用寄存器组 │  │  │   IR    │  │  │
│  │  ├──────┤       │  │  ├─────────┤  │  │
│  │  │ PSW  │       │  │  │  时序电路│  │  │
│  │  └──────┘       │  │  ├─────────┤  │  │
│  │                 │  │  │微操作信号│  │  │
│  │                 │  │  │ 发生器   │  │  │
│  │                 │  │  └─────────┘  │  │
│  └─────────────────┘  └───────────────┘  │
└─────────────────────────────────────────┘
```


### 6.1.2 运算器 -- "工人"

运算器是CPU中负责**数据加工处理**的部件。它包含三个核心部分：

#### (1) ALU（算术逻辑单元）-- 核心打工人

**类比**：就像工厂里的**车床**，你给它两个数，告诉它"加"或者"比较"，它就能给出结果。

- ALU是**组合逻辑电路**（没有存储功能，输入变了输出立刻变）
- 能做：加法、减法、与、或、异或、比较等操作
- ALU的控制信号叫 **ALUop**，告诉ALU这次做什么操作

#### (2) 通用寄存器组 -- 工人的工作台

**类比**：工人不能每次都去仓库（内存）拿材料，那样太慢。工作台（寄存器）上放一些常用的材料和半成品。

- 通用寄存器 = CPU内部的高速"小仓库"
- **位数 = 机器字长**（32位CPU的寄存器就是32位）
- 既可以存**数据**，也可以存**地址**
- 汇编程序员可以**直接使用**（通过寄存器名访问）

#### (3) 状态寄存器 PSW / 标志寄存器 -- 工人的记事本

**类比**：工人做完一个操作后，在记事本上记录这次操作的结果特点：结果是不是0？有没有溢出？是正还是负？

PSW（Program Status Word，程序状态字寄存器）存放的是**算术/逻辑运算结果的状态信息**：

| 标志位 | 含义 | 例子 |
|--------|------|------|
| ZF（零标志） | 结果为0时置1 | 5-5=0 → ZF=1 |
| SF（符号标志） | 结果为负时置1 | 3-5=-2 → SF=1 |
| OF（溢出标志） | 有符号数溢出时置1 | 127+1=溢出 → OF=1 |
| CF（进位/借位标志） | 无符号数进位/借位时置1 | 加法最高位有进位 → CF=1 |


### 6.1.3 控制器 -- "厂长"

控制器是CPU的**指挥中心**，它负责：
1. 从内存中**取指令**
2. **分析指令**（这条指令要干啥）
3. **发出控制信号**（指挥运算器和其他部件干活）

控制器包含以下部件：

#### (1) PC（程序计数器）-- 厂长的工序单

**类比**：厂长手里拿着一张工序单，上面写着下一步要做的工序编号。做完一道工序，就把编号加1，指向下一道。

- PC存放**下一条指令的地址**
- 每取完一条指令，PC**自动加1**（指向下一条指令）
- 遇到跳转指令 → PC被修改为跳转目标地址
- **位数**取决于**存储容量和指令字长**

> PC位数公式：若主存有 $2^n$ 个存储单元，指令字长=存储字长，则PC至少需要**n位**

#### (2) IR（指令寄存器）-- 厂长的当前任务单

**类比**：工序单（PC）指向的指令取出来后，放在桌上（IR），厂长看着这条指令安排工作。

- IR存放**当前正在执行**的指令
- **位数 = 指令字长**
- 取指阶段：从内存取出指令 → 送入IR
- 执行阶段：控制器分析IR中的指令 → 发出对应控制信号

#### (3) MAR（存储器地址寄存器）-- 仓库的货架号

- 存放**要访问的内存单元的地址**
- 位数 = 地址线根数 = $\log_2(主存容量)$

#### (4) MDR（存储器数据寄存器）-- 仓库的取货台

- 存放**要从内存读出或写入的数据**
- 位数 = 存储字长 = 数据线根数

```
取指令过程（图解）：
┌─────┐     地址      ┌──────────┐
│ PC  │─────────────→│   MAR    │
└─────┘              └────┬─────┘
                          │ 送地址到地址总线
                          ↓
                     ┌──────────┐
                     │   主存    │
                     └────┬─────┘
                          │ 返回指令
                          ↓
┌─────┐     指令      ┌──────────┐
│ IR  │←─────────────│   MDR    │
└─────┘              └──────────┘
```

#### (5) 时序电路 -- 厂长的时钟

- 产生各种时序信号，确保各部件**按正确的时间顺序**工作
- 就像工厂里的**节拍器**，一拍一拍地推进

#### (6) 微操作信号发生器 -- 厂长的发令枪

- 根据IR中的指令 + 时序信号 → 生成各种**微操作控制信号**
- 这些信号控制：寄存器读写、ALU操作、总线传输等


### 6.1.4 CPU不包含的部件 ⚠️

**地址译码器不在CPU中，它在主存中！**

**类比**：你给了一个房间号（地址），需要**楼管员**（地址译码器）帮你找到具体是哪个房间。楼管员不住在厂长办公室（CPU），而是住在仓库楼（主存）。

> **常考判断题**："地址译码器属于CPU" → **错误** ❌

| 属于CPU | 不属于CPU（在主存） |
|---------|-------------------|
| ALU、通用寄存器、PSW | 地址译码器 |
| PC、IR、MAR、MDR | 存储单元阵列 |
| 时序电路、微操作信号发生器 | |


#### 例题 Q82-Q83（CPU组成）

**例题 Q82**：CPU中不包含下列哪个部件？
A. ALU
B. 通用寄存器
C. 地址译码器
D. PSW

**解析**：地址译码器是主存中的部件，用于将地址信号翻译为具体的存储单元选择信号。ALU、通用寄存器、PSW都是CPU内部的部件。

**答案**：C


**例题 Q83**：CPU中控制器的功能是什么？
A. 进行算术运算和逻辑运算
B. 存储程序和数据
C. 从内存取指令、分析指令、发出控制信号
D. 进行输入输出操作

**解析**：控制器的核心功能是指挥协调，即取指令、分析指令、产生控制信号驱动各部件工作。ALU负责算术逻辑运算（A），存储器负责存储程序和数据（B）。

**答案**：C


## 6.2 重要寄存器（必考！）

同学们，寄存器是本章**最重要的考点**，几乎每次考试都有3-5道题。下面我一张表给你讲清楚所有寄存器的关键属性。

### 6.2.1 寄存器速查表

| 寄存器 | 全称 | 存放内容 | 位数 | 程序员可见？ |
|--------|------|---------|------|:----------:|
| **PC** | 程序计数器 | 下条指令的地址 | 取决于存储容量和指令字长 | ✅ 可见 |
| **IR** | 指令寄存器 | 当前执行的指令 | = 指令字长 | ❌ 透明 |
| **通用寄存器** | 通用寄存器组 | 数据或地址 | = 机器字长 | ✅ 可见 |
| **MAR** | 存储器地址寄存器 | 要访问的存储单元地址 | = 地址线数 | ❌ 透明 |
| **MDR** | 存储器数据寄存器 | 读出或写入的数据 | = 存储字长 | ❌ 透明 |
| **PSW** | 程序状态字寄存器 | 运算结果的状态（ZF/SF/OF/CF） | 与机器字长有关 | ✅ 可见 |

### 6.2.2 逐个详解

#### PC（程序计数器）-- 必考重点

**存放**：下一条将要执行指令的地址

**位数怎么确定？**
- 如果主存有 $2^n$ 个存储单元（按字节编址），指令字长 = 存储字长
- 那么PC至少需要 **n位** 才能寻址所有单元
- 例：64KB（$2^{16}$ 字节）主存 → PC至少 **16位**

**工作过程**：
1. 取指：将PC内容送到MAR，从主存取指令
2. PC自动加1：指向下一条指令
3. 遇到转移指令：PC被修改为目标地址

#### IR（指令寄存器）

**存放**：当前正在执行的指令

**位数**：**一定等于指令字长**

**为什么程序员不可见？**
- IR是控制器内部的工作寄存器
- 程序员通过编程无法直接读取或修改IR
- 它由控制器自动管理，对程序员是"透明的"

#### 通用寄存器

**位数**：**一定等于机器字长**

**用途**：
- 存放**操作数**（参与运算的数据）
- 存放**地址**（指针、索引等）
- 汇编程序员可以**直接通过名字使用**（如EAX、EBX、R0、R1等）

#### MAR 和 MDR

**类比**：你去图书馆借书。
- MAR = 你在**索书条上写的书号**（告诉管理员你要哪本书）
- MDR = 管理员**取出来递给你的书**

**关键关系**：
- MAR位数 = 地址线根数 = $\log_2(存储单元总数)$
- MDR位数 = 数据线根数 = 存储字长

**例子**：某计算机主存容量为 4M×32位
- 4M = $2^{22}$ 个存储单元 → MAR = **22位**
- 每个单元32位 → MDR = **32位**

#### PSW（程序状态字寄存器）

**存放**：运算结果的**状态标志**

常用标志位：
- **ZF**（Zero Flag）：结果为0 → ZF=1
- **SF**（Sign Flag）：结果为负 → SF=1
- **OF**（Overflow Flag）：有符号溢出 → OF=1
- **CF**（Carry Flag）：无符号进位/借位 → CF=1

**考试技巧**：当题目说"标志寄存器"时，指的就是PSW。

### 6.2.3 可见 vs 不可见（透明）-- 必考分类题 ⚠️

**汇编程序员可见的寄存器**（可以直接在程序中引用）：
| 寄存器 | 为什么可见 |
|--------|-----------|
| **PC** | 转移指令需要修改PC（JMP、CALL等） |
| **通用寄存器** | 直接存放操作数和运算结果（MOV、ADD等） |
| **PSW** | 条件转移指令需要判断标志位（JZ、JNZ等） |
| **基址寄存器** | 用于地址计算（某些体系结构中） |

**程序员不可见的寄存器**（透明寄存器）：
| 寄存器 | 为什么不可见 |
|--------|------------|
| **IR** | 控制器内部自动使用，程序员不需干预 |
| **MAR** | 地址传输由控制器自动完成 |
| **MDR** | 数据传输由控制器自动完成 |
| **微指令寄存器** | 微程序控制器内部使用 |

**记忆口诀**："PC通用PSW可见，IR玛德（MAR/MDR）不可见"

> 注："玛德"= MAR + MDR 的谐音记忆

## 6.3 指令周期

### 6.3.1 三个基本概念

同学们，理解指令周期之前，我们先搞清楚三个"周期"的关系。

**生活类比**：做一道菜的过程。

| 计算机概念 | 生活类比 | 说明 |
|-----------|---------|------|
| **时钟周期** | 切一刀的时间 | 最小、最基础的时间单位 |
| **机器周期** | 完成一个步骤的时间（如"洗菜"） | 由几个时钟周期组成 |
| **指令周期** | 完成一道菜的总时间（如"番茄炒蛋"） | 由几个机器周期组成 |

**时间关系**：
```
指令周期 ≥ 机器周期 ≥ 时钟周期

时钟周期 ← 组成 → 机器周期 ← 组成 → 指令周期
```

#### 时钟周期（Clock Cycle）-- 最小时间单位

- **定义**：CPU主频的倒数，计算机操作的**最基本时间单位**
- **公式**：$时钟周期 = \dfrac{1}{主频}$
- **例子**：主频2GHz → 时钟周期 = 0.5ns
- **与CPI的关系**：CPI = 执行一条指令所需的时钟周期数，**与时钟频率无关**

#### 机器周期（Machine Cycle / CPU周期）

- **定义**：通过一次总线事务访问一次主存或I/O的时间
- **确定依据**：由**存取周期**决定（主存的速度）
- 一个机器周期通常包含**若干个时钟周期**
- 各机器周期长度**可以不同**
- 各指令的机器周期数**可以不同**

#### 指令周期（Instruction Cycle）

- **定义**：从取指令到执行完该指令的全部时间
- **组成**：指令周期 = **取指周期** + **间址周期（可选）** + **执行周期** + **中断周期（可选）**

```
指令周期完整结构：
┌────────────────────────────────────────────────────┐
│                    指令周期                          │
│  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐   │
│  │取指周期│→│间址周期│→│执行周期│→│中断周期│   │
│  │(必有)  │  │(可选)  │  │(必有)  │  │(可选)  │   │
│  └────────┘  └────────┘  └────────┘  └────────┘   │
└────────────────────────────────────────────────────┘
```

**关键结论**：**每个指令周期至少访问一次内存**（取指令）

### 6.3.2 各周期的详细说明

#### (1) 取指周期（Fetch Cycle）

- **每个指令的第一个机器周期**总是**取指周期**
- 做的事情：从主存中取出指令 → 送入IR
- **取指令是控制器的固有功能，不受操作码控制** ⚠️
- 不管是什么指令，取指阶段做的事情都一样

**取指周期流程**：
```
1. PC → MAR          （指令地址送MAR）
2. 读主存             （启动存储器读）
3. MDR → IR          （指令送入IR）
4. PC + 1 → PC       （PC指向下一条指令）
```

#### (2) 间址周期（Indirect Cycle）

- **只有间接寻址的指令才有**间址周期
- 间址周期结束后，**MDR中存放的是操作数的地址**
- 如果指令不涉及间接寻址，就没有间址周期

#### (3) 执行周期（Execution Cycle）

- 真正**执行指令**的阶段
- 不同指令的执行周期不同：
  - ADD指令：取操作数 → ALU做加法 → 存结果
  - JMP指令：修改PC
  - NOP指令：什么都不做
- 执行周期的工作**受操作码控制**

#### (4) 中断周期（Interrupt Cycle）

- 处理中断请求的周期
- **只有允许中断且存在中断请求时才进入**
- 不是每条指令都有中断周期

### 6.3.3 区分指令和数据

**最常考的判断题** ⚠️

**CPU区分指令和数据的依据是什么？**

**答案**：**指令周期的不同阶段**

| 阶段 | 从内存读出的内容 | 送入 |
|:----:|:--------------:|:----:|
| 取指阶段 | **指令** | IR |
| 执行阶段 | **数据** | 运算器 |

同一段二进制代码，在取指阶段被当作指令，在执行阶段被当作数据。这是冯·诺依曼"存储程序"思想的体现——指令和数据在存储器中形式上没有差别。

## 6.4 数据通路

### 6.4.1 什么是数据通路？

**类比**：数据通路就像工厂里的**传送带系统**。

- 各个部件（寄存器、ALU）是工厂里的**工位**
- 传送带（总线）连接各个工位
- 三态门是传送带上的**开关阀门**，控制是否连接传送带

数据通路 = 数据在各部件之间传输的**路径** + **控制逻辑**

### 6.4.2 三态门 -- 关键概念

**三态门**是一种特殊的电子开关，有三种输出状态：
1. **0**（低电平）
2. **1**（高电平）
3. **高阻态**（断开连接，相当于断路）

**作用**：控制部件与总线的**连接与断开**

**为什么需要三态门？**
- 单总线结构下，所有部件都挂在同一组总线上
- 如果不加控制，多个部件同时往总线上发送信号会**冲突**
- 三态门保证：每个时刻**只有一个器件向总线发送信息**

### 6.4.3 单总线结构

**结构特点**：
- CPU内部用**一组总线**连接所有部件
- ALU的一个输入端接总线，另一输入端通过**暂存器**接总线

```
单总线结构示意：
┌────────────────────────────────────────────────┐
│                                                │
│   ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐      │
│   │  通用  │  │      │  │      │  │      │      │
│   │  寄存器│  │  PC  │  │  IR  │  │ MAR  │      │
│   │  组   │  │      │  │      │  │      │      │
│   └──┬───┘  └──┬───┘  └──┬───┘  └──┬───┘      │
│      │         │         │         │           │
│   ┌──┴─────────┴─────────┴─────────┴───┐       │
│   │           内部总线                  │       │
│   └──┬─────────┬─────────┬─────────────┘       │
│      │         │         │                      │
│   ┌──┴──┐  ┌──┴──┐  ┌──┴──┐                   │
│   │暂存器│  │ MDR │  │  ALU│                   │
│   └──┬──┘  └─────┘  └──┬──┘                   │
│      │                 │                       │
│      └─────────────────┘                       │
│                                                │
└────────────────────────────────────────────────┘
```

**关键特性** ⚠️：
1. 每个时刻只能有**一个器件发送**信息到总线
2. 每个时刻可以有**多个器件接收**总线上的信息
3. ALU的输入端需要**暂存器**：因为ALU是组合逻辑，如果不暂存，输入会因总线变化而变化

**单总线优点**：结构简单、成本低
**单总线缺点**：一次只能传输一个数据，速度慢（瓶颈）

### 6.4.4 多总线结构

为了解决单总线的瓶颈，引入多总线结构：
- **双总线结构**：两条总线，可同时传输两个数据
- **三总线结构**：三条总线，进一步增加并行度

多总线结构允许**多个数据传输同时进行**，提高了CPU的处理速度。

### 6.4.5 操作元件 vs 状态元件 ⚠️

这是一个常考点！

| 类型 | 特点 | 包含 |
|:----:|:----:|:----:|
| **操作元件**（组合逻辑） | 输出仅取决于当前输入，无存储功能 | **ALU**、**多路选择器(MUX)** |
| **状态元件**（时序逻辑） | 输出取决于当前输入和之前的状态，有存储功能 | 寄存器、**PC**、通用寄存器组 |

**记忆方法**：
- 操作元件 = "加工"（组合逻辑，没有记忆）
- 状态元件 = "存储"（时序逻辑，有记忆）

**ALU一定是操作元件**，因为它是组合逻辑电路，没有存储功能。

### 6.4.6 控制信号

数据通路上数据的流动路径由**控制信号**控制：

| 控制信号 | 作用 |
|---------|------|
| **ALUop** | 控制ALU执行哪种操作（加、减、与、或等） |
| **SRout** | 控制移位寄存器的输出 |
| **Rin / Rout** | 控制寄存器的输入/输出（读写） |
| **MemRead / MemWrite** | 控制主存的读/写 |

## 6.5 控制器

### 6.5.1 控制器的两种实现方式

**生活类比**：假设你要控制家里的电器。

**方式一**：用**硬接线**把开关和电器直接连好，一按开关就工作。
- 速度快（直接反应）
- 但想改线路需要重新布线（修改难）

**方式二**：用**Arduino/单片机**编写程序来控制。
- 速度慢一点（要执行程序指令）
- 但想改功能，改改程序就行了（修改易）

这就是硬布线控制器和微程序控制器的本质区别。

### 6.5.2 硬布线控制器（Hardwired Controller）

**工作原理**：
- 用**组合逻辑电路**（门电路、触发器等）直接产生控制信号
- 控制信号 = f(指令译码信号, 时钟信号, 状态信号)

**特点** ⚠️：
| 特性 | 评价 |
|:----:|:----:|
| **速度** | **快**（组合逻辑直接输出） |
| **修改难度** | **难**（要改硬件电路） |
| **时序系统** | **复杂**（需要精细的时序设计） |
| **灵活性** | 差（固定功能后难扩展） |
| **应用** | RISC处理器（如ARM、MIPS早期版本） |

**硬布线控制器的结构**：
```
指令 → 指令译码器 → 组合逻辑电路 → 控制信号
时钟 → 时序电路   ┘        ↑
                           │
其他状态信号 ──────────────┘
```

### 6.5.3 微程序控制器（Microprogrammed Controller）

**类比**：微程序控制器像一本**菜谱**。

- 每个菜（机器指令）对应一个**烹饪流程**（微程序）
- 流程中的每一步（微指令）告诉厨师（CPU）做什么
- 菜谱放在厨房（控制存储器CS）里

**基本概念**：
```
一条机器指令 → 一个微程序 → 若干条微指令 → 若干个微命令
```

**特点** ⚠️：
| 特性 | 评价 |
|:----:|:----:|
| **速度** | **慢**（需要从控制存储器读微指令） |
| **修改难度** | **容易**（修改控制存储器内容即可） |
| **时序系统** | **简单**（微指令周期化） |
| **灵活性** | 好（可扩展性强） |
| **应用** | CISC处理器（如x86系列） |

**控制存储器（CS）** ⚠️：
- 存放**微指令**
- 在**CPU内部**
- 用**ROM**实现（只读存储器，掉电不丢失）
- 不是主存！

**微程序控制器结构**：
```
取指微程序 → 取出一条机器指令 → 送IR
IR中的操作码 → 映射到控制存储器中的对应微程序入口
执行该微程序的各条微指令 → 产生控制信号
```


### 6.5.4 两种控制器对比总结 ⚠️（必考对比题）

| 比较项 | 硬布线控制器 | 微程序控制器 |
|:------:|:----------:|:-----------:|
| **速度** | 快 ✅ | 慢（读CS耗时） |
| **修改难度** | 难（改电路） | 容易（改CS内容） |
| **时序系统** | 复杂 | 简单 |
| **控制信号产生** | 组合逻辑直接产生 | 微指令译码产生 |
| **规整性** | 不规整，设计复杂 | 规整，设计简单 |
| **应用** | RISC | CISC |
| **成本** | 高（设计复杂） | 低（规整易实现） |

**记忆口诀**："硬快难复杂，微慢易简单"

## 6.6 微程序设计（必考！）

同学们，这一节是本章的**最高频考点**，概念多、易混淆，一定要理解清楚。

### 6.6.1 四个层次的"微"概念

从大到小的层次关系：

```
一条机器指令
    ↓ 由 ... 实现
一段微程序（由若干条微指令组成）
    ↓ 每条微指令包含 ...
若干个微命令
    ↓ 每个微命令控制执行 ...
若干个微操作
```

**层次图**：
```
机器指令（加法、减法、跳转...）
    │
    ▼
微程序（对应一条机器指令的微指令序列）
    │  ┌──────────────────┐
    │  │ 微指令1 微指令2 ... │  ← 存放在控制存储器CS中
    │  └──────────────────┘
    │
    ▼
微命令（组成微指令的最小控制单位）
    │  例：寄存器输入控制、ALU操作控制、总线控制
    ▼
微操作（微命令驱动的实际动作）
    例：R1内容送总线、ALU执行加法
```

**关键关系** ⚠️：
> **一条机器指令 → 一个微程序 → 若干条微指令 → 若干个微命令 → 若干个微操作**

### 6.6.2 详细解释每个概念

#### 微命令（Micro-command）
- **定义**：控制部件向执行部件发出的**最基本控制信号**
- 例子："R1out"（让寄存器R1的内容输出到总线）、"ALUadd"（让ALU做加法）
- 微命令是**控制信号的最小单位**

#### 微操作（Micro-operation）
- **定义**：微命令控制下执行的**最基本操作**
- 例子：R1的内容送到总线上、ALU完成加法运算
- 微操作是**操作的最小单位**

**微命令 vs 微操作**：
- 微命令 = "发令"（控制信号）
- 微操作 = "执行"（实际动作）
- 一个微命令对应一个微操作

#### 微指令（Micro-instruction）
- **定义**：**一组微命令**的集合，存放在控制存储器的一个单元中
- 一个CPU周期中，执行一条微指令
- 微指令包含：**操作控制字段** + **后继地址字段**（或顺序控制字段）

#### 微程序（Micro-program）
- **定义**：**若干条微指令的有序集合**
- 一条机器指令对应一个微程序
- 微程序存放在控制存储器（CS）中

### 6.6.3 相容性 vs 互斥性微命令 ⚠️

这是微程序设计中的**核心分类**：

| 类型 | 定义 | 例子 |
|:----:|:----:|:----:|
| **相容性微命令** | 在同一CPU周期中**可以同时出现**的微命令 | 不同部件的控制信号（如：R1out和ALUadd） |
| **互斥性微命令** | 在同一CPU周期中**不允许同时出现**的微命令 | 同一部件的互斥操作（如：R1out和R2out不能同时输出到总线） |

**判断方法**：
- 如果两个微命令的操作对象不同且互不影响 → **相容**
- 如果两个微命令会竞争同一资源 → **互斥**

**常见互斥例子**：
- 同一总线的不同源寄存器输出（R1out和R2out）
- 同一寄存器的读和写
- ALU的不同操作（ALUadd和ALUsub）

### 6.6.4 水平型 vs 垂直型微指令

| 比较项 | 水平型微指令 | 垂直型微指令 |
|:------:|:----------:|:----------:|
| **并行能力** | **强**（一条可执行多个操作） | 弱（一次只能一个操作） |
| **字长** | **长** | 短（类似机器指令） |
| **控制信号** | 直接表示 | 编码表示（需译码） |
| **效率** | 高 | 低 |
| **编程难度** | 难（并行控制复杂） | 易（类似汇编） |

**水平型微指令特点**：
```
┌──────────────────────────────────────────────┐
│  控制字段（多位）  │  后继地址字段            │
│  bit1: R1out       │  下一条微指令地址        │
│  bit2: R2in        │                          │
│  bit3: ALUadd      │                          │
│  ...               │                          │
└──────────────────────────────────────────────┘
```
- 每个控制位直接对应一个微命令
- 可以同时发出多个微命令（并行执行）
- 字长较长

**垂直型微指令特点**：
```
┌────────────────────────────────┐
│  微操作码  │  地址/数据  │  后继地址  │
└────────────────────────────────┘
```
- 类似机器指令格式
- 一次只能完成一个基本操作
- 字长短

### 6.6.5 字段直接编码法

**问题**：如果每个微命令都用1位表示，微指令字长会非常长。

**解决方案**：字段直接编码法——将**互斥的微命令**放在同一个字段中，用不同的编码表示不同的微命令。

**原理**：
```
互斥微命令集合A = {R1out, R2out, R3out, R4out}
   ↓ 编码
字段A = 2位（00=R1out, 01=R2out, 10=R3out, 11=R4out）
```

**原则** ⚠️：
- **互斥性微命令** → 放在**同一字段**（编码表示）
- **相容性微命令** → 放在**不同字段**（可同时发出）

**优点**：以较少的位数表示更多的微命令

### 6.6.6 微指令后继地址确定 -- 断定方式

微指令执行完后，如何确定下一条微指令的地址？

**断定方式**（最常见）：
- 每条微指令中有一个**后继地址字段**
- 该字段直接指定了下一条微指令的地址（全部或部分）
- 如有分支，由转移控制字段配合实现

**优点**：灵活，可以方便地实现条件转移


### 例题 Q149-Q161（微程序设计）

**例题 Q149**：一条机器指令对应？
A. 一条微指令
B. 一个微程序
C. 一个微命令
D. 一个微操作

**解析**：一条机器指令的功能由一段微程序实现，该微程序包含若干条微指令。

**答案**：B


**例题 Q150**：以下关于微指令、微程序、机器指令的关系，正确的是？
A. 机器指令由微指令组成
B. 微程序由机器指令组成
C. 微程序由微指令组成
D. 微指令由微程序组成

**解析**：层次关系是：微指令组成微程序，微程序实现机器指令。所以"C. 微程序由微指令组成"正确。

**答案**：C


**例题 Q151**：在同一CPU周期中不能同时出现的微命令称为？
A. 相容性微命令
B. 互斥性微命令
C. 相容性微操作
D. 互斥性微操作

**解析**：互斥性微命令是指在同一CPU周期中不允许同时出现的微命令，例如同一总线上不同寄存器的输出信号。

**答案**：B


**例题 Q152**：将互斥性微命令放在同一字段的设计方法是？
A. 水平型微指令
B. 字段直接编码法
C. 断定方式
D. 垂直型微指令

**解析**：字段直接编码法将互斥性微命令划分到同一字段，用不同编码表示不同的微命令，从而压缩微指令字长。

**答案**：B


**例题 Q153**：水平型微指令的特点是？
A. 字长短，并行能力弱
B. 字长短，并行能力强
C. 字长长，并行能力强
D. 字长长，并行能力弱

**解析**：水平型微指令的每个控制位直接对应一个微命令，字长长，可同时发出多个微命令，并行能力强。

**答案**：C


**例题 Q154**：垂直型微指令的特点是？
A. 一次只能完成一个基本操作
B. 可以执行多个并行操作
C. 字长较长
D. 效率高

**解析**：垂直型微指令结构类似机器指令，采用微操作码，一次只能完成一个基本操作（串行），字长短但效率低。

**答案**：A


**例题 Q155**：微指令中的后继地址字段用于？
A. 存放操作数
B. 存放运算结果
C. 指定下一条微指令的地址
D. 存放指令的操作码

**解析**：微指令通常包含操作控制字段和后继地址字段。后继地址字段（断定方式）用于指定下一条将要执行的微指令在控制存储器中的地址。

**答案**：C


**例题 Q156**：在微程序控制器中，微指令存放在？
A. 主存中
B. Cache中
C. 控制存储器（CS）中
D. 通用寄存器中

**解析**：微指令存放在**控制存储器（CS）**中，CS位于CPU内部，用ROM实现。

**答案**：C


**例题 Q157**：以下关于微程序设计的说法错误的是？
A. 一条机器指令对应一个微程序
B. 微程序由微指令组成
C. 控制存储器用RAM实现
D. 相容性微命令可以同时出现

**解析**：
- A正确：每条机器指令的功能由一个微程序实现
- B正确：微程序是若干微指令的有序集合
- C**错误**：控制存储器用**ROM**实现（不是RAM），因为ROM掉电不丢失
- D正确：相容性微命令可以在同一CPU周期同时出现

**答案**：C


**例题 Q158**：关于微命令和微操作的关系，正确的是？
A. 微命令是实际执行的动作
B. 微操作是控制信号
C. 微命令控制微操作的执行
D. 微命令和微操作没有关系

**解析**：微命令是控制信号（发令），微操作是实际执行的动作（执行）。微命令驱动微操作的执行。

**答案**：C


**例题 Q159**：若某计算机有32个互斥性微命令，采用字段直接编码法，至少需要几位？
A. 32位
B. 5位
C. 6位
D. 取决于具体设计

**解析**：32个互斥性微命令放在同一字段，采用编码表示。32种不同状态需要 $2^n \ge 32$ → n = 5位。

**答案**：B


**例题 Q160**：若某计算机有16个相容性微命令，采用直接表示法需要几位？
A. 4位
B. 16位
C. 8位
D. 32位

**解析**：相容性微命令可以同时出现，必须用直接表示法（每个微命令对应1位），所以16个微命令需要16位。如果放在同一字段编码，反而失去了并行能力。

**答案**：B


**例题 Q161**：以下关于微程序控制器的说法正确的是？
A. 微程序控制器不需要控制存储器
B. 微程序控制器的时序系统比硬布线复杂
C. 微程序控制器通过执行微指令来产生控制信号
D. 微程序控制器中，微指令的长度等于机器字长

**解析**：
- A错误：微程序控制器的核心就是控制存储器（CS）
- B错误：微程序控制器的时序系统**简单**（不是复杂）
- C正确：微程序控制器从CS中读取微指令，由微指令译码产生控制信号
- D错误：微指令长度不等于机器字长，微指令长度由设计决定

**答案**：C

### 一、知识点速记卡

#### CPU组成

| 部件 | 子部件 | 功能 | 备注 |
|:----:|:------:|:----|:----:|
| 运算器 | ALU | 算术逻辑运算 | 组合逻辑电路，无存储 |
| 运算器 | 通用寄存器 | 存数据/地址 | 位数=机器字长，程序员可见 |
| 运算器 | PSW | 存运算状态 | ZF/SF/OF/CF |
| 控制器 | PC | 存下条指令地址 | 程序员可见，位数取决于存储容量+指令字长 |
| 控制器 | IR | 存当前指令 | 透明，位数=指令字长 |
| 控制器 | MAR | 存内存地址 | 透明，位数=地址线数 |
| 控制器 | MDR | 存内存数据 | 透明，位数=存储字长 |

#### 可见性

```
可见：PC、通用寄存器、PSW、基址寄存器
透明：IR、MAR、MDR、微指令寄存器
```

#### 周期对比

| 周期 | 定义 | 关系 |
|:----:|:----|:----:|
| 时钟周期 | CPU主频的倒数，最小时间单位 | 组成机器周期 |
| 机器周期 | 访问一次主存的时间，由存取周期决定 | 由多个时钟周期组成 |
| 指令周期 | 取指令到执行完的时间 | 由多个机器周期组成 |

#### 指令周期组成

```
指令周期 = 取指周期（必有） + 间址周期（可选） + 执行周期（必有） + 中断周期（可选）
```

#### 控制器对比

| 特性 | 硬布线 | 微程序 |
|:----:|:------:|:------:|
| 速度 | 快 | 慢 |
| 修改 | 难 | 易 |
| 时序系统 | 复杂 | 简单 |

#### 微程序设计层次

```
机器指令 → 微程序 → 微指令 → 微命令 → 微操作
```

#### 微命令分类

| 类型 | 能否同时出现 | 编码方法 |
|:----:|:----------:|:--------:|
| 相容性 | 能 | 不同字段 |
| 互斥性 | 不能 | 同字段编码 |

#### 微指令类型

| 类型 | 字长 | 并行能力 | 效率 |
|:----:|:----:|:--------:|:----:|
| 水平型 | 长 | 强 | 高 |
| 垂直型 | 短 | 弱 | 低 |


### 二、综合检测（自测题）

**Q1**：某计算机主存容量为16M×32位，指令字长等于存储字长。问：
(1) PC至少多少位？
(2) IR多少位？
(3) 通用寄存器多少位？
(4) MAR多少位？
(5) MDR多少位？

**解析**：
- (1) 16M = $2^{24}$个存储单元，PC至少 **24位**
- (2) IR = 指令字长 = 存储字长 = **32位**
- (3) 通用寄存器 = 机器字长 = **32位**（通常等于存储字长）
- (4) MAR = $\log_2(16M)$ = 24位
- (5) MDR = 存储字长 = **32位**


**Q2**：判断正误并说明理由：
(1) 取指令的操作受指令操作码控制。
(2) CPU中的地址译码器用于将地址转换为片选信号。
(3) IR对汇编程序员是可见的。
(4) 微程序控制器的时序系统比硬布线控制器复杂。
(5) 每条机器指令都必须经过取指、间址、执行、中断四个周期。

**答案**：
(1) **错误**。取指令是控制器的固有功能，不受操作码控制。
(2) **错误**。地址译码器在主存中，不在CPU中。
(3) **错误**。IR是透明寄存器，程序员不可见。
(4) **错误**。微程序控制器的时序系统更简单。
(5) **错误**。间址周期和中断周期是可选的，不是每条指令都有。


**Q3**：某微程序控制器有40个微命令，其中20个互斥、20个相容。采用字段直接编码法，最少需要多少位？

**解析**：
- 20个互斥微命令：放在同一字段，需要 $2^n \ge 20$ → n = **5位**（可表示32种状态）
- 20个相容微命令：每个需要1位，共 **20位**
- 总计：5 + 20 = **25位**

如果再加一个后继地址字段的话，还需要额外位数。


**Q4**：以下微命令对哪些是互斥的？哪些是相容的？
A. R1out（R1输出到总线）
B. R2out（R2输出到总线）
C. ALUadd（ALU执行加法）
D. R1in（总线数据写入R1）
E. R3out（R3输出到总线）

**解析**：
- **互斥**：A和B（不能同时输出到同一条总线）；A和E（同为总线输出竞争）；B和E（同为总线输出竞争）
- **相容**：A和C（R1输出到总线和ALU加法的资源不冲突）；C和D（ALU加法和R1写入不冲突）；B和D（R2输出和R1写入不冲突）


### 三、考场速记口诀

> **CPU = 运算器 + 控制器**
> **可见PC与通用，还有PSW基址**
> **透明IR玛德（MAR+MDR+微指令）**
> **时钟最小，机器周期靠存取**
> **每指令周期必访存（取指令）**
> **硬快快，微慢慢**
> **硬复杂，微简单**
> **互斥编同段，相容用直表**
> **水平长且强，垂直短而弱**
