## 2.1 数制转换

### 2.1.1 为什么有不同数制？

**生活类比**：
- 十进制（Decimal）：我们日常用的，0-9，逢十进一
- 二进制（Binary）：计算机用的，0-1，逢二进一
- 八进制（Octal）：0-7，逢八进一
- 十六进制（Hexadecimal）：0-9和A-F，逢十六进一

计算机内部所有数据都是**二进制**（0和1），但二进制写起来太长，所以用**八进制**或**十六进制**作为缩写法。

```
二进制    八进制    十进制    十六进制
0000        0        0         0
0001        1        1         1
0010        2        2         2
0011        3        3         3
0100        4        4         4
0101        5        5         5
0110        6        6         6
0111        7        7         7
1000       10        8         8
1001       11        9         9
1010       12       10         A
1011       13       11         B
1100       14       12         C
1101       15       13         D
1110       16       14         E
1111       17       15         F
```

**十六进制为什么常用？**
- 1位十六进制数 = 4位二进制数，一一对应
- 例如：二进制 `1011 1110` = 十六进制 `BE`
- 比二进制短4倍，比八进制更紧凑

### 2.1.2 十进制转二进制 -- 整数部分

**方法：除2倒取余法**

用十进制数不断除以2，记录余数，直到商为0，然后将余数从下往上倒序排列。

**例：将十进制 66 转为二进制**

```
66 ÷ 2 = 33 ... 余 0  ↑
33 ÷ 2 = 16 ... 余 1  │
16 ÷ 2 =  8 ... 余 0  │
 8 ÷ 2 =  4 ... 余 0  │
 4 ÷ 2 =  2 ... 余 0  │
 2 ÷ 2 =  1 ... 余 0  │
 1 ÷ 2 =  0 ... 余 1  │
                       │
从下往上读：1000010
```

所以 66(D) = 1000010(B)

**验证**：1×2^6 + 0×2^5 + 0×2^4 + 0×2^3 + 0×2^2 + 1×2^1 + 0×2^0 = 64 + 2 = 66 ✅

**扩充到8位（补足高位）**：0100 0010

**口诀**：除2取余，倒序排列。

### 2.1.3 十进制转二进制 -- 小数部分

**方法：乘2取整法**

用小数部分不断乘以2，取出整数部分，直到小数部分为0或达到精度要求。

**例：将十进制 0.625 转为二进制**

```
0.625 × 2 = 1.25 → 取整数 1  ↓
0.25  × 2 = 0.5  → 取整数 0  │
0.5   × 2 = 1.0  → 取整数 1  │
                                │
从上往下读：0.101
```

所以 0.625(D) = 0.101(B)

### 2.1.4 任何十进制整数都可以用二进制表示，但任何十进制小数不一定

**重要结论**：任何一个十进制整数都能用有限位二进制精确表示，但**不是所有十进制小数都能用有限位二进制表示**。

**例：十进制 0.1 转二进制（无限循环）**

```
0.1 × 2 = 0.2 → 0
0.2 × 2 = 0.4 → 0
0.4 × 2 = 0.8 → 0
0.8 × 2 = 1.6 → 1
0.6 × 2 = 1.2 → 1
0.2 × 2 = 0.4 → 0  ← 开始循环！
0.4 × 2 = 0.8 → 0
...
```

所以 0.1(D) = 0.00011001100110011...(B) **无限循环小数**

**这意味着什么？**
- 计算机无法精确表示 0.1！
- 这就是为什么在编程中 `0.1 + 0.2 != 0.3`（浮点精度问题）
- 也是 IEEE754 中 1.2 无法精确表示的原因

**哪些小数可以精确表示？**
- 分母是2的幂的分数可以精确表示
- 例如：0.5(=1/2), 0.25(=1/4), 0.125(=1/8), 0.625(=5/8)
- 例如：0.3(=3/10)不能精确表示，因10不是2的幂

### 2.1.5 二进制、八进制、十六进制之间的转换

**二进制转八进制**：每3位一组（从右向左），每组转换为1位八进制数

**例**：`010 000 010`(B) → `2 0 2`(O) = 202(O)

**二进制转十六进制**：每4位一组（从右向左），每组转换为1位十六进制数

**例**：`0100 0010`(B) → `4 2`(H) = 42(H)

**十六进制转二进制**：1位十六进制数展开为4位二进制数

**例**：`BE`(H) → `1011 1110`(B)

## 2.2 原码、反码、补码、移码

### 2.2.1 为什么需要不同的编码方式？

**生活类比**：如何表示"欠钱"和"存钱"？

假设你有一个数字显示牌，只能显示"+"和"-"以及数字。现在我们要在计算机中用0和1表示正负数。

计算机面临的问题：**只有0和1，没有+和-，怎么办？**

### 2.2.2 原码 -- 最直观的方式

**定义**：最高位为**符号位**（0正1负），其余位为**数值的绝对值**。

```
原码 = 符号位 + 绝对值
```

**例：用8位原码表示**

```
+66 = 0 1000010 = 0100 0010
-66 = 1 1000010 = 1100 0010
  +0 = 0000 0000
  -0 = 1000 0000
```

**特点**：
- 直观易懂，符号位+绝对值
- **0有两种表示**：+0 = 0000...0，-0 = 1000...0
- 8位原码能表示 255 个不同数据（0占两个位置）
- n位原码整数范围：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$

**缺点**：
1. 0有两种表示，浪费一个编码
2. 做加减法时，符号位需单独处理，电路复杂

**记忆口诀**："原码最直观，符号+绝对值，0有两个，运算最麻烦。"

### 2.2.3 反码 -- 过渡方案

**定义**：
- **正数**：与原码相同
- **负数**：原码符号位不变，数值位**按位取反**

**例：用8位反码表示**

```
+66 = 0100 0010（同原码）
-66 = 1011 1101（符号位1不变，数值位取反）
  +0 = 0000 0000
  -0 = 1111 1111
```

**特点**：
- **0有两种表示**：+0=0000...0，-0=1111...1
- n位反码整数范围：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$（与原码相同）
- 反码是**补码的中间步骤**

**记忆口诀**："反码正同原，负则符号不变位取反，0也有两个。"

### 2.2.4 补码 -- 计算机最常用的编码

**定义**：
- **正数**：与原码相同
- **负数**：原码符号位不变，数值位**取反再加1**（即反码+1）

**例：求-66的8位补码**

```
  +66的原码 = 0100 0010
  取反     = 1011 1101（这就是-66的反码）
  加1      = 1011 1110（这就是-66的补码）
```

所以 [-66]补 = 1011 1110 = BE(H) = 0xBE

**例：求-1的8位补码**

```
  +1的原码 = 0000 0001
  取反     = 1111 1110
  加1      = 1111 1111（-1的补码）
```

**特点**：
- **0只有一种表示**：0000...0 ✅ -- 这是补码最重要的优势
- n位补码整数范围：$-2^{n-1} \sim 2^{n-1}-1$（比原码多表示一个数）
- 8位补码范围：-128 ~ 127（共256个数）
- **计算机系统中实际使用的是补码**

**为什么负数补码用"取反加1"？**

根本原因是：**让加减法统一用加法器实现**，不需要单独的减法器。

```
[x]补 + [y]补 = [x+y]补
[x]补 - [y]补 = [x]补 + [-y]补
```

**关键转换 -- 由[X]补求[-X]补**：

规则：**[X]补连同符号位取反，末位加1**（注意是"连同符号位"！）

```
例：[66]补 = 0100 0010
求[-66]补：
取反：1011 1101
加1： 1011 1110  ✅ = 0xBE

例：[-66]补 = 1011 1110
求[66]补：
取反：0100 0001
加1： 0100 0010  ✅
```

**补码的符号位扩展**：
- 正数补码：高位补0
- 负数补码：高位补1（符号位扩展）

```
例：将8位补码扩展为16位
[66]补 = 0100 0010 → 0000 0000 0100 0010
[-66]补 = 1011 1110 → 1111 1111 1011 1110
```

### 2.2.5 移码 -- 浮点数的阶码

**定义**：补码的符号位取反即得移码。

```
[X]移 = 2^(n-1) + X（偏置值为2^(n-1)）
```

**特点**：
- **0只有一种表示**（与补码一样）
- 数值部分与补码相同，符号相反
- 主要用于浮点数的**阶码**表示
- IEEE754中移码的偏置值为127（单精度）或1023（双精度）

**补码和移码的关系 -- 常考！**

| 对比项 | 补码 | 移码 |
|--------|------|------|
| 0的表示 | 唯一：000...0 | 唯一：100...0 |
| 数值部分 | 与移码相同 | 与补码相同 |
| 符号位 | 与移码相反 | 与补码相反 |
| 表示范围 | $-2^{n-1} \sim 2^{n-1}-1$ | 同补码 |

**记忆口诀**："补码移码是一家，数值相同符号反。"

**例：比较8位补码和移码**

```
真值     补码         移码（补码符号位取反）
  0     0000 0000    1000 0000
  1     0000 0001    1000 0001
 -1     1111 1111    0111 1111
127     0111 1111    1111 1111
-128    1000 0000    0000 0000
```

**⚠️ 注意**：移码的偏置值可以是不同的值（取决于位数），不只是2^(n-1)。IEEE754中用的是2^(n-1)-1。

## 2.3 定点数

### 2.3.1 什么是定点数？

**定义**：小数点位置固定不变的数。

计算机中定点数有两种：
- **定点整数**：小数点在最低位之后（可以认为小数点固定在末尾）
- **定点小数**：小数点在最高位之后（符号位之后）

### 2.3.2 n位定点整数范围

**n位无符号定点整数**：0 ~ 2^n - 1

**n位原码定点整数**：$-(2^{n-1}-1) \sim +(2^{n-1}-1)$

**n位补码定点整数**：$-2^{n-1} \sim 2^{n-1}-1$

| 位数 | 补码范围 |
|------|---------|
| 8位 | -128 ~ 127 |
| 16位 | -32768 ~ 32767 |
| 32位 | -2^31 ~ 2^31-1 |
| n位 | $-2^{n-1} \sim 2^{n-1}-1$ |

### 2.3.3 补码的符号位扩展

将一个n位补码扩展为m位（m>n）时：
- **正数**补码：高位补0
- **负数**补码：高位补1

```
例1：将8位补码[66]补 = 0100 0010 扩展为16位
    16位[66]补 = 0000 0000 0100 0010
    高位全部补0（因为符号位是0）

例2：将8位补码[-66]补 = 1011 1110 扩展为16位
    16位[-66]补 = 1111 1111 1011 1110
    高位全部补1（因为符号位是1）
```

**验证**：`1111 1111 1011 1110` 的补码 = ？
- 取反：`0000 0000 0100 0001`
- 加1：`0000 0000 0100 0010` = 66
- 所以这个是-66 ✅

### 2.3.4 n位定点小数

对于n位补码定点小数（1位符号位 + n-1位数值位）：
- 范围：$-1 \sim 1-2^{-(n-1)}$
- 最小负数：-1（补码表示：1.000...0）
- 最大正数：$1-2^{-(n-1)}$（补码表示：0.111...1）

## 2.4 IEEE754浮点数（难点！）

### 2.4.1 为什么需要浮点数？

**生活类比**：科学记数法。

我们在科学计算中经常需要表示：
- 很大的数：$6.02 \times 10^{23}$（阿伏伽德罗常数）
- 很小的数：$1.6 \times 10^{-19}$（电子电荷量）

如果用定点数表示，位数不够用。所以需要用**浮点数**，类似于科学记数法。

```
科学记数法：  ±1.xxxx × 2^(yyyy)
浮点数：      符号位 + 阶码 + 尾数
```

### 2.4.2 IEEE754单精度浮点数（32位）

**格式**：

```
┌─────┬───────────┬──────────────────────────────┐
│ 1位 │    8位     │             23位              │
│  S  │     E      │              M               │
│符号位│  阶码（移码）│          尾数（原码）           │
└─────┴───────────┴──────────────────────────────┘
```

| 字段 | 位数 | 含义 |
|------|------|------|
| S | 1位 | 符号位（0正1负） |
| E | 8位 | 阶码，用**移码**表示，偏置值**127** |
| M | 23位 | 尾数，原码小数，**隐含整数位1** |

**真值计算公式**：

$$x = (-1)^S \times 1.M \times 2^{E-127}$$

**关键点 -- 隐含整数位1（"隐藏1"）**

规格化浮点数的尾数部分满足 $1 \leq 尾数 < 2$，因此整数部分总是1。为了省1位精度，这个1**不存储**，只存小数部分（M）。

所以：实际尾数 = 1.M

**例如**：
- 如果M = 010000...0（23位）
- 则实际尾数 = 1.01...（二进制）

### 2.4.3 IEEE754双精度浮点数（64位）

**格式**：

```
┌─────┬─────────────┬────────────────────────────────────────┐
│ 1位 │    11位      │                  52位                   │
│  S  │      E       │                   M                   │
│符号位│ 阶码（移码）  │              尾数（原码）               │
└─────┴─────────────┴────────────────────────────────────────┘
```

| 字段 | 位数 | 含义 |
|------|------|------|
| S | 1位 | 符号位 |
| E | 11位 | 阶码，偏置值**1023** |
| M | 52位 | 尾数，隐含整数位1 |

$$x = (-1)^S \times 1.M \times 2^{E-1023}$$

### 2.4.4 规格化数、非规格化数、无穷大、0 的区分

IEEE754 用阶码E的特殊值来区分不同类型：

| 类型 | E的值 | M的值 | 含义 |
|------|-------|-------|------|
| **0** | E=0 | M=0 | 真值0（+0和-0） |
| **非规格化数** | E=0 | M≠0 | 非常接近0的数，隐含整数位为0 |
| **规格化数** | 1≤E≤254 | 任意 | 正常浮点数，隐含整数位为1 |
| **无穷大** | E=255(全1) | M=0 | 正无穷/负无穷 |
| **NaN** | E=255(全1) | M≠0 | Not a Number（非法运算结果） |

**规格化数**（E的范围：1 ~ 254）：
- 隐含整数位：1
- 公式：$x = (-1)^S \times 1.M \times 2^{E-127}$
- 单精度范围：约 $\pm 1.18 \times 10^{-38} \sim \pm 3.4 \times 10^{38}$

**非规格化数**（E=0，M≠0）：
- 隐含整数位：0（不是1！）
- 公式：$x = (-1)^S \times 0.M \times 2^{-126}$
- 用于表示非常接近0的数，填补了规格化数到0之间的"间隙"

**零**（E=0，M=0）：
- 有+0（S=0）和-0（S=1）两种

**无穷大**（E=255，M=0）：
- 有正无穷（S=0）和负无穷（S=1）

### 2.4.5 单精度浮点数举例

**例：将十进制数 -66.0 转为IEEE754单精度**

1. 先转二进制：66 = 1000010(B)
2. 规格化：66.0 = 1.000010 × 2^6
3. 符号位 S = 1（负数）
4. 阶码 E = 6 + 127 = 133 = 10000101(B)
5. 尾数 M = 00001000000000000000000（去掉整数1，取23位）

所以：S=1, E=10000101, M=00001000000000000000000
合起来：1 10000101 00001000000000000000000
= 1100 0010 1000 0100 0000 0000 0000 0000
= C2 84 00 00(H)

**例：IEEE754单精度 0x3F800000 表示什么？**

0x3F800000 = 0 01111111 00000000000000000000000

- S = 0（正数）
- E = 01111111 = 127
- M = 000...0

真值 = $(-1)^0 \times 1.0 \times 2^{127-127} = 1.0$

所以 0x3F800000 表示 1.0 ✅

### 2.4.6 不能精确表示的十进制小数

**结论**：**1.2 不能用IEEE754精确表示**（二进制无限循环）。

原因：1.2 = 1 + 0.2
- 1 = 1(B)
- 0.2 = 0.0011001100110011...(B) **无限循环**

所以 1.2(B) = 1.0011001100110011...(无限循环) → 只能近似存储

**能精确表示的十进制数**：
- 1.25（二进制1.01，有限位）
- 2.0（二进制10.0，有限位）
- 2.5（二进制10.1，有限位）
- 0.5, 0.25, 0.125 等（分母为2的幂的分数）

## 2.5 边界对齐与大小端

### 2.5.1 为什么需要边界对齐？

**生活类比**：图书馆的书架。

- **对齐排放**：每本书对应一个固定大小的格子，找书快（效率高）
- **不对齐**：书可以随意放，但找书慢（效率低）

在计算机中：
- **边界对齐**：数据的起始地址必须是数据长度的整数倍
- 如：int（4字节）的地址必须是4的倍数
- 如：short（2字节）的地址必须是2的倍数

**边界对齐（Alignment）规则**：

```
数据的起始地址 = 数据长度（字节数）的整数倍

char（1字节）：地址可以是任意值
short（2字节）：地址必须是2的倍数
int（4字节）：地址必须是4的倍数
double（8字节）：地址必须是8的倍数
```

**为什么需要对齐？**
- 现代CPU一次读取固定长度的数据（如4字节或8字节）
- 如果数据跨边界（不对齐），CPU需要两次内存访问，效率低
- 对齐后一次就能读完

**结构体中的对齐 -- 常考！**

结构体成员按照**定义顺序**依次分配空间，每个成员按其对齐系数（自身大小）对齐，**结构体总大小是最大成员大小的整数倍**。

**例**：
```c
struct {
    char  a;   // 1字节，偏移0
    short b;   // 2字节，偏移2（需2的倍数，跳过1个字节）
    int   c;   // 4字节，偏移4（需4的倍数）
} s;
// sizeof(s) = 8

内存布局：
地址：0   1   2   3   4   5   6   7
     [a] [pad] [  b  ] [     c     ]
```

### 2.5.2 大端和小端

**生活类比**：写信的地址格式。

- **大端（Big-Endian）**：先写国名、再写城市、再写街道（高位在前）
- **小端（Little-Endian）**：先写街道、再写城市、再写国名（低位在前）

**正式定义**- **大端（Big-Endian）**：高位字节存放在低地址（最高有效字节在前）
- **小端（Little-Endian）**：低位字节存放在低地址（最低有效字节在前）

**例**：存储32位整数 0x12345678

```
内存地址递增 → 
大端：  12  34  56  78
小端：  78  56  34  12
```

**如何记忆？**

- **大端**：和书写习惯一致（从左到右，高位到低位），类似人类的阅读习惯
- **小端**：和计算机的运算习惯一致（低位先运算）

**判断大小端的方法**：在C语言中，可以用union来测试。

```c
union {
    int i;
    char c[4];
} u = {0x12345678};
// 如果 u.c[0] == 0x12 → 大端
// 如果 u.c[0] == 0x78 → 小端
```

**x86架构都是小端**，网络协议通常用大端。

### Q25 溢出判断电路

> **题目**：补码加减法运算中，判断溢出的电路最常用的是（ ）
> A. 与门
> B. 或门
> C. 异或门
> D. 非门

**解析**：

补码加减法采用**单符号位+进位判断法**：

溢出条件 = 符号位进位 $C_0$ ⊕ 最高数值位进位 $C_1$

即：$溢出 = C_0 \oplus C_1$

| $C_0$（符号位进位） | $C_1$（最高数位进位） | 异或结果 | 含义 |
|:---:|:---:|:---:|------|
| 0 | 0 | **0** | 无溢出 |
| 0 | 1 | **1** | **正溢出**（正+正=负） |
| 1 | 0 | **1** | **负溢出**（负+负=正） |
| 1 | 1 | **0** | 无溢出 |

- 只有 $C_0 \neq C_1$ 时才溢出，异或门正好实现这个功能
- **异或门 = 判断两个输入是否不同**

因此溢出判断电路的核心是**异或门**。

### Q29 借位/溢出标志

> **题目**：在补码加减法运算器中，借位标志 Borrow 和溢出标志 Overflow 的关系是（ ）
> A. 两者等价
> B. 借位一定溢出
> C. 溢出一定借位
> D. 两者没有必然联系

**解析**：

**借位（Borrow）**：无符号数减法运算中，被减数小于减数时产生借位。
- 关注的是**无符号数**运算
- 表示数值"不够减"

**溢出（Overflow）**：有符号数补码运算中，结果超出表示范围。
- 关注的是**有符号数**运算
- 判断依据是符号位进位⊕最高数位进位

**两者没有必然联系**：

```
例1：有借位，但无溢出
  1111 + 1111 = 1110（无符号：15+15=30，有进位）
  但作为补码：-1 + (-1) = -2，在范围内，无溢出

例2：有溢出，但无借位
  0111 + 0001 = 1000（作为补码：7+1=-8，正溢出）
  无符号：7+1=8，无借位

例3：同时有借位和溢出
  1000 + 1001 = 0001
  无符号：8+9=17，有进位
  补码：-8+(-7)=1（正溢出，负+负=正）
```

所以两者**没有必然联系**。

**答案：D**

### Q31 结构体对齐与小端

> **题目**：在32位小端机中，有以下结构体：
> ```c
> struct {
>     char  a;    // 1字节
>     short b;    // 2字节
>     int   c;    // 4字节
> } s = {0x12, 0x3456, 0x789ABCDE};
> ```
> 若结构体起始地址为0x1000，则地址0x1002中的内容是（ ）
> A. 0x12
> B. 0x34
> C. 0x56
> D. 0xCD

**解析**：

**第一步：分析结构体成员的内存布局（边界对齐）**

```
struct {
    char  a;   // 1字节，对齐系数1，偏移0
    short b;   // 2字节，对齐系数2，偏移必须是2的倍数
    int   c;   // 4字节，对齐系数4，偏移必须是4的倍数
};
```

内存布局：
```
地址    内容      说明
0x1000  0x12      a（char，1字节）
0x1001  [填充]    对齐填充（short b需要2字节对齐，地址0x1002才是2的倍数）
0x1002  0x34      b的高字节 ← 注意：小端模式下
0x1003  0x56      b的低字节
0x1004  0x78      c的最高字节（小端：低位在低地址）
0x1005  0x9A
0x1006  0xBC
0x1007  0xDE      c的最低字节
```

**第二步：理解小端存储**

小端（Little-Endian）：**低位字节在低地址**

对于 short b = 0x3456：
- 低字节 = 0x56 → 存在低地址 0x1003
- 高字节 = 0x34 → 存在高地址 0x1002

所以地址0x1002中存放的是b的高字节 = 0x34

**答案：B**

### Q32 IEEE754精确表示

> **题目**：以下哪个十进制数可以用IEEE754单精度浮点数精确表示（ ）
> A. 1.2
> B. 1.25
> C. 1.3
> D. 1.1

**解析**：

判断标准：十进制小数能否用**有限位二进制小数**表示。

关键在于：分母能否表示为2的幂（即分母是2、4、8、16...）。

**A. 1.2** = 1 + 0.2 = 1 + 1/5
- 0.2 = 0.001100110011...（二进制无限循环）
- 因为分母5不是2的幂
- **不能精确表示** ❌

**B. 1.25** = 1 + 0.25 = 1 + 1/4
- 0.25 = 0.01(二进制) ✓
- 分母4 = 2^2，有限位
- **能精确表示** ✅

**C. 1.3** = 1 + 0.3 = 1 + 3/10
- 0.3 = 0.010011001100...（二进制无限循环）
- 因为分母10不是2的幂
- **不能精确表示** ❌

**D. 1.1** = 1 + 0.1 = 1 + 1/10

- 因为分母10不是2的幂

#### Q175 补码表示个数

> **题目**：8位补码可以表示多少个不同的整数（ ）
> A. 255
> B. 256
> C. 254
> D. 128

**解析**：

8位补码的表示范围：-128 ~ 127

总共：128 + 127 + 1 = **256**个不同的整数

**对比**：
- 8位原码：255个（0占两个）
- 8位补码：256个（0唯一）
- 8位无符号：256个（0 ~ 255）

- 原码/反码：0有重复 → 2^N - 1个

#### Q176 补码加法

> **题目**：用8位补码计算 64 + 65，结果和溢出情况是（ ）
> A. 129，溢出
> B. 129，无溢出
> C. 127，无溢出
> D. -127，溢出

**解析**：

64 + 65 = 129，但8位补码最大值是127

```
 64 = 0100 0000
+65 = 0100 0001
    ----------
      1000 0001
```

结果的符号位是1（负数），但两个正数相加不应该得到负数 → **正溢出** ✅

结果 1000 0001 作为补码表示 -127（不是129）

溢出判断：
- $C_1$（最高数位进位）= 1（数值位最高位相加有进位）
- $C_0$（符号位进位）= 0（+ + +，符号位没有进位）
- $溢出 = C_0 \oplus C_1 = 0 \oplus 1 = 1$ → 溢出 ✅

**答案：A**

#### Q177 补码减法

> **题目**：用8位补码计算 20 - 30，结果是（ ）
> A. -10
> B. -9
> C. -11
> D. 10

**解析**：

20 - 30 = 20 + (-30)

```
[20]补 = 0001 0100
[30]补 = 0001 1110
[-30]补 = [30]补取反加1 = 1110 0001 + 1 = 1110 0010

[20]补 + [-30]补 = 0001 0100 + 1110 0010 = 1111 0110
```

1111 0110 是补码，最高位1 → 负数
- 取反：1000 1001
- 加1：1000 1010 = -10

所以结果是 -10 ✅

**答案：A**

#### Q178 双符号位溢出判断

> **题目**：双符号位补码（模4补码）中，运算结果的双符号位为10，表示（ ）
> A. 正溢出
> B. 负溢出
> C. 无溢出（正数）
> D. 无溢出（负数）

**解析**：

双符号位判断规则：

| 双符号位 | 含义 |
|:------:|------|
| 00 | 正数，无溢出 |
| 01 | **正溢出**（两个正数相加，结果变负数） |
| 10 | **负溢出**（两个负数相加，结果变正数） |
| 11 | 负数，无溢出 |

10 → 最高位1，次高位0 → 负溢出

**记忆口诀**："00正数无溢出，01负正溢；11负数无溢出，10正负溢"

具体含义：
- 01：正确符号位0，但出现了进位→**正溢出**
- 10：正确符号位1，但出现了进位→**负溢出**

**答案：B**

#### Q179 符号位扩展

> **题目**：将8位补码 10110011 扩展为16位，结果是（ ）
> A. 0000000010110011
> B. 1111111110110011
> C. 1111111110110010
> D. 0000000010110100

**解析**：

补码符号位扩展规则：
- **负数补码**：高位补1

8位补码 10110011，最高位1 → 负数

扩展为16位：高位补8个1

```
10110011 → 11111111 10110011
```

验证：这个数是否真的是负数？
- 8位补码 10110011，数值 = -(0100 1100 + 1) = -0100 1101 = -77
- 16位补码 11111111 10110011，数值 = -(00000000 01001100 + 1) = -00000000 01001101 = -77 ✅

**答案：B**

#### Q180 补码求负

> **题目**：已知 [X]补 = 11010011，则 [-X]补 为（ ）
> A. 00101101
> B. 00101100
> C. 11010011
> D. 10101101

**解析**：

由 [X]补 求 [-X]补 的规则：**连同符号位取反，末位加1**

```
[X]补 = 1101 0011
取反  = 0010 1100（连同符号位！）
加1   = 0010 1101
```

所以 [-X]补 = 0010 1101

**注意**：是"连同符号位取反"，不是"符号位不变"！这要和"补码转原码"区分开。

**对比记忆**：
- 补码求原码：符号位不变，数值位取反加1
- 补码求[-X]补：连同符号位取反，末位加1

**答案：A**

#### Q181 原码乘法符号处理

> **题目**：原码乘法中，乘积的符号位由两个乘数的符号位如何决定（ ）
> A. 相加
> B. 相乘
> C. 异或
> D. 同或

**解析**：

原码乘法规则：
1. **符号位单独处理**：乘积符号 = 乘数符号 ⊕ 被乘数符号（**异或**运算）
2. **数值部分**：绝对值相乘

为什么用异或？
- 正 × 正 = 正：0⊕0=0 ✅
- 正 × 负 = 负：0⊕1=1 ✅
- 负 × 正 = 负：1⊕0=1 ✅
- 负 × 负 = 正：1⊕1=0 ✅

异或正好实现"相同为正，不同为负"的符号规则。

**答案：C**

#### Q182 移位运算

> **题目**：补码的算术右移，高位补入的是（ ）
> A. 0
> B. 1
> C. 符号位
> D. 最低位

**解析**：

算术右移的规则：**高位补符号位**，低位丢失。

- 正数补码（符号位0）：高位补0 → 相当于除以2后向下取整
- 负数补码（符号位1）：高位补1 → 相当于除以2后向下取整

| 移位类型 | 规则 | 用途 |
|---------|------|------|
| 算术左移 | 低位补0，高位丢失（符号位变化则溢出） | 乘以2 |
| 算术右移 | 高位**补符号位**，低位丢失 | 除以2 |
| 逻辑右移 | 高位**补0**，低位丢失 | 无符号数移位 |

**例**：
```
[-66]补 = 1011 1110
算术右移1位：1101 1111（高位补1=符号位）
验证：-66/2 = -33，而 1101 1111 正是 [-33]补 ✅
```

**答案：C**

#### Q183 补码左移溢出判断

> **题目**：补码左移时，发生溢出的条件是（ ）
> A. 符号位变化
> B. 符号位与最高数位不同
> C. 最高位丢失
> D. 移出1

**解析**：

补码左移溢出的判断标准：**移位后符号位发生变化**。

具体来说：移位前符号位 ≠ 移位后符号位 → 溢出

或者等价地：**符号位与最高数位不同时，左移会溢出**。

```　　
例1：无溢出
   [66]补 = 0100 0010
   左移1位 = 1000 0100（此时符号位=1，与原来不同！）
   但仔细看：符号位从0变1，但这个变化是因为进位导致的
   实际上左移前符号位=0 ≠ 最高数位=1 → 左移后会溢出

   验证：66×2=132，超出8位补码范围-128~127 → 确实溢出
    
例2：无溢出
   [32]补 = 0010 0000
   左移前：符号位=0，最高数位=0（相同）
   左移后：0100 0000 = 64（32×2，在范围内）
   符号位不变，无溢出 ✅
```

**答案：A**（符号位变化）

更精确地说：**左移前，符号位 ≠ 最高数位 → 左移后符号位变化 → 溢出**

但题目中A选项"符号位变化"是最直接的判断标准。

**答案：A**

#### Q185 IEEE754真值计算

> **题目**：IEEE754单精度浮点数 0x40400000 的十进制真值是（ ）
> A. 3.0
> B. 4.0
> C. 2.5
> D. 3.5

**解析**：

0x40400000 = 0100 0000 0100 0000 0000 0000 0000 0000

分解：
```
S = 0（正数）
E = 10000000 = 128
M = 10000000000000000000000
```

真值 = $(-1)^0 \times 1.1 \times 2^{128-127} = 1.1 \times 2^1 = 11.0(B) = 3.0$

**关键步骤**：
- E = 128，减去偏置值127，指数 = 1
- 尾数 M = 100...0，实际尾数 = 1.1（二进制）
- 1.1(B) = 1 + 0.5 = 1.5
- 1.5 × 2 = 3.0

**答案：A**

> A. 阶码位数越多，精度越高
> B. 尾数位数越多，表示范围越大
> C. 阶码位数越多，表示范围越大

#### Q191 小端存储

> **题目**：小端存储方式中，32位整数 0x12345678 在内存中从低地址到高地址的字节顺序是（ ）
> A. 12 34 56 78
> B. 78 56 34 12
> C. 78 34 56 12
> D. 12 78 56 34

**解析**：

小端（Little-Endian）：**低位字节在低地址**。

0x12345678 的字节分解：
- 最高字节（Byte3）= 0x12
- 次高字节（Byte2）= 0x34
- 次低字节（Byte1）= 0x56
- 最低字节（Byte0）= 0x78

小端存储（从低地址到高地址）：
```
地址 n+0: 78（最低字节）
地址 n+1: 56
地址 n+2: 34
地址 n+3: 12（最高字节）
```

所以顺序是：**78 56 34 12**

**对比记忆**：
- **大端**（Big-Endian）：12 34 56 78（和书写习惯相同，高字节在低地址）
- **小端**（Little-Endian）：78 56 34 12（低字节在低地址）

**答案：B**
