## 1.1 计算机系统的组成

### 1.1.1 完整的计算机系统 = 硬件 + 软件

**生活类比**：把计算机看作一个人。
- **硬件** = 人的身体（骨骼、肌肉、五官）-- 看得见摸得着的部分
- **软件** = 人的知识、经验、思维 -- 让身体能做事的"灵魂"

缺了硬件，软件无处安放；缺了软件，硬件只是一堆废铁。**两者缺一不可**。

```
完整的计算机系统
├── ⚙️ 硬件系统
│   ├── 运算器
│   ├── 控制器
│   ├── 存储器
│   ├── 输入设备
│   └── 输出设备
└── 📦 软件系统
    ├── 系统软件（操作系统、编译程序等）
    └── 应用软件（游戏、浏览器等）
```

**⚠️ 考点：完整的计算机系统 = 硬件系统 + 软件系统**（不是只有硬件，也不是只有主机）

---

### 1.1.2 冯·诺依曼计算机

冯·诺依曼是计算机界的"祖师爷"。他在 1945 年提出的**存储程序**思想，至今仍然是几乎所有通用计算机的基石。

#### (1) 存储程序思想 -- 核心中的核心

**生活类比**：做菜。
- 传统方式：师傅在旁边一步步指导你（边想边做）
- 存储程序方式：师傅先把**整本菜谱**写在纸上，你拿着菜谱一步步照着做

菜谱 = 程序（指令的集合）；纸 = 存储器；你 = CPU

**关键点**：指令和数据**以二进制形式存放在存储器中，形式上没有差别**。

```
存储器中的一段二进制：10110011 01001110
它究竟是指令还是数据？
→ CPU在"取指阶段"把它当指令
→ CPU在"执行阶段"把它当数据
→ 依据：指令周期的不同阶段 ⚠️
```

#### (2) 冯·诺依曼机的五大部件

- **运算器**：负责算术运算和逻辑运算（像计算器）
- **控制器**：负责指挥协调，从存储器取指令、分析指令、控制执行（像指挥官）
- **存储器**：存放指令和数据（像仓库）
- **输入设备**：把外部信息送进计算机（键盘、鼠标）
- **输出设备**：把结果展示出来（显示器、打印机）

#### (3) 基本特点总结

| 特点 | 说明 |
|------|------|
| **控制流驱动** | 按指令序列依次执行（不是数据驱动，也不是请求驱动） |
| **按地址访问** | 指令和数据都通过地址来读写 |
| **顺序执行指令** | 默认一条接一条执行（除非遇到跳转指令） |
| **存储程序** | 程序和数据预先存入存储器 |
| **二进制表示** | 指令和数据都用0和1表示 |

**⚠️ 区分指令和数据的依据**：**指令周期的不同阶段**
- 取指阶段 → 从存储器读出的内容当作**指令**，送入IR
- 执行阶段 → 从存储器读出的内容当作**数据**，送入运算器

---

### 1.1.3 运算器的组成

**运算器包含**：
- ⭕ **ALU**（算术逻辑单元）-- 真正做计算的部件
- ⭕ **状态寄存器**（标志寄存器）-- 记录运算结果的状态（是否溢出、是否为0等）
- ⭕ **数据总线** -- 传输数据的通路

**运算器不包含**：
- ❌ **地址寄存器** -- 地址寄存器属于CPU的**控制器**部分，用于存放要访问的内存地址

> 记忆口诀：运算器做"算术" -- ALU算数，状态寄存器记结果，数据总线传数据。**地址**的事情归控制器管。

## 1.2 计算机的工作层次与语言

### 1.2.1 三种语言层次

**生活类比**：不同层次的沟通方式。

| 语言层次 | 类比 | 特点 |
|---------|------|------|
| **机器语言** | 0和1组成的"机器脑电波" | 计算机唯一能直接"听懂"的语言 |
| **汇编语言** | 给机器语言起的英文缩写"外号" | 一一对应机器指令，但和人更亲近一点 |
| **高级语言** | 我们说的人话（C、Java、Python） | 最接近人类思维方式，但计算机听不懂 |

**层次关系**：

```
高级语言（C/Java）    ← 人类最容易写
     ↓ 编译/解释
汇编语言              ← 用助记符（ADD、MOV等）
     ↓ 汇编
机器语言（二进制）     ← 计算机唯一可直接执行 ⚠️
```

---

### 1.2.2 编译程序 vs 解释程序

高级语言不能被计算机直接执行，需要翻译。翻译方式有两种：

**生活类比**：把一本英文小说变成中文电影。

| 方式 | 类比 | 过程 | 特点 |
|------|------|------|------|
| **编译程序** | 先把整本书翻译成中文剧本，再拍电影 | 全部翻译 → 生成目标文件 → 执行 | 一次翻译，执行**快** |
| **解释程序** | 找个翻译在旁边，说一句翻一句 | 边翻译边执行 | 不生成目标文件，执行**慢** |

**⚠️ 考点**：
- **编译程序**：一次翻译全部源代码 → 生成目标代码文件 → 运行速度快
  - 代表语言：C、C++
- **解释程序**：逐句翻译执行 → 不生成目标文件 → 运行速度慢
  - 代表语言：JavaScript、Python

---

### 1.2.3 从高级语言到可执行文件的四个阶段

**生活类比**：做一道复杂的菜。
1. **预处理** → 洗菜、切菜、准备调料（处理 #include、宏定义等）
2. **编译** → 按菜谱步骤做出半成品（高级语言 → 汇编语言）
3. **汇编** → 把半成品装盘（汇编语言 → 机器语言目标文件）
4. **链接** → 把各个盘子组合成完整宴席（多个目标文件 → 可执行文件）

**⚠️ 记忆口诀**："预编译汇链"（预 → 编 → 汇 → 链）

```
源代码(.c/.cpp)
    ↓ 预处理（处理 #define、#include 等）
预处理后的源文件(.i)
    ↓ 编译（高级语言 → 汇编语言）
汇编文件(.s/.asm)
    ↓ 汇编（汇编语言 → 机器语言）
目标文件(.o/.obj)
    ↓ 链接（多个目标文件 + 库文件 → 可执行文件）
可执行文件(.exe)
```

### Q10 冯·诺依曼思想错误选项

> **题目**：下列关于冯·诺依曼思想的描述，错误的是（ ）
> A. 采用存储程序方式
> B. 指令和数据用二进制表示

### Q11 编译链接过程

> **题目**：将高级语言程序转换为可执行文件，正确的顺序是（ ）
> A. 编译 → 汇编 → 预处理 → 链接
> B. 预处理 → 编译 → 汇编 → 链接
> C. 预处理 → 汇编 → 编译 → 链接
> D. 编译 → 预处理 → 汇编 → 链接

**解析**：
- 正确顺序是：**预处理 → 编译 → 汇编 → 链接**
- **预处理**：处理 #define（宏定义）、#include（文件包含）等预处理指令
- **编译**：将预处理后的源文件翻译成**汇编语言**（高级 → 汇编）
- **汇编**：将汇编语言翻译成**机器语言目标文件**（汇编 → 机器）
- **链接**：将多个目标文件和库文件组合成**可执行文件**

> 记忆口诀：**"预编译汇链"** -- 预习、编译、汇编、串链

**答案：B**

## 1.3 计算机的性能指标

### 1.3.1 三种"字长" -- 最容易混淆的概念

**生活类比**：三条不同宽度的公路。

| 字长类型 | 定义 | 类比 |
|---------|------|------|
| **机器字长** | CPU一次能处理的**数据**位数（通用寄存器宽度） | 一次能拉多少货 |
| **指令字长** | 一条指令的二进制位数 | 一张送货单有多长 |
| **存储字长** | 一个存储单元中的二进制位数 = **MDR的位数** | 一个仓库隔间的大小 |

**⚠️ 核心考点：三者数值上可以相等也可以不等！**

也就是说：CPU一次处理64位数据（机器字长），但一条指令可能是32位（指令字长），一个存储单元也可能是32位（存储字长）-- 这完全正常。

#### 机器字长（最重要）

- CPU**运算器**的位数
- = **通用寄存器**的宽度
- = 数据总线的宽度（通常）
- 决定了计算机的**运算精度**
- 常见值：16位、32位、64位

> 注意区分：
> - **机器字长**：CPU处理数据的宽度
> - **指令字长**：一条指令的二进制位数（和机器字长无关）
> - **存储字长**：一个存储单元的位数 = MDR的位数

**⚠️ 易错**：计算机字长**不一定都是32位**，有16位、32位、64位等。

---

### 1.3.2 时钟相关概念

#### 时钟周期（Clock Cycle）

**生活类比**：心跳。
- 心脏每跳动一次 = 一个时钟周期
- 心跳越快，完成动作越快

**定义**：CPU的最小时间单位，是CPU工作的"节拍"。

```
时钟周期 = 1 / 主频
```

#### 主频（Clock Frequency）

**定义**：时钟频率，即每秒有多少个时钟周期。

单位：**Hz**（赫兹）

| 单位 | 换算 | 举例 |
|------|------|------|
| 1 Hz | 1次/秒 | -- |
| 1 MHz | 10^6 Hz | 早期的CPU |
| 1 GHz | 10^9 Hz | 现代CPU（如3.0GHz） |

**例如**：主频 2GHz = 2 × 10^9 Hz
时钟周期 = 1 / (2 × 10^9) = 0.5 ns（纳秒）

**⚠️ 时钟周期是计算机操作的最小时间单位**

---

### 1.3.3 CPI -- 平均执行一条指令需要几个时钟周期

**生活类比**：做一道菜需要几个步骤。
- CPI = 平均做一道菜所需的步骤数
- 不同"菜"（指令）需要的步骤不同
  - 简单的菜（空操作 NOP）：1步
  - 复杂的菜（乘法 MUL）：可能5步

**CPI = 总时钟周期数 / 指令条数**

**⚠️ CPI与什么无关？ -- 与时钟频率（主频）无关！**

CPI由**系统结构**、**指令集**、**计算机组织**决定，但不取决于时钟跑多快。

> 就好比：做一道菜需要几个步骤（CPI）和你手脚多快（主频）是两回事。
> 步骤固定了，不论你动作多快，步骤数不变。

---

### 1.3.4 MIPS -- 每秒百万条指令

**定义**：Million Instructions Per Second

```
MIPS = 主频（MHz）/ CPI
```

**生活类比**：你的工作效率。
- 主频 = 你每秒能眨眼的次数（节拍数）
- CPI = 做一件事需要眨几次眼
- MIPS = 你每秒能做多少件事

**单位注意**：主频用 MHz（兆赫兹），计算时 1GHz = 1000MHz

---

### 1.3.5 MFLOPS -- 科学计算的王者

**定义**：Million Floating-point Operations Per Second，每秒百万次浮点运算。

**⚠️ 重点：科学计算最看重的指标是MFLOPS，不是MIPS！**

为什么？
- 科学计算主要做大量的浮点运算（小数运算），而不是整数运算
- MIPS衡量的是指令执行速度，但不同的指令"含金量"不同
- MFLOPS专门衡量**浮点运算能力**，对科学计算更有意义

| 指标 | 适用场景 |
|------|---------|
| MIPS | 通用性能衡量 |
| **MFLOPS** | **科学计算**、工程仿真 |

### Q12 字长相关错误

> **题目**：下列关于字长的说法，错误的是（ ）
> A. 机器字长是指CPU一次能处理的二进制数据位数
> B. 存储字长是指一个存储单元的二进制位数
> C. 机器字长、指令字长、存储字长三者必须相等
> D. 指令字长是指一条指令的二进制位数

**解析**：
- A 正确：机器字长 = CPU运算器的位数 = 通用寄存器宽度
- B 正确：存储字长 = MDR的位数
- ❌ **C 错误：三者数值上可能相等也可能不等，不必须相等！** -- 这是最常考的易错点
- D 正确：指令字长就是一条指令的二进制位数

**答案：C**

### Q13 机器字长/指令字长/存储字长

> **题目**：某计算机的机器字长为32位，指令字长为16位，存储字长为8位，以下说法正确的是（ ）
> A. 三者必须统一，设计不合理
> B. CPU一次能够处理32位数据
> C. 每个存储单元有32位
> D. 指令字长必须等于机器字长

**解析**：
- A 错误：**三者可以不等**，这是常见的设计
- ⭕ **B 正确：机器字长32位 = CPU一次能处理32位数据**
- C 错误：存储字长8位 = 每个存储单元8位，不是32位
  > 注意：机器字长和存储字长没有必然的大小关系
- D 错误：指令字长可以小于、等于或大于机器字长

**答案：B**

> **题目**：以下哪个指标最适合衡量计算机的科学计算性能（ ）
> A. CPI
> B. MIPS
> C. MFLOPS
> D. 主频

## 1.4 性能计算（必考！）

### 1.4.1 三大公式

#### 公式一：执行时间

```
执行时间 = 指令条数 × CPI / 主频
```

**生活类比**：完成一项任务的耗时。
- 指令条数 = 多少个工作任务
- CPI = 每个任务需要几步
- 主频 = 每秒能做几步

```
执行时间 = (工作量 × 每步耗时) / 速度
         = 指令条数 × (CPI / 主频)
```

#### 公式二：平均CPI

当程序的指令有不同种类时，需要计算加权平均：

```
平均CPI = Σ (CPI_i × 比例_i)
```

#### 公式三：MIPS

```
MIPS = 主频（MHz）/ 平均CPI
```

### 1.4.2 性能变化计算（必考计算题型）

#### 场景一：CPU速度提高

公式：
```
新执行时间 = CPU时间 / 提速倍数 + I/O时间
```

注意：**I/O时间不受CPU提速影响**，只有CPU计算时间会缩短。

#### 场景二：编译优化

公式：
```
新执行时间 = 原时间 × 新指令比例 × 新CPI比例
```

### Q15 对用户透明的寄存器

> **题目**：以下哪个寄存器对程序员（用户）是透明的（不可见的）（ ）
> A. 通用寄存器
> B. 程序计数器（PC）
> C. 指令寄存器（IR）
> D. 状态寄存器

**解析**：

首先理解"透明"的含义：在计算机中，**"透明"意味着"不可见"**，跟日常生活中的"透明"正好相反。

| 寄存器 | 对程序员可见性 | 说明 |
|--------|--------------|------|
| 通用寄存器 | **可见** | 程序员可以显式读写 |
| PC（程序计数器） | **可见** | 程序员可以访问（如跳转指令） |
| 状态/标志寄存器 | **可见** | 程序员可以读取标志位 |
| **IR（指令寄存器）** | **透明（不可见）** | 存放当前正在执行的指令，程序员无需关心 |
| MAR / MDR | **透明（不可见）** | 与主存交互的中间寄存器 |
| 微指令寄存器 | **透明（不可见）** | 微程序控制器内部使用 |

- A 通用寄存器：可见 ✅
- B PC：可见 ✅
- ❌ **C 指令寄存器（IR）：透明（不可见）**
- D 状态寄存器：可见 ✅

**答案：C**

### Q16 最小时间单位

> **题目**：微型计算机操作的最小时间单位是（ ）
> A. 指令周期
> B. 机器周期
> C. 时钟周期
> D. 存取周期

**解析**：

首先理清几个"周期"的关系：

```
指令周期 > 机器周期 > 时钟周期
```

- **指令周期**：从取指令到执行完这条指令的时间（最长）
- **机器周期**：由存取周期决定，通常为一次内存访问所需时间
- **时钟周期**：主频的倒数，CPU工作的基本节拍
- **存取周期**：存储器进行一次完整读写操作所需时间

- A 指令周期：不是最小单位，它包含多个机器周期
- B 机器周期：不是最小单位，它包含多个时钟周期
- ⭕ **C 时钟周期：CPU的最小时间单位**
  > 时钟周期 = 1/主频，是CPU一切操作的"心跳"节奏
- D 存取周期：是存储器的性能参数，不是CPU的操作单位

**答案：C**

### Q17 CPI影响因素

> **题目**：以下哪个因素会影响CPU的CPI（ ）
> A. 时钟频率
> B. 系统结构
> C. 主存容量
> D. 硬盘速度

**解析**：

**⚠️ CPI与时钟频率（主频）无关！** 这是最重要的考点之一。

CPI（Cycles Per Instruction）取决于：
- ✅ **系统结构**（是否采用流水线、超标量等）
- ✅ **指令集**（RISC vs CISC，指令复杂度不同）
- ✅ **计算机组织**（硬件实现方式）

CPI与以下因素**无关**：
- ❌ 时钟频率（主频）
- ❌ 主存容量
- ❌ 硬盘速度

- A ❌ 时钟频率：**不相关**！CPI是"步骤数"，时钟频率是"每秒步骤数"，两者独立
- ⭕ **B 系统结构：相关**，不同的系统结构导致CPI不同
- C ❌ 主存容量：不直接影响每个指令的时钟周期数
- D ❌ 硬盘速度：不直接影响

**答案：B**

### Q18 CPU速度提高计算

> **题目**：某程序在计算机上运行，CPU时间占90s，I/O时间占10s，总共100s。若CPU速度提高50%，则新的总执行时间为（ ）
> A. 70s
> B. 66.7s
> C. 75s
> D. 60s

**解析**：

这是经典题型，关键要点：
1. **CPU速度提高50%** → 原来的1.5倍快 → CPU时间**缩短**，而不是变长
2. **I/O时间不受CPU提速影响**

计算过程：
```
原CPU时间 = 90s
CPU速度提高50% → 新CPU时间 = 90 / 1.5 = 60s
I/O时间不变 = 10s
新总时间 = 60 + 10 = 70s
```

> 常见错误：90 × 0.5 = 45，然后 90 - 45 = 45 + 10 = 55 ❌
> 正确理解：速度提高50%"意味着"变成原来的1.5倍"，时间是原来的1/1.5

**答案：A**

### Q19 MIPS计算

> **题目**：某CPU主频为1.2GHz，各类指令的CPI和比例如下：
> - A类指令：CPI=2，占50%
> - B类指令：CPI=3，占20%
> - C类指令：CPI=4，占10%
> - D类指令：CPI=5，占20%
>
> 则该CPU的平均CPI和MIPS分别是（ ）
> A. CPI=3, MIPS=400
> B. CPI=3.5, MIPS=342
> C. CPI=2.8, MIPS=428
> D. CPI=3, MIPS=360

**解析**：

**第一步：计算平均CPI**

平均CPI = Σ（CPI_i × 比例_i）

```
平均CPI = 2×50% + 3×20% + 4×10% + 5×20%
        = 2×0.5 + 3×0.2 + 4×0.1 + 5×0.2
        = 1.0 + 0.6 + 0.4 + 1.0
        = 3.0
```

**第二步：计算MIPS**

先换算主频：1.2GHz = 1200MHz

```
MIPS = 主频(MHz) / 平均CPI
     = 1200 / 3
     = 400
```

**答案：A**

### Q20 编译优化执行时间

> **题目**：某程序P在原编译器下执行时间为20s。经过编译优化后，指令条数减少到原来的70%，但CPI增加到原来的1.2倍。假设主频不变，问优化后的执行时间为（ ）
> A. 14.0s
> B. 16.8s
> C. 24.0s
> D. 11.7s

**解析**：

利用公式：执行时间 = 指令条数 × CPI / 主频

主频不变 → 执行时间与（指令条数 × CPI）成正比

```
原执行时间 = 原指令条数 × 原CPI / 主频 = 20s

新指令条数 = 原指令条数 × 70% = 原指令条数 × 0.7
新CPI = 原CPI × 1.2

新执行时间 = 新指令条数 × 新CPI / 主频
           = (原指令条数 × 0.7) × (原CPI × 1.2) / 主频
           = 原执行时间 × 0.7 × 1.2
           = 20 × 0.7 × 1.2
           = 20 × 0.84
           = 16.8s
```

> 注意：指令数减少到70% × CPI增大到1.2倍 = 0.7 × 1.2 = 0.84，所以时间缩短到原来的84%

**答案：B**

### Q21 不同主频CPI比较

> **题目**：有两台计算机M1和M2，M1主频1.5GHz，CPI=2；M2主频1.2GHz，CPI=1。运行同一程序，比较两台计算机的执行时间（ ）
> A. M1比M2快
> B. M2比M1快
> C. 一样快
> D. 无法比较

**解析**：

运行**同一程序** → 指令条数相同，设为N

**计算M1的执行时间**：
```
T1 = N × CPI1 / 主频1
   = N × 2 / (1.5 × 10^9)
   = N × 1.33 × 10^(-9)
```

**计算M2的执行时间**：
```
T2 = N × CPI2 / 主频2
   = N × 1 / (1.2 × 10^9)
   = N × 0.83 × 10^(-9)
```

对比：
```
T1 / T2 = (2/1.5) / (1/1.2) = 1.333 / 0.833 = 1.6

M2的执行时间是M1的1/1.6 = 62.5%
M2比M1快！(M1时间 : M2时间 = 1.6 : 1)
```

> 速算方法：
> M1时间 : M2时间 = (CPI1/主频1) : (CPI2/主频2)
> = (2/1.5) : (1/1.2)
> = 1.333 : 0.833
> = 1.6 : 1
>
> 所以M2更快

**扩展思考**：主频高不一定快！M1主频1.5GHz > M2主频1.2GHz，但M2的CPI更低，反而跑得更快。

**答案：B**

## 1.5 寄存器及其可见性

### 寄存器的分类（按程序员视角）

| 类别 | 寄存器 | 对程序员 |
|------|--------|---------|
| **可见** | 通用寄存器 | 可直接读写 |
| **可见** | PC（程序计数器） | 可通过跳转指令等访问 |
| **可见** | 状态/标志寄存器 | 可读取标志位 |
| **透明** | IR（指令寄存器） | **不可见**，存放当前指令 |
| **透明** | MAR / MDR | **不可见**，与主存交互 |
| **透明** | 微指令寄存器 | **不可见**，微程序控制用 |

**记忆方法**：
- 程序员**需要知道**的 → 可见（通用寄存器、PC、状态寄存器）
- 程序员**不需要关心**的 → 透明（IR、MAR、MDR）
