# 5.1 指令格式

## 5.1.1 指令的基本构成

每一条机器指令由两部分组成：

```
指令 = 操作码 + 地址码
```

| 字段 | 作用 | 说明 |
|------|------|------|
| **操作码 (OP)** | 指明指令要做什么操作 | 如：加法、减法、传送、转移 |
| **地址码 (A)** | 指明操作数在哪里 | 可以是数据本身、地址、寄存器编号等 |

**类比**：操作码是"动词"，地址码是"宾语"。
- `ADD R1, R2` → 操作码=ADD（加），地址码=R1（目标）, R2（源）

---

## 5.1.2 指令的分类（按地址码个数）

根据指令中包含的地址码个数，指令分为四类：

### (1) 零地址指令

```
格式：OP
```

- 指令中只有操作码，没有地址码
- 适用场景：
  - **空操作指令**（NOP）：什么都不做
  - **停机指令**（HLT）：停止CPU
  - **堆栈操作**：操作数隐含在栈顶（如 POP, PUSH 在某些架构中）

### (2) 一地址指令

```
格式：OP + A1
```

- 只有一个地址码
- 两种典型用法：
  - **单目运算**：`OP (A1) → A1` 如：取反 NEG R1、自增 INC R1
  - **双目运算（隐含约定）**：`(ACC) OP (A1) → ACC`，另一个操作数隐含在累加器ACC中

> **重要理解**：一地址指令也能处理双操作数运算，只是第二个操作数隐含在ACC中。

### (3) 二地址指令

```
格式：OP + A1 + A2
```

- 最常用的指令格式
- 操作：`(A1) OP (A2) → A1`（A1既是源也是目的）
- 例：`ADD R1, R2` → R1 = R1 + R2

### (4) 三地址指令

```
格式：OP + A1 + A2 + A3
```

- 三个地址码：两个源操作数 + 一个目的操作数
- 操作：`(A1) OP (A2) → A3`
- 例：`ADD R1, R2, R3` → R3 = R1 + R2
- 优点：源操作数不会被破坏
- 缺点：指令字长长，占用空间大

### 对比总结

| 类型 | 地址码数 | 操作数来源 | 结果存放 | 示例 |
|------|---------|-----------|---------|------|
| 零地址 | 0 | 隐含（栈顶） | 隐含 | NOP, HLT |
| 一地址 | 1 | A1 + (隐含ACC) | A1 或 ACC | INC R1 |
| 二地址 | 2 | A1 + A2 | A1（覆盖） | ADD R1, R2 |
| 三地址 | 3 | A1 + A2 | A3（保留源） | ADD R1, R2, R3 |

> **发展趋势**：RISC架构普遍采用**三地址指令**格式。

## 5.1.3 定长指令字 vs 变长指令字

### 定长指令字

- 所有指令的**长度相同**
- 优点：控制简单，取指时间固定，易于流水线
- 缺点：浪费空间（短指令也被分配长指令的空间）
- 典型：**RISC架构**普遍采用定长指令（如ARM 32位定长）

### 变长指令字

- 不同指令长度可以不同
- 优点：节省存储空间（短指令用短编码）
- 缺点：控制复杂，取指时间不固定
- 典型：**CISC架构**（x86指令长度1~15字节不等）

### 核心考点

> **指令字长 ≠ 机器字长 ≠ 存储字长**（三者可以相等，也可以不等）

## 5.1.4 操作码扩展技术

### 问题背景

指令字长有限，如果操作码固定为 n 位，最多只能表示 $2^n$ 种指令。但实际需要的指令条数往往超过 $2^n$，怎么办？

### 核心思想

**短操作码留出一些编码作为"扩展窗口"，用更长的操作码表示更多指令。**

### 工作原理图解

假设指令字长16位，操作码从4位开始：

```
第一层（4位操作码）：
0000 ~ 1110   → 15条三地址指令（操作码4位）
1111          → 扩展窗口（操作码扩展到8位）

第二层（8位操作码，前4位=1111）：
1111 0000 ~ 1111 1110 → 15条二地址指令
1111 1111              → 继续扩展（操作码扩展到12位）

第三层（12位操作码，前8位全1）：
1111 1111 0000 ~ 1111 1111 1110 → 15条一地址指令
1111 1111 1111                   → 继续扩展

第四层（16位操作码，前12位全1）：
1111 1111 1111 0000 ~ 1111 1111 1111 1111 → 16条零地址指令
```

### 关键原则

1. **短操作码的编码不能是长操作码的前缀**
2. 短操作码会预留一些编码作为扩展标识
3. 每扩展一层，操作码位数增加一个固定长度

### 例题：操作码扩展

> **题目**：某计算机指令字长16位，地址码4位。要求设计三地址指令15条、二地址指令15条、一地址指令15条、零地址指令16条。是否可行？

**解析**：

| 层级 | 操作码位数 | 地址码位数 | 指令条数 | 计算过程 |
|------|-----------|-----------|---------|---------|
| 三地址 | 4 | 4+4+4=12 | 15条 | $2^4-1=15$，留1个编码扩展 |
| 二地址 | 8 | 4+4=8 | 15条 | 1×$(2^4-1)=15$，留1个编码扩展 |
| 一地址 | 12 | 4 | 15条 | 1×$(2^4-1)=15$，留1个编码扩展 |
| 零地址 | 16 | 0 | 16条 | 1×$2^4=16$ |

**答案**：可行。正满足设计要求，每层都恰好用满。

### 解题套路

**给定指令条数，判断是否可行的步骤：**

1. 确定基础操作码位数
2. 从指令条数最多的（地址码最多的）开始分配
3. 每条指令消耗 $2^{剩余地址码位数}$ 个编码空间
4. 检查是否够用

# 5.2 寻址方式（必考重点！）

## 什么是寻址方式？

寻址方式就是**指令如何找到操作数**的方式。地址码字段给出的不是完整地址，而是一个"形式地址"，需要经过某种变换才能得到"有效地址（EA）"。

## 九大寻址方式详解

### 5.2.1 立即寻址（Immediate Addressing）

| 项目 | 内容 |
|------|------|
| **操作数位置** | 指令中**直接给出** |
| **有效地址** | 无（操作数就是地址码字段本身） |
| **优点** | 最快，无需访存 |
| **缺点** | 操作数大小受限（字段位数有限） |
| **应用** | 给寄存器赋常数值 |

```
指令:   ADD R1, #100
        操作数100直接放在指令中
```

> **注意**：立即寻址的"操作数"在指令中，不需要访问内存就能拿到。

### 5.2.2 直接寻址（Direct Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = A（地址码就是操作数地址） |
| **访存次数** | 1次（取操作数） |
| **优点** | 简单直观 |
| **缺点** | 地址范围受限 |

```
指令:   LOAD R1, [1000]
        地址码=1000 → 去内存地址1000处取数据
```

### 5.2.3 间接寻址（Indirect Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = (A)（A指向的内存单元里存的是地址） |
| **访存次数** | **2次**（第一次取地址，第二次取操作数） |
| **优点** | 可寻址范围大，编程灵活 |
| **缺点** | 速度慢（两次访存） |

```
指令:   LOAD R1, [1000]    （间接寻址标记）
        地址码=1000
        第一步：去内存1000读出内容，假设是2000
        第二步：去内存2000读出真正的操作数
```

### 5.2.4 寄存器寻址（Register Addressing）

| 项目 | 内容 |
|------|------|
| **操作数位置** | **寄存器中** |
| **有效地址** | 就是寄存器编号 |
| **访存次数** | 0次（访问寄存器，不访存） |
| **优点** | 最快（比立即慢一点因为需先取到寄存器编号，但远快于访存） |

```
指令:   ADD R1, R2
        R1和R2中的内容直接参与运算
```

### 5.2.5 寄存器间接寻址（Register Indirect Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | EA = (R)（**寄存器中存放的是地址**） |
| **访存次数** | 1次（取操作数） |
| **优点** | 比内存间接寻址快，且可灵活改变地址 |

```
指令:   LOAD R1, [R2]
        R2中存放的是地址值（如1000）
        去内存地址1000取操作数
```

> **对比**：寄存器寻址是"寄存器中就是数据"；寄存器间接寻址是"寄存器中存的是地址，地址指向的数据才是操作数"。

### 5.2.6 相对寻址（Relative Addressing）⭐必考

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (PC) + A** |
| **访存次数** | 1次 |
| **用途** | **转移指令**（如JMP、JZ、JNZ等） |
| **特点** | 与当前指令位置相关，代码可浮动 |

```
指令:   JMP +50
        目标地址 = 当前PC值 + 50
        （注意：PC在取指阶段已自动加1！）
```

> **详细计算见5.3节**

### 5.2.7 基址寻址（Base Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (BR) + A**（BR = 基址寄存器） |
| **用途** | **多道程序、程序浮动**（重定位） |
| **特点** | 基址寄存器内容由**操作系统**管理，用户不能修改 |

```
多道程序场景：
程序A装在内存地址2000开始处
程序B装在内存地址5000开始处
→ 基址寄存器存放各自的起始地址
→ 用户程序中使用"相对自己的偏移量"访问数据
→ 实际地址 = BR + 偏移量
```

> **基址寄存器对用户是透明的**（用户不可修改）

### 5.2.8 变址寻址（Indexed Addressing）

| 项目 | 内容 |
|------|------|
| **有效地址** | **EA = (IX) + A**（IX = 变址寄存器） |
| **用途** | **数组访问**（最适合！） |
| **特点** | 变址寄存器内容由**用户**修改 |

```
访问数组 int arr[10] 中的元素：
A = 数组首地址（固定）
IX不断自增 → 依次访问arr[0], arr[1], ..., arr[9]

变址寄存器自动修改（通过增量指令），适合循环遍历数组
```

### 5.2.9 偏移寻址（Offset Addressing）

**偏移寻址** = 基址寻址 + 变址寻址 + 相对寻址

这三个方式的共同特点：**有效地址 = 某个寄存器内容 + 偏移量**

> **间接寻址不属于偏移寻址！**（因为它没有"基准+偏移"的结构）

### 5.2.10 寻址方式速查对比表

| 寻址方式 | 有效地址公式 | 访存次数 | 典型用途 |
|----------|------------|---------|---------|
| **立即寻址** | 操作数在指令中 | 0 | 赋常数 |
| **直接寻址** | EA = A | 1 | 简单数据访问 |
| **间接寻址** | EA = (A) | 2 | 指针操作 |
| **寄存器寻址** | 操作数在寄存器R中 | 0 | 算数逻辑运算 |
| **寄存器间接寻址** | EA = (R) | 1 | 指针访问 |
| **相对寻址** | EA = (PC) + A | 1 | 转移指令 |
| **基址寻址** | EA = (BR) + A | 1 | 多道程序/程序浮动 |
| **变址寻址** | EA = (IX) + A | 1 | 数组访问 |
| **先变址后间址** | EA = ((IX) + A) | 2+ | 复合寻址 |

# 5.3 相对寻址计算（必考！）

## 核心公式

```
目标地址 = 当前指令地址 + 指令长度 + 位移量
```

或者等价的：

```
目标地址 = (PC) + 偏移量
```

## 为什么要加"指令长度"？

**因为PC在取指阶段已经自动加1了！**

详细过程：
1. CPU从PC指向的地址取出当前指令
2. 每取一个字节，PC自动+1（因为PC默认"指向下一个字节"）
3. 取完整个指令后，PC已指向**下一条指令的起始地址**
4. 相对寻址中的"相对"是相对于**下一条指令**而言的

## 记忆口诀

> **相对寻址不要慌，先更新PC再算账。**
> **PC指向下一条，加上偏移就是目标。**

## 典型例题

### 例题1：基础计算

**题目**：某指令地址为1000H，指令长度为4字节，位移量为8，则转移目标地址为？

**计算**：
```
目标地址 = 1000H + 4 + 8 = 100CH
```

### 例题2：带符号位移量

**题目**：当前指令地址为2000H，指令长度为2字节，位移量为 -10（补码表示），目标地址为？

**计算**：
```
目标地址 = 2000H + 2 + (-10)
        = 2002H - 0AH
        = 1FF8H
```

### 例题3：位移量取值范围

**题目**：如果位移量字段为8位补码，其取值范围是？

**解析**：
- 8位补码范围：-128 ~ +127
- 向前最多跳 -128（向低地址）
- 向后最多跳 +127（向高地址）
- 注意：如果指令长度为2字节，可跳转范围为：
  - 最小：当前地址 + 2 + (-128) = 当前地址 - 126
  - 最大：当前地址 + 2 + (+127) = 当前地址 + 129

# 5.4 指令格式设计

## 设计步骤

**第一步**：根据指令条数确定操作码位数
- n位操作码最多表示 $2^n$ 条指令
- 若需要 m 条指令 → 操作码位数 ≥ $\lceil \log_2{m} \rceil$

**第二步**：确定地址码位数
- 地址码位数决定寻址范围
- 若地址码为 n 位，直接寻址可访问 $2^n$ 个存储单元

**第三步**：计算指令字长
- 定长指令字长 = 操作码位数 + 各地址字段位数之和

**第四步**：字长对齐
- 指令字长必须是 **8的倍数**（字节的整数倍）
- 若不足，向上取整

## 综合例题

> **题目**：某计算机要求设计如下指令格式：
> - 三地址指令 29 条
> - 二地址指令 107 条
> - 一地址指令 50 条
> - 地址字段均为 6 位
> - 求指令字长至少为多少位？

**解析**：

```
三地址指令29条：
  操作码至少5位（2^5=32 ≥ 29）
  剩余编码数 = 32-29 = 3（作为扩展窗口）
  地址总位数 = 6+6+6 = 18

二地址指令107条：
  扩展窗口3个，每个窗口可表示2^6=64条
  最多可表示 3×64 = 192条
  需要107条 → 够用
  实际用掉 107/64 向上取整 = 2个扩展窗口 → 剩余1个窗口

一地址指令50条：
  剩余1个窗口，可表示64条
  需要50条 → 够用

指令字长 = 5 + 18 = 23位
向上取整到8的倍数 → 24位
```

**答案**：指令字长至少 **24位**

# 5.5 ISA（指令集体系结构）

## 什么是ISA？

ISA是**Instruction Set Architecture**的缩写，是**程序员/编译器**能看到的那部分计算机体系结构。

## ISA包含的内容

| 包含 | 说明 |
|------|------|
| ① **指令格式** | 指令的编码方式、长度 |
| ② **操作类型** | 支持哪些操作（算术、逻辑、转移、访存等） |
| ③ **寻址方式** | 如何找到操作数 |
| ④ **操作数类型** | 支持的数据类型（整数、浮点、字符等） |
| ⑤ **寄存器定义** | 通用寄存器、专用寄存器数量与功能 |

## ISA不包含的内容

| 不包含 | 说明 |
|-------|------|
| **每条指令执行时具体的控制信号** | 这是**微体系结构**（Microarchitecture）的事 |
| 数据通路的具体实现 | 同一ISA可以有不同实现 |
| 流水线级数 | Pentium和Core i9都是x86 ISA |
| Cache大小/策略 | 属于微体系结构 |
| 时钟频率 | 属于微体系结构 |

## 关键区分

```
ISA（指令集体系结构）
    ↓ 同一个ISA可以有多种实现
微体系结构（Microarchitecture）
    ↓ 不同的实现
物理芯片
```

**经典类比**：ISA好比"驾照考试规则"，微体系结构好比"你怎么练车"——规则相同，练习方法可以不同。

## CISC vs RISC（ISA的两种风格）

| 特性 | CISC（复杂） | RISC（精简） |
|------|------------|------------|
| 指令数量 | 多（数百条） | 少（几十条） |
| 指令格式 | 变长 | 定长 |
| 寻址方式 | 多（十几种） | 少（几种） |
| 访存方式 | 算术指令可直接访存 | **LOAD/STORE结构**（只有LOAD/STORE访存） |
| 控制方式 | 微程序控制 | 硬布线控制 |
| 典型代表 | x86 | ARM, RISC-V, MIPS |

## Q63：三地址+二地址指令字长

**题目**：某计算机指令字长32位，三地址指令29条、二地址指令m条，地址字段均为6位。若采用扩展操作码方式，则m的最大值为？

**解析**：

```
三地址指令29条：
  操作码至少5位（2^5=32 ≥ 29）
  剩余编码 = 32-29 = 3个（作为扩展窗口）

三地址指令格式：OP(5) + A1(6) + A2(6) + A3(6) = 23位
  剩余32-23=9位未用... 

  等等，这里指令字长32位，不是由操作码决定的。
  重新分析：

  指令字长32位固定。
  三地址指令：地址码总位数 = 6+6+6 = 18位
  三地址指令操作码位数 = 32 - 18 = 14位
  三地址指令29条，用掉29个编码
  剩余编码空间 = 2^14 - 29 个编码

  二地址指令：地址码位数 = 6+6 = 12位
  操作码位数 = 32 - 12 = 20位
  但操作码扩展需要"短操作码的编码不是长操作码的前缀"

  更准确的计算：
  三地址29条 → 操作码14位（32-6-6-6=14）
  剩余编码数 = 2^14 - 29
  每个剩余编码可以扩展出 2^12 条二地址指令（因为地址码少了6位）

  所以m最大值 = (2^14 - 29) × 2^6

  Wait, let me think more carefully.

  操作码扩展的视角：
  三地址指令：操作码14位，用了29个编码
  剩下的编码：2^14 - 29个
  这些剩下的编码，作为二地址指令的"前缀"
  二地址指令操作码 = 14 + 6 = 20位（因为少了一个地址字段，多出的6位加入操作码）
  每个三地址剩下的编码，对应 2^6 条二地址指令

  所以 m_max = (2^14 - 29) × 2^6

  但实际上，二地址也要预留扩展编码给一地址...
  题目问的是m的最大值，所以先不预留。
```

**答案**：m的最大值 = (2^14 - 29) x 2^6

## Q64：寻址范围

**题目**：某计算机地址码字段为16位，采用直接寻址方式，则其寻址范围为多少？

**解析**：
- 直接寻址：EA = A，地址码就是有效地址
- 地址码16位 → 可以表示 $2^{16}$ 个不同的地址
- 寻址范围 = $2^{16} = 65536$ 个存储单元

**答案**：64K（或65536个存储单元）

## Q65：先变址后间址

**题目**：某计算机采用先变址后间址的寻址方式，指令中给出的形式地址为D，变址寄存器为IX，则该指令操作数的有效地址为？

**解析**：

"先变址后间址" = 先做变址运算，再做间接寻址

步骤拆解：
1. 先变址：计算临时地址 = (IX) + D
2. 后间址：以临时地址为指针，去内存读取真正的地址
3. 有效地址 EA = ((IX) + D)

**答案**：EA = ((IX) + D)

## Q66：相对寻址转移

**题目**：某指令地址为2000H，指令长度为4字节，采用相对寻址转移到地址2010H，则指令中的位移量为多少？

**解析**：

```
目标地址 = 当前指令地址 + 指令长度 + 位移量
2010H = 2000H + 4 + 位移量
位移量 = 2010H - 2000H - 4
       = 10H - 4
       = 16 - 4
       = 12 = 0CH
```

**答案**：位移量 = 0CH （即十进制的12）

## Q67：不属于偏移寻址

**题目**：以下哪种寻址方式不属于偏移寻址？
A. 基址寻址  B. 变址寻址  C. 相对寻址  D. 间接寻址

**解析**：

偏移寻址的三个成员：
- 基址寻址：EA = (BR) + A ✓
- 变址寻址：EA = (IX) + A ✓
- 相对寻址：EA = (PC) + A ✓

间接寻址：EA = (A) — 没有"基准+偏移"结构 ✗

**答案**：D. 间接寻址

## Q68：变址寻址

**题目**：变址寻址方式最适合用于？
A. 实现程序浮动  B. 实现多道程序  C. 处理数组问题  D. 实现程序转移

**解析**：
- 变址寻址：EA = (IX) + A
- IX可自动修改，适合循环遍历数组元素
- A固定为数组首地址，IX变化依次访问不同元素

| 选项 | 对应寻址方式 |
|------|------------|
| 程序浮动/重定位 | 基址寻址 |
| 多道程序 | 基址寻址 |
| **数组访问** | **变址寻址** |
| 程序转移 | 相对寻址 |

**答案**：C. 处理数组问题

## Q69：数组访问寻址

**题目**：以下哪种寻址方式最适合按下标顺序访问一维数组元素？
A. 直接寻址  B. 变址寻址  C. 寄存器寻址  D. 间接寻址

**解析**：
- 一维数组元素在内存中连续存放
- 变址寻址中，变址寄存器可自动修改（增量）
- 形式地址 = 数组首地址，变址寄存器 = 下标
- 每次循环变址寄存器+1，就能访问下一个元素

**答案**：B. 变址寻址

## Q70：基址+大端

**题目**：某计算机采用基址寻址，基址寄存器BR=1000H，形式地址为数据在结构体中的偏移量。结构体成员的偏移地址分别为：A偏移0，B偏移2，C偏移4。采用大端方式存储，变量A的值为0x1234，变量B的值为0x5678。问执行指令 LOAD R1, 2(BR) 后，R1中的内容为？

**解析**：

```
基址寻址：EA = (BR) + A = 1000H + 2 = 1002H

偏移量A=2对应结构体中的B成员

大端存储（Big Endian）：
  高字节存低地址，低字节存高地址

变量B的值为0x5678，占2个字节
在大端模式下：
  地址1002H 存 0x56（高字节）
  地址1003H 存 0x78（低字节）

LOAD R1, 2(BR) 从地址1002H读取一个字（假设字长为16位）
R1 = 0x5678
```

**答案**：R1 = 0x5678

## Q71：数组+变址

**题目**：已知数组A的首地址为2000H，每个元素占2字节。变址寄存器IX初始值为0。采用变址寻址方式，执行指令 LOAD R1, A(IX) 三次后（每次IX自增2），R1最终读到的值为？

**解析**：

```
第一次：EA = (IX) + A = 0 + 2000H = 2000H → 读A[0]
        然后IX自增2 → IX=2

第二次：EA = 2 + 2000H = 2002H → 读A[1]
        然后IX自增2 → IX=4

第三次：EA = 4 + 2000H = 2004H → 读A[2]
        R1最终 = 内存地址2004H处的内容 = A[2]的值
```

**答案**：R1中为 A[2] 的值（即地址2004H处的内容）

## Q72-Q81：ISA概念题

### Q72

**题目**：以下哪个不属于ISA（指令集体系结构）的范畴？
A. 指令格式  B. 寻址方式  C. 控制信号  D. 操作数类型

**解析**：
ISA定义的是**程序员可见**的内容：
- 指令格式 ✓
- 寻址方式 ✓
- 操作类型 ✓
- 操作数类型 ✓
- 寄存器 ✓

**控制信号**是微体系结构（控制器）的设计内容，ISA不涉及。

**答案**：C. 控制信号

### Q73

**题目**：ISA规定的内容不包括？
A. 指令的操作码  B. 寄存器的数量和功能  C. 指令的执行周期  D. 数据存储格式

**解析**：
- 操作码：ISA的一部分 ✓
- 寄存器数量和功能：ISA的一部分 ✓
- 数据存储格式（大端/小端）：ISA的一部分 ✓
- **指令的执行周期**（需要几个时钟周期）：属于微体系结构 ✗

**答案**：C. 指令的执行周期

### Q74

**题目**：以下关于ISA的描述，正确的是？
A. ISA包括CPU的时钟频率
B. ISA不包括指令的编码格式
C. ISA是机器语言程序员看到的计算机属性
D. ISA的实现方式固定不变

**解析**：
- A：时钟频率属于微体系结构 ✗
- B：指令编码格式是ISA的核心部分 ✗
- C：ISA正是**程序员/编译器**看到的计算机属性 ✓
- D：同一ISA可以有不同实现 ✗

**答案**：C

### Q75

**题目**：指令集体系结构（ISA）不包含以下哪个？
A. 指令格式  B. 数据类型  C. 寄存器组织  D. Cache容量

**解析**：
- ISA包含：指令格式、数据类型（操作数类型）、寄存器组织 ✓
- Cache容量属于**微体系结构**的设计选择 ✗

**答案**：D. Cache容量

### Q76

**题目**：以下关于RISC说法正确的是？
A. 指令格式可变
B. 只有LOAD和STORE指令可以访问内存

D. 指令条数较多

**解析**：
RISC特点：
- 指令**定长** ✓ → A错

- 硬布线控制为主（不是微程序） → C错
- 指令条数少 → D错

### Q77

A. 指令系统复杂
B. 指令格式可变
C. 寻址方式丰富
D. 采用硬布线控制器

**解析**：
CISC特点：
- 指令复杂 ✓
- 变长指令 ✓
- 寻址方式多 ✓
- **微程序控制**为主（不是硬布线）→ D错

**答案**：D

### Q78

**题目**：以下哪个是RISC的特点？
A. 指令格式可变长
B. 使用微程序控制
C. 指令使用频率差别大
D. 优化的编译系统

RISC特点：
- **指令定长** → A错
- **硬布线控制**为主 → B错
- 指令使用频率差别大是CISC的特点（所以引入RISC）→ C错
- RISC需要**优化的编译系统**来有效利用寄存器 ✓ → D对

### Q79

**题目**：ISA中规定了什么？
A. 指令具体需要几个时钟周期

- 控制信号的产生：微体系结构 ✗
- **指令的二进制编码格式**：ISA的核心内容 ✓
- Cache替换策略：微体系结构 ✗

**答案**：C

### Q80

**题目**：以下哪个不是指令集体系结构设计的内容？
A. 确定指令的操作码编码

### Q81

**题目**：以下关于指令集体系结构（ISA）的说法中，正确的是？

- D：执行时间不是ISA内容 ✗

# 附录：速记卡片

## 一、寻址方式速记

```
立即     — 操作数在指令中
直接     — 地址码是地址
间接     — 地址码指向的单元存着地址（两次访存）
寄存器   — 操作数在寄存器（最快）
寄存器间 — 寄存器存的是地址（一次访存）
相对     — PC + 偏移量（转移专用）
基址     — BR + 偏移量（系统管理，程序浮动）
变址     — IX + 偏移量（用户修改，数组访问）
偏移寻址 — 基址+变址+相对（不含间接）
```

## 二、相对寻址公式

```
目标地址 = 当前指令地址 + 指令长度 + 位移量
```

## 三、ISA包含/不包含

| 包含 | 不包含（微体系结构的） |
|------|---------------------|
| 指令格式 | 控制信号 |
| 操作类型 | 数据通路实现 |
| 寻址方式 | 流水线级数 |
| 操作数类型 | Cache大小 |
| 寄存器定义 | 时钟频率 |

## 四、RISC vs CISC 速记

| RISC | CISC |
|------|------|
| 定长指令 | 变长指令 |
| LOAD/STORE | 可直接访存 |
| 硬布线控制 | 微程序控制 |
| 少量寻址方式 | 丰富寻址方式 |
| 适合流水线 | 不易流水线 |
